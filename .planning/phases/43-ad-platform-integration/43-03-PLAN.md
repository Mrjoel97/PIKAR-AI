Create new file with tools that bridge agents to real ad platform APIs via the approval gate.

**Tools to create (raw async functions following existing pattern):**

1. `connect_google_ads_status()` -- Check if Google Ads is connected. Returns connection status and customer_id.

2. `connect_meta_ads_status()` -- Same for Meta Ads.

3. `list_google_ads_campaigns()` -- List campaigns from Google Ads API. No approval needed.

4. `list_meta_ads_campaigns()` -- Same for Meta Ads.

5. `create_google_ads_campaign(name, daily_budget, campaign_type="SEARCH")` -- Check budget cap headroom. Create in PAUSED status. Also create local ad_campaigns record with platform_campaign_id. Return note about activation needing approval.

6. `create_meta_ads_campaign(name, daily_budget, objective="OUTCOME_TRAFFIC")` -- Same pattern for Meta.

7. `activate_ad_campaign(platform, campaign_name, ad_campaign_id)` -- GATED operation. Call AdApprovalService.check_and_gate("activate_campaign", ...). Return approval card data if approval required.

8. `pause_ad_campaign(platform, ad_campaign_id)` -- NON-GATED. Execute immediately via platform service. Update local record.

9. `change_ad_budget(platform, campaign_name, ad_campaign_id, new_daily_budget)` -- GATED if increase, immediate if decrease.

10. `get_ad_campaign_performance(platform, ad_campaign_id, days=7)` -- NON-GATED read-only. Query ad_spend_tracking.

11. `refresh_ad_performance(platform)` -- Trigger on-demand sync via AdPerformanceSyncService.

12. `get_ad_budget_cap(platform)` -- Get current monthly cap.

13. `set_ad_budget_cap(platform, monthly_cap)` -- Set/update monthly cap.

All tools use lazy imports and get_current_user_id(). Export as AD_PLATFORM_TOOLS list.
