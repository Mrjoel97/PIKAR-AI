**File 1: app/services/ad_approval_service.py** -- New file.

Create `AdApprovalService`:

1. Define which operations require approval:
```python
GATED_OPERATIONS = {
    "activate_campaign",      # draft -> active (starts spending)
    "resume_campaign",        # paused -> active (re-starts spending)
    "increase_daily_budget",  # daily budget goes up
    "increase_total_budget",  # total budget goes up
    "increase_bid_amount",    # bid goes up
    "change_bid_strategy",    # could increase spend
}

NON_GATED_OPERATIONS = {
    "create_draft_campaign",   # no money moves
    "pause_campaign",          # stops spending
    "decrease_budget",         # reduces spending
    "update_targeting",        # on paused campaigns
    "update_creative",         # on paused campaigns
}
```

2. `async def request_budget_approval(self, user_id: str, operation: str, platform: str, campaign_name: str, details: dict) -> dict` --
   - Validate operation is in GATED_OPERATIONS
   - Build approval payload with: operation, platform, campaign_name, current_budget, new_budget, projected_monthly_impact (daily × 30), cap_headroom (from AdBudgetCapService.check_budget_headroom)
   - Call existing `request_human_approval()` with action_type="AD_BUDGET_CHANGE", enriched payload
   - Return structured dict: {"approval_required": True, "approval_id": id, "message": message, "card_data": {...}} where card_data includes all fields for frontend rendering

3. `async def execute_approved_operation(self, approval_id: str) -> dict` --
   - Fetch approval_requests row by id
   - Validate status is "APPROVED"
   - Parse payload to determine operation
   - Call the appropriate GoogleAdsService or MetaAdsService method
   - Update local ad_campaigns table via AdCampaignService
   - Return result

4. `async def is_operation_gated(self, operation: str) -> bool` -- Check if operation requires approval.

5. `async def check_and_gate(self, user_id: str, operation: str, platform: str, campaign_name: str, details: dict) -> dict` --
   - If operation not in GATED_OPERATIONS: return {"approval_required": False, "execute": True}
   - If budget cap would be exceeded: return {"approval_required": False, "execute": False, "blocked": True, "message": cap_exceeded_message}
   - Otherwise: return result of request_budget_approval()

**File 1: app/services/ad_performance_sync_service.py** -- New file.

Create `AdPerformanceSyncService`:

1. `async def sync_all_users(self) -> dict` -- Main entry point for scheduled sync:
   - Query integration_credentials for all users with google_ads or meta_ads provider where credentials are not expired
   - For each user+provider pair, call sync_user_platform()
   - Return summary: {"users_synced": N, "platforms_synced": N, "errors": [...]}

2. `async def sync_user_platform(self, user_id: str, platform: str) -> dict` --
   - If platform == "google_ads": call _sync_google_ads(user_id)
   - If platform == "meta_ads": call _sync_meta_ads(user_id)
   - After sync: call _check_budget_pacing(user_id, platform)
   - Return {"platform": platform, "records_synced": N}

3. `async def _sync_google_ads(self, user_id: str) -> int` --
   - Get GoogleAdsService, get customer_id from integration_credentials account_name or metadata
   - Pull performance for last 7 days (covers Google's ~3hr reporting delay + catches late conversions)
   - For each campaign+date row: find matching ad_campaign by platform_campaign_id, call AdSpendTrackingService.record_daily_spend()
   - Skip campaigns paused >30 days (check updated_at on ad_campaigns)
   - Return count of records synced

4. `async def _sync_meta_ads(self, user_id: str) -> int` --
   - Same pattern as Google but using MetaAdsService.get_campaign_insights()
   - Meta budgets are in cents — divide by 100 when storing spend
   - Return count

5. `async def _check_budget_pacing(self, user_id: str, platform: str) -> None` --
   - Get monthly cap from AdBudgetCapService
   - Sum total spend this month for the platform (query ad_spend_tracking for current month)
   - Calculate daily average spend rate
   - Project when cap would be reached at current rate
   - If projected to exceed cap before month end:
     ```python
     from app.notifications.notification_service import NotificationService, NotificationType
     notif_svc = NotificationService()
     await notif_svc.create_notification(
         user_id=user_id,
         title="Budget Pacing Alert",
         message=f"{platform_name} is spending ${daily_avg:.0f}/day — at this rate you'll hit your ${cap:,.0f}/mo cap by the {projected_date}.",
         type=NotificationType.WARNING,
         link="/dashboard/configuration",
         metadata={"platform": platform, "daily_avg": daily_avg, "monthly_cap": cap},
     )
     ```

6. `async def sync_user_on_demand(self, user_id: str, platform: str) -> dict` -- Same as sync_user_platform but for on-demand refresh from agent or API call.
