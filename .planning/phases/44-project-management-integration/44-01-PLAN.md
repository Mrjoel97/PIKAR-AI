**File 1: app/services/linear_service.py** — New file, httpx-based GraphQL client.

Create `LinearService(BaseService)`:

1. `async def _graphql(self, user_id: str, query: str, variables: dict = None) -> dict` — POST to `https://api.linear.app/graphql` with Bearer token from IntegrationManager.get_valid_token(user_id, "linear"). Return `data` from response JSON. Raise on errors.

2. `async def list_teams(self, user_id: str) -> list[dict]` — Query `{ teams { nodes { id name key description } } }`. Return list of team dicts.

3. `async def list_issues(self, user_id: str, team_id: str, updated_after: str = None, limit: int = 100) -> list[dict]` — Query issues for team with fields: id, identifier, title, description, state { id name type }, priority, assignee { name }, labels { nodes { name } }, url, updatedAt. Optional `updatedAt` filter for incremental sync. Paginate with `after` cursor.

4. `async def create_issue(self, user_id: str, team_id: str, title: str, description: str = "", priority: int = 0, state_id: str = None) -> dict` — Mutation `issueCreate(input: {...})`. Return created issue.

5. `async def update_issue(self, user_id: str, issue_id: str, title: str = None, description: str = None, state_id: str = None, priority: int = None) -> dict` — Mutation `issueUpdate(id: $id, input: {...})`. Return updated issue.

6. `async def list_workflow_states(self, user_id: str, team_id: str) -> list[dict]` — Query `{ workflowStates(filter: { team: { id: { eq: "$team_id" } } }) { nodes { id name type position } } }`. Return sorted by position. Used for status mapping UI.

All calls use `httpx.AsyncClient` with 30s timeout. Lazy import for httpx.

**File 2: app/services/asana_service.py** — New file, httpx-based REST client.

Create `AsanaService(BaseService)`:

1. `async def _get(self, user_id: str, path: str, params: dict = None) -> dict` — GET `https://app.asana.com/api/1.0{path}` with Bearer token. Return `data` from response.

2. `async def _post(self, user_id: str, path: str, body: dict) -> dict` — POST with JSON body. Return `data`.

3. `async def _put(self, user_id: str, path: str, body: dict) -> dict` — PUT with JSON body. Return `data`.

4. `async def list_workspaces(self, user_id: str) -> list[dict]` — GET `/workspaces`. Return list.

5. `async def list_projects(self, user_id: str, workspace_id: str) -> list[dict]` — GET `/projects?workspace={id}&opt_fields=name,color,archived`. Filter out archived.

6. `async def list_tasks(self, user_id: str, project_id: str, modified_since: str = None, limit: int = 100) -> list[dict]` — GET `/tasks?project={id}&opt_fields=name,notes,completed,assignee.name,memberships.section.name,permalink_url,modified_at`. Optional `modified_since` filter. Paginate with `offset`.

7. `async def create_task(self, user_id: str, project_id: str, name: str, notes: str = "", assignee: str = None) -> dict` — POST `/tasks` with `{ data: { name, notes, projects: [project_id], assignee } }`.

8. `async def update_task(self, user_id: str, task_id: str, name: str = None, notes: str = None, completed: bool = None) -> dict` — PUT `/tasks/{id}` with updated fields.

9. `async def list_sections(self, user_id: str, project_id: str) -> list[dict]` — GET `/sections?project={id}`. Used for status mapping.

10. `async def move_task_to_section(self, user_id: str, section_id: str, task_id: str) -> None` — POST `/sections/{section_id}/addTask` with `{ data: { task: task_id } }`. Used for Asana status changes (sections = statuses).

**File 3: app/services/pm_sync_service.py** — New file, bidirectional sync orchestrator.

Create `PMSyncService(BaseService)`:

1. **Default status mappings:**
```python
LINEAR_DEFAULT_MAPPINGS: ClassVar[dict[str, str]] = {
    "triage": "pending", "backlog": "pending", "unstarted": "pending",
    "started": "in_progress", "completed": "completed", "cancelled": "cancelled",
}
# Maps Linear state.type to pikar_status
```

2. `async def seed_status_mappings(self, user_id: str, provider: str, states: list[dict]) -> None` — For each workflow state (from Linear) or section (from Asana), insert default mapping into pm_status_mappings if not already set. Uses AdminService for writes.

3. `async def get_status_mapping(self, user_id: str, provider: str) -> dict[str, str]` — Query pm_status_mappings for user+provider. Return dict of external_state_id → pikar_status.

4. `async def map_external_to_pikar(self, user_id: str, provider: str, external_state_id: str) -> str` — Look up mapping, return pikar_status. Default to "pending" if unmapped.

5. `async def map_pikar_to_external(self, user_id: str, provider: str, pikar_status: str) -> str | None` — Reverse lookup: find external_state_id that maps to this pikar_status. Return first match. Used for Pikar→PM status push.

6. `async def sync_from_external(self, user_id: str, provider: str, external_issue: dict) -> dict` — 
   - Check Redis skip-flag `pikar:pm:skip:{provider}:{external_id}` — if set, skip
   - Map external status to pikar_status via status mapping
   - Upsert to synced_tasks with on_conflict="user_id,provider,external_id"
   - Set Redis skip-flag (30s TTL)
   - Return synced record

7. `async def sync_to_external(self, user_id: str, task_id: str, updates: dict) -> dict` — 
   - Read synced_tasks record
   - Set Redis skip-flag before API call
   - If provider == "linear": call LinearService.update_issue()
   - If provider == "asana": call AsanaService.update_task() + move_task_to_section() if status changed
   - Return result

8. `async def initial_sync(self, user_id: str, provider: str, project_ids: list[str]) -> dict` — 
   - For each project: fetch issues updated in last 30 days
   - Call sync_from_external for each
   - Return {"synced": count, "errors": error_count}

9. `async def get_sync_config(self, user_id: str, provider: str) -> dict` — Read sync_cursor from integration_sync_state.

10. `async def save_sync_config(self, user_id: str, provider: str, project_ids: list[str]) -> dict` — Save project_ids to sync_cursor JSONB. Seed status mappings. Trigger initial sync.

**File 4: app/routers/integrations.py** — Add new endpoints:

3. `GET /integrations/{provider}/sync-config` — Return current sync config (selected projects).

4. `GET /integrations/{provider}/status-mappings` — Return current status mappings for user+provider.

5. `PUT /integrations/{provider}/status-mappings` — Body: `[{"external_state_id": "...", "pikar_status": "..."}]`. Update custom status mappings.
