Linear webhook payload:
```json
{
  "action": "create" | "update" | "remove",
  "type": "Issue",
  "data": { "id": "...", "title": "...", "state": {"id": "...", "name": "...", "type": "..."}, ... },
  "organizationId": "..."
}
```
Signature: HMAC-SHA256 of body with webhook signing secret in `Linear-Signature` header.

Asana webhook:
- First call: responds with X-Hook-Secret header (handshake)
- Subsequent: POST with `{"events": [{"action": "changed", "resource": {"gid": "...", "resource_type": "task"}, "change": {...}}]}`
- Signature: HMAC-SHA256 in `X-Hook-Signature` header using shared secret

  <action>
Add two new webhook endpoints to the existing webhooks router:

**1. Linear webhook handler:**

```python
@router.post("/webhooks/linear")
async def linear_webhook(request: Request) -> dict:
```

- Read raw body bytes
- Verify HMAC-SHA256: compute `hmac.new(signing_secret, body, sha256).hexdigest()` and compare with `Linear-Signature` header
- The signing secret comes from: look up all users with Linear credentials, use the webhook secret from the Linear app config (stored in env `LINEAR_WEBHOOK_SECRET`)
- Parse JSON payload: check `type == "Issue"` and `action in ("create", "update")`
- Extract issue data: id, title, description, state (id, name, type), priority, assignee, labels, url
- Resolve user_id: query `integration_credentials` for the `organizationId` in payload (store org_id in account_name or metadata during OAuth callback)
- Check if issue's team_id is in user's synced_projects config — skip if not
- Build external_issue dict and call `PMSyncService().sync_from_external(user_id, "linear", external_issue)`
- Return `{"ok": True}`
- For `action == "remove"`: update synced_tasks status to "cancelled"

**2. Asana webhook handler:**

```python
@router.post("/webhooks/asana")
async def asana_webhook(request: Request) -> dict:
```

- **Handshake:** If `X-Hook-Secret` header present, respond with 200 and `X-Hook-Secret` header echoed back. Store the secret for future verification.
- **Events:** Read body, verify HMAC-SHA256 with `X-Hook-Signature` header using stored hook secret
- Parse events array: for each event where `resource.resource_type == "task"` and `action in ("changed", "added")`
  - Fetch full task data from Asana API (webhook only sends GID, not full task)
  - Resolve user_id from webhook subscription metadata
  - Check if task's project is in synced_projects — skip if not
  - Map Asana section to status via status mapping
  - Build external_issue dict and call `PMSyncService().sync_from_external(user_id, "asana", external_issue)`
- Return `{"ok": True}`

Follow the same error handling pattern as existing Stripe/Shopify/HubSpot webhook handlers. Log webhook events. Use AdminService for DB operations since webhooks have no user JWT.
