Output: Supabase migration with 3 webhook tables, event catalog module, webhook delivery service, extended webhook router, worker loop integration, unit tests.

  <action>
    1. **Webhook delivery service** (`app/services/webhook_delivery_service.py`):

       Constants:
       - `RETRY_BACKOFF_SECONDS = [1, 5, 30, 300, 1800]` (1s, 5s, 30s, 5min, 30min)
       - `MAX_ATTEMPTS = 5`
       - `CIRCUIT_BREAKER_THRESHOLD = 10` (consecutive failures before disabling)

       Functions:

       **`async def enqueue_webhook_event(event_type: str, payload: dict) -> int`**:
       - Get service client
       - Query `webhook_endpoints` where `active = true` and `event_type = ANY(events)`
       - For each matching endpoint, insert a row into `webhook_deliveries` with status "pending", `next_retry_at = now()`
       - Return count of deliveries enqueued
       - Use service role client (system operation, not user-scoped)

       **`async def run_webhook_delivery_tick() -> list[dict]`**:
       - Get service client
       - Fetch deliveries where `status IN ('pending', 'failed')` AND `next_retry_at <= now()` AND `attempts < 5`, joined with `webhook_endpoints(*)`, ordered by `created_at`, limit 50
       - For each delivery, call `_deliver_single(service_client, delivery)`
       - Return list of delivery results

       **`async def _deliver_single(client, delivery: dict) -> dict`**:
       - Extract endpoint URL, secret (decrypt with `decrypt_secret`)
       - Compute HMAC-SHA256 signature of JSON payload using endpoint secret
       - POST to endpoint URL with headers: `Content-Type: application/json`, `X-Pikar-Signature: sha256={hex_signature}`, `X-Pikar-Event: {event_type}`, `X-Pikar-Delivery: {delivery_id}`
       - Use httpx with 10-second timeout
       - On 2xx response:
         - Update delivery: `status = "delivered"`, `attempts += 1`, `response_code`, `response_body` (truncated to 1000 chars)
         - Reset endpoint: `consecutive_failures = 0`
       - On non-2xx or exception:
         - Increment `attempts`
         - If `attempts >= MAX_ATTEMPTS`: set `status = "dead"`
         - Else: set `status = "failed"`, `next_retry_at = now() + RETRY_BACKOFF_SECONDS[attempts - 1]`
         - Store `response_code` and `response_body` (or error message)
         - Increment endpoint `consecutive_failures`
         - If `consecutive_failures >= CIRCUIT_BREAKER_THRESHOLD`: set endpoint `active = false`, `disabled_at = now()`
       - Return `{delivery_id, status, attempts, response_code}`
