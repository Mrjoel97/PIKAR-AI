  <action>
    Create `app/agents/admin/tools/diagnosis.py`:
    - Single async function `diagnose_user_problem(user_id: str)` with autonomy check via `_check_autonomy("diagnose_user_problem")`.
    - Uses `get_service_client()` and `execute_async()` (same pattern as users_intelligence.py).
    - Runs 4 parallel diagnostic checks via `asyncio.gather`:
      1. OAuth check: query `user_mcp_integrations` for the user_id where is_active=True, flag any where token status suggests expiration (check if tokens exist)
      2. API health: query latest row per endpoint from `api_health_checks` (reuse pattern from monitoring.py get_api_health_summary), flag non-healthy
      3. Budget caps: query `ad_budget_caps` for user_id where current_spend_usd >= monthly_cap_usd
      4. Pending approvals: query `governance_approvals` for user_id where status='pending'
    - Aggregate results into a structured dict with `issues` list, each issue having `category`, `severity` (critical/warning/info), `details`, and `recommended_action`.
    - Generate `plain_english_summary` string from issues list. When no issues: "All systems look good for this user. OAuth integrations are connected, platform health is normal, budget caps are within limits, and there are no pending approvals."
    - When issues found: "I found {N} issue(s) that may be affecting this user: {bullet list of issues with recommended actions}"

    Create `app/services/feature_adoption_service.py`:
    - Class `FeatureAdoptionService(AdminService)` with method `async def compute_adoption(self, days: int = 30, user_id: str | None = None)`.
    - Queries `tool_telemetry` for the last N days, optionally filtered by user_id.
    - Groups by (agent_name, tool_name) to produce per-agent adoption:
      - `agent_name`: which agent
      - `unique_tools_used`: count of distinct tool_name values
      - `total_calls`: sum of calls
      - `top_tools`: list of {tool_name, call_count} top 5 by calls
      - `unique_users`: count of distinct user_ids (only when user_id filter is None)
    - Returns `{"agent_adoption": [...], "total_agents_active": int, "total_unique_tools": int, "period_days": int}`

    Create `app/agents/admin/tools/adoption.py`:
    - Single async function `get_feature_adoption(days: int = 30, user_id: str | None = None)` with autonomy check.
    - Instantiates `FeatureAdoptionService()` and calls `compute_adoption()`.
    - Returns the service result directly.

    Write tests in `tests/unit/admin/test_diagnosis_tool.py` and `tests/unit/admin/test_adoption_tool.py`:
    - Follow the mock pattern from `test_user_intelligence_tools.py` — patch `app.agents.admin.tools.diagnosis._check_autonomy` to return None, mock `execute_async` with fixture data for each diagnostic signal.
    - Test each diagnostic path independently (OAuth issues, health issues, budget issues, approval issues, all-clear).
    - For adoption: mock `execute_async` returning tool_telemetry rows, verify grouping logic.
