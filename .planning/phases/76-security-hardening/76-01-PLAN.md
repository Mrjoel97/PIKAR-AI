<objective>
Enforce webhook signing-secret requirements for Linear and Asana endpoints, and harden the auth header resolution function to prevent user-id spoofing via request headers.

Purpose: Close two security gaps where (1) unauthenticated webhook payloads are processed when signing secrets are misconfigured, and (2) x-user-id headers can bypass JWT authentication.

Output: Hardened webhook handlers that fail-closed when secrets are absent, and a resolve_request_user_id function that defaults to rejecting header-based identity.
</objective>
