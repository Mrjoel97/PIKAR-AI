## Decisions Made

- **upload_knowledge is post-upload confirmation, not binary receiver.** Files go via REST endpoint. Tool reads entry by entry_id from DB and returns status — clean separation between HTTP multipart handling and AgentTool interface.
- **search_system_knowledge passes agent_name=None intentionally** — returns all system-scoped entries (global + any agent-scoped). Full knowledge access per KNOW-06 design; agent-specific filtering can be added later if needed.
- **Deferred import in system_knowledge.py** — `from app.services.knowledge_service import _search` inside function body avoids circular import. Tests must patch `app.services.knowledge_service.search_system_knowledge`, not the wrapper module path.
- **SKIL-09 validate_knowledge_relevance uses keyword intersection** — domain keyword frozensets defined in-module for all 10 agent domains; no LLM call, pure deterministic keyword matching for speed and reliability.
- **REST upload returns JSONResponse for status code control** — FastAPI `@router.post` return type annotation can't express conditional 200/202; `JSONResponse(status_code=202)` for video is the correct pattern.
