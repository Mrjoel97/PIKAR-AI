    **Tool 1: upload_knowledge (confirm tier)**
    - Params: entry_id (str), filename (str), mime_type (str), agent_scope (str|None=None), confirmation_token (str|None=None)
    - Gate: _check_autonomy("upload_knowledge")
    - NOTE: The actual file upload goes through the REST endpoint (multipart POST). This tool is called AFTER the file is already stored. It receives the entry_id returned by the upload endpoint and triggers the processing pipeline.
    - Dispatch to process_document / process_image / process_video based on mime_type prefix (application/pdf, image/*, video/*, text/*, application/vnd.openxmlformats*)
    - For document/image: the REST upload endpoint already called knowledge_service before this tool runs. This tool confirms the action and returns the result from the entry record.
    - ALTERNATIVE SIMPLER DESIGN: Since the REST endpoint already processes the file, this tool serves as the AdminAgent's interface to acknowledge and report the upload result. It reads the admin_knowledge_entries row by entry_id and returns its status, chunk_count, and details.

    **Tool 2: list_knowledge_entries (auto tier)**
    - Params: agent_scope (str|None=None), status (str|None=None), limit (int=20)
    - Query admin_knowledge_entries filtered by agent_scope and status, ordered by created_at DESC

    **Tool 3: search_knowledge (auto tier)**
    - Params: query (str), agent_name (str|None=None), top_k (int=5)
    - Delegate to knowledge_service.search_system_knowledge

    **Tool 4: delete_knowledge_entry (confirm tier)**
    - Params: entry_id (str), confirmation_token (str|None=None)
    - Gate: _check_autonomy("delete_knowledge_entry")
    - Delete from admin_knowledge_entries, delete associated embeddings rows where source_id matches, delete file from Storage

    **Tool 5: get_knowledge_stats (auto tier)**
    - Delegate to knowledge_service.get_knowledge_stats()

    **Tool 6: check_knowledge_duplicate (auto tier) — SKIL-09**
    - Params: text_sample (str), agent_scope (str|None=None), threshold (float=0.92)
    - Take first 500 chars of text_sample, generate embedding, call match_system_knowledge RPC with match_count=3
    - If top result similarity > threshold: return {"near_duplicate": True, "similar_entry": ..., "similarity": ...}
    - Else: return {"near_duplicate": False, "closest_similarity": ...}

    **Tool 7: validate_knowledge_relevance (auto tier) — SKIL-09**
    - Params: text_sample (str), target_agent (str)
    - Check if text_sample content is relevant to the target agent's domain by searching existing knowledge for that agent and comparing domain keywords
    - Return {"relevant": bool, "confidence": float, "reason": str}

    **Tool 8: recommend_chunking_strategy (auto tier) — SKIL-09**
    - Params: filename (str), file_size_bytes (int), mime_type (str)
    - Based on file type and size, recommend chunk_size and chunk_overlap
    - Documents >50KB: chunk_size=500, overlap=50 (standard)
    - Documents <5KB: chunk_size=300, overlap=30 (smaller chunks for short docs)
    - Large documents >500KB: warn about chunk count, suggest summarization
    - Return {"chunk_size": int, "chunk_overlap": int, "estimated_chunks": int, "warnings": list}

    **Create `app/routers/admin/knowledge.py`** following the pattern from `config.py`:

    ```python
    router = APIRouter(prefix="/knowledge")
    ```

    **Endpoint 1: POST /knowledge/upload** (multipart form)
    - Accept: UploadFile (file), Form fields: agent_scope (str, optional), uploaded_by (str, required)
    - Read file bytes from UploadFile
    - Determine file category from content_type: application/pdf, application/vnd.openxml*, text/* -> process_document; image/* -> process_image; video/* -> process_video
    - Call appropriate knowledge_service function
    - For video: return 202 status code (accepted for background processing)
    - For document/image: return 200 with result
    - Requires admin auth (use require_admin dependency from app.routers.admin.auth)

    **Endpoint 2: GET /knowledge/entries**
    - Query params: agent_scope (str, optional), status (str, optional), limit (int, default 50), offset (int, default 0)
    - Query admin_knowledge_entries with filters, return list + total count
    - Requires admin auth

    **Endpoint 3: GET /knowledge/entries/{entry_id}**
    - Return single admin_knowledge_entries row by id, 404 if not found
    - Requires admin auth

    **Endpoint 4: DELETE /knowledge/entries/{entry_id}**
    - Delete entry, its embeddings (WHERE source_id = entry_id AND metadata->>'scope' = 'system'), and Storage file
    - Requires admin auth
    - Return 200 with {"deleted": True}

    **Endpoint 5: GET /knowledge/stats**
    - Delegate to knowledge_service.get_knowledge_stats()
    - Return stats dict
    - Requires admin auth

    **Endpoint 6: GET /knowledge/search**
    - Query params: q (str, required), agent_scope (str, optional), top_k (int, default 5)
    - Delegate to knowledge_service.search_system_knowledge
    - Return list of results
    - Requires admin auth
