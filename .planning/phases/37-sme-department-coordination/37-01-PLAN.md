Create a new SQL migration for the `department_tasks` table. This table is DISTINCT from `inter_dept_requests` (which is for autonomous department-to-department coordination). `department_tasks` is for USER-initiated cross-department task handoffs visible in the SME department dashboard.

Schema:
```sql
CREATE TABLE IF NOT EXISTS department_tasks (
    id              UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
    title           TEXT        NOT NULL,
    description     TEXT,
    from_department_id UUID    NOT NULL REFERENCES departments(id),
    to_department_id   UUID    NOT NULL REFERENCES departments(id),
    created_by      UUID        NOT NULL,  -- user who created the handoff
    assigned_to     UUID,                  -- optional user assignment
    status          TEXT        NOT NULL DEFAULT 'pending'
                    CHECK (status IN ('pending', 'in_progress', 'completed', 'cancelled')),
    priority        TEXT        NOT NULL DEFAULT 'medium'
                    CHECK (priority IN ('low', 'medium', 'high', 'urgent')),
    due_date        TIMESTAMPTZ,
    completed_at    TIMESTAMPTZ,
    metadata        JSONB       NOT NULL DEFAULT '{}',
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

Add indexes:
- `idx_dept_tasks_to_dept_status` on (to_department_id, status) WHERE status IN ('pending', 'in_progress') — for department task list queries
- `idx_dept_tasks_from_dept` on (from_department_id) — for tracking outbound handoffs
- `idx_dept_tasks_created_by` on (created_by) — for user's task list
- `idx_dept_tasks_due_date` on (due_date) WHERE status IN ('pending', 'in_progress') — for overdue detection

Enable RLS. Add policies:
- "Authenticated users can read department tasks": FOR SELECT USING (auth.role() = 'authenticated')
- "Service role full access": FOR ALL USING (auth.role() = 'service_role')

Add an auto-update trigger for updated_at using `_governance_set_updated_at()` (already exists from Phase 36 governance migration).

Also create a SQL view `department_health_summary` that computes per-department health:
```sql
CREATE OR REPLACE VIEW department_health_summary AS
SELECT
    d.id AS department_id,
    d.name AS department_name,
    d.type AS department_type,
    d.status AS department_status,
    COUNT(dt.id) FILTER (WHERE dt.status IN ('pending', 'in_progress')) AS active_tasks,
    COUNT(dt.id) FILTER (WHERE dt.status = 'completed'
        AND dt.completed_at >= now() - INTERVAL '30 days') AS completed_30d,
    COUNT(dt.id) FILTER (WHERE dt.status IN ('pending', 'in_progress', 'completed')
        AND dt.created_at >= now() - INTERVAL '30 days') AS total_30d,
    CASE
        WHEN COUNT(dt.id) FILTER (WHERE dt.created_at >= now() - INTERVAL '30 days') = 0 THEN 'green'
        WHEN (COUNT(dt.id) FILTER (WHERE dt.status = 'completed'
              AND dt.completed_at >= now() - INTERVAL '30 days')::FLOAT
            / NULLIF(COUNT(dt.id) FILTER (WHERE dt.status IN ('pending', 'in_progress', 'completed')
              AND dt.created_at >= now() - INTERVAL '30 days'), 0)) > 0.8 THEN 'green'
        WHEN (COUNT(dt.id) FILTER (WHERE dt.status = 'completed'
              AND dt.completed_at >= now() - INTERVAL '30 days')::FLOAT
            / NULLIF(COUNT(dt.id) FILTER (WHERE dt.status IN ('pending', 'in_progress', 'completed')
              AND dt.created_at >= now() - INTERVAL '30 days'), 0)) > 0.5 THEN 'yellow'
        ELSE 'red'
    END AS health_status
FROM departments d
LEFT JOIN department_tasks dt ON dt.to_department_id = d.id
GROUP BY d.id, d.name, d.type, d.status;
```

Health logic: green (>80% completion in 30d), yellow (50-80%), red (<50%). If no tasks exist, default to green.
