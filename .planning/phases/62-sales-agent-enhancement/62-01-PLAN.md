Create `app/agents/tools/sales_followup.py` with a single async tool function:

```python
async def generate_followup_email(
    contact_name: str,
    meeting_subject: str,
    meeting_notes: str,
    next_steps: list[str] | None = None,
    meeting_date: str | None = None,
    attendees: list[str] | None = None,
) -> dict[str, Any]:
```

Implementation:
1. Extract user_id from request context via `_get_user_id()` helper (same pattern as hubspot_tools.py).
2. Attempt to enrich with CRM context: call `HubSpotService().get_deal_context(user_id, contact_name)` in a try/except -- if HubSpot is not connected or contact not found, proceed without deal data.
3. Build a structured email dict with:
   - `subject`: "Following up: {meeting_subject}" 
   - `to`: extracted from attendees or contact lookup
   - `body`: Multi-section plain text with:
     - Greeting with contact name
     - Meeting recap paragraph summarizing `meeting_notes`
     - Deal context paragraph (if available): current stage, amount, pipeline
     - Next steps as numbered list from `next_steps` param
     - CTA paragraph (schedule next meeting, review proposal, etc.)
     - Professional sign-off
   - `suggested_cta`: A one-line call-to-action recommendation
   - `deal_context`: The enriched deal data (or null)
4. Return `{"success": True, "email": {subject, to, body, suggested_cta}, "deal_context": ...}`.
5. On error, return `{"success": False, "error": str(e)}`.

Export as `SALES_FOLLOWUP_TOOLS = [generate_followup_email]`.

For tests in `tests/unit/test_sales_followup.py`:
- Mock `get_current_user_id` to return a test UUID.
- Mock `HubSpotService.get_deal_context` to return sample deal data for enrichment tests and raise ValueError for no-connection tests.
- Verify email structure (subject, body sections, CTA) in each scenario.
- Use `pytest.mark.asyncio` for all tests.
