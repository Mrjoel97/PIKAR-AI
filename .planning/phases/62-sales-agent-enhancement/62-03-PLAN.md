Create `app/agents/tools/proposal_generator.py` with:

```python
async def generate_sales_proposal(
    client_name: str | None = None,
    client_company: str | None = None,
    client_email: str | None = None,
    deal_id: str | None = None,
    executive_summary: str = "",
    line_items: list[dict] | None = None,
    total_amount: float | None = None,
    timeline: str = "",
    terms: str = "",
    validity_days: int = 30,
    discount_percent: float = 0,
) -> dict[str, Any]:
```

Implementation:
1. Get user_id from request context.
2. If `deal_id` is provided, enrich from HubSpot:
   - Query `hubspot_deals` table to get deal details (name, amount, associated contacts).
   - Use `HubSpotService().get_deal_context()` for contact info.
   - Auto-populate client_name, client_company, client_email, and total_amount from deal data.
3. Validate: must have at least `client_name` after enrichment, otherwise return error.
4. Build the template data dict matching `sales_proposal.html` variables.
5. Calculate line item totals if line_items provided: each item needs `total = quantity * unit_price`. Calculate subtotal, apply discount, compute final total.
6. If no line_items but total_amount is provided, create a single "As quoted" line item.
7. Call `DocumentService().generate_pdf(template_name="sales_proposal", data=data, user_id=user_id, title=f"Proposal - {client_company or client_name}")`.
8. Return `{"success": True, "widget": widget, "proposal_data": {client, amount, items_count}}`.

Export as `PROPOSAL_TOOLS = [generate_sales_proposal]`.

Wire into Sales Agent:
1. Import `PROPOSAL_TOOLS` in `app/agents/sales/agent.py`.


For tests: Mock DocumentService.generate_pdf to return a sample widget dict. Mock HubSpotService for deal enrichment. Test auto-population, manual entry, line item calculations, and error cases.
