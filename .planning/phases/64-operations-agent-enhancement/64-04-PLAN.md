       - Class `VendorOpsService` extending `BaseService`.

       - Method `async update_inventory_real(user_id: str, item: str = "Inventory item", quantity: int | None = None, product_id: str | None = None, **kwargs) -> dict`:
         - Backward-compatible with the degraded tool signature.
         - If product_id is provided, call InventoryService.update_stock(product_id, quantity or 0) directly.
         - If only item name is provided, search inventory by name:
           - Call InventoryService.list_inventory(user_id).
           - Find product where name matches (case-insensitive partial match).
           - If found, call update_stock with the product_id and quantity.
           - If not found, return `{"success": False, "error": f"Product '{item}' not found in inventory. Use list_inventory to see available products."}`.
         - On success, return `{"success": True, "status": "completed", "product_id": ..., "product_name": ..., "new_quantity": ..., "change": quantity, "tool": "update_inventory"}`.
         - Note: Does NOT fall back to ShopifyService -- Shopify inventory is managed via the separate Shopify sync. The existing INVENTORY_TOOLS (add_inventory_item, list_inventory, update_inventory_quantity) handle the internal product system. This tool wraps that for workflow compatibility.

         - Backward-compatible with the degraded tool signature.
         - Insert into vendor_subscriptions table:
           ```python
           row = {
               "user_id": user_id,
               "name": name,
               "category": category,
               "monthly_cost": monthly_cost,
               "billing_cycle": kwargs.get("billing_cycle", "monthly"),
               "notes": notes or f"Created via operations workflow",
               "is_active": True,
           }
           ```
         - Return `{"success": True, "status": "completed", "vendor": row_result, "tool": "create_vendor"}`.

       - Method `async create_purchase_order(user_id: str, vendor: str = "Vendor", amount: float | None = None, items: list[dict] | None = None, **kwargs) -> dict`:
         - Backward-compatible with the degraded tool signature.
         - Create a PO as a task with structured metadata (using existing create_task from sales tools for the tracking record) PLUS store the PO details:
           - Generate PO reference: `PO-{YYMMDD}-{random_4_digits}`.
           - Create a task via `create_task(description=f"Purchase Order {po_ref}: {vendor}")`.
           - Store PO metadata in the task or as a separate data event via `track_event`.
         - Return `{"success": True, "status": "completed", "po_reference": po_ref, "vendor": vendor, "total_amount": amount, "items": items or [], "task": task_result, "tool": "create_po"}`.
