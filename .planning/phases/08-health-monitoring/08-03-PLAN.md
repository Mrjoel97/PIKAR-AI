API response shape from GET /admin/monitoring/status:
```typescript
interface MonitoringStatusResponse {
  endpoints: EndpointStatus[];
  open_incidents: Incident[];
  latest_check_at: string | null; // ISO timestamp or null
}

interface EndpointStatus {
  name: string;              // "live", "connections", "cache", "embeddings", "video"
  current_status: string;    // "healthy" | "unhealthy" | "unknown"
  latest_check_at: string | null;
  response_time_ms: number | null;
  history: HistoryEntry[];   // last 20 checks, ordered by checked_at DESC
}

interface HistoryEntry {
  checked_at: string;
  response_time_ms: number;
  status: string;
}

interface Incident {
  id: string;
  endpoint: string;
  category: string | null;
  incident_type: string;     // "outage"
  started_at: string;
  resolved_at: string | null;
  details: Record<string, unknown>;
  created_at: string;
}
```

    **Install recharts:** Run `cd frontend && npm install recharts` to add recharts to the project. This is the only new frontend dependency for Phase 8.

    **Create `frontend/src/components/admin/monitoring/Sparkline.tsx`:**
    - 'use client' directive (recharts requires client rendering)
    - Import `{ LineChart, Line, ResponsiveContainer }` from 'recharts'
    - Props: `data: { response_time_ms: number }[]`, `isHealthy: boolean`
    - Render a `ResponsiveContainer` (width="100%", height={40}) containing a `LineChart`
    - Set `accessibilityLayer={false}` on LineChart (prevent spurious DOM attributes on sparklines)
    - Single `Line` with `type="monotone"`, `dataKey="response_time_ms"`, `dot={false}`, `isAnimationActive={false}`
    - Stroke color: `isHealthy ? '#4ade80' : '#f87171'` (Tailwind green-400 / red-400)
    - strokeWidth={1.5}
    - No XAxis, YAxis, Tooltip, Legend -- pure sparkline

    **Create `frontend/src/components/admin/monitoring/StaleDataBanner.tsx`:**
    - Props: `latestCheckAt: string | null`
    - If `latestCheckAt` is null, render nothing (no data yet, not stale)
    - Compute: `isStale = (Date.now() - new Date(latestCheckAt).getTime()) > 5 * 60 * 1000`
    - If stale: render amber warning banner with `bg-amber-900/50 border border-amber-600 text-amber-300 rounded-xl px-4 py-3 text-sm`
    - Message: "Warning: Health check data is stale -- last updated {relative time}. Cloud Scheduler may be paused."
    - If not stale: render nothing

    **Create `frontend/src/components/admin/monitoring/StatusCard.tsx`:**
    - Props: `endpoint: EndpointStatus` (inline the type or import)
    - Card container: `bg-gray-800 rounded-xl p-5 border border-gray-700`
    - Header row: endpoint name (capitalize first letter) + status badge
    - Status badge colors: healthy = `bg-emerald-900 text-emerald-300`, unhealthy = `bg-rose-900 text-rose-300`, unknown = `bg-gray-700 text-gray-400`
    - Response time: `{response_time_ms}ms` or "N/A" if null
    - Last checked: relative time from `latest_check_at` or "Never"
    - Sparkline component at bottom with history data (reverse the history array so oldest is first for left-to-right chronological display)

    **Create `frontend/src/components/admin/monitoring/IncidentPanel.tsx`:**
    - Props: `incidents: Incident[]`
    - If empty: render a subtle "No active incidents" message with a green checkmark
    - If incidents exist: render a list with each showing:
      - Endpoint name + incident_type badge
      - Started at (formatted)
      - Duration: compute from `started_at` to now
    - Container: `bg-gray-800 rounded-xl p-5 border border-gray-700`

    **Create `frontend/src/app/(admin)/monitoring/page.tsx`:**
    - 'use client' directive
    - Fetch `GET {API_URL}/admin/monitoring/status` with Bearer token from Supabase session
    - Use `useEffect` + `setInterval(30_000)` for 30-second auto-refresh polling
    - Use `useCallback` for the fetch function to avoid stale closures
    - Get Supabase session via `createClient` from `@/lib/supabase/client`
    - Loading state: show skeleton cards while fetching
    - Error state: show error message if fetch fails
    - Layout:
      - Page title: "System Monitoring" with `text-2xl font-bold text-gray-100`
      - `<StaleDataBanner latestCheckAt={data.latest_check_at} />` immediately below title
      - Status grid: `grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 xl:grid-cols-5 gap-4`
      - One `<StatusCard>` per endpoint
      - Below the grid: `<IncidentPanel incidents={data.open_incidents} />`

    Use the existing Tailwind dark theme (bg-gray-950, text-gray-100) consistent with the admin layout. The page renders inside the `<main>` from layout.tsx, so add padding: `p-6`.

    Run `cd frontend && npx tsc --noEmit` to verify TypeScript compiles. Run `cd frontend && npm run build` to verify the build succeeds (or at minimum `npx next lint`).
