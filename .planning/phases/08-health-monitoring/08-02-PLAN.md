    Create `app/agents/admin/tools/monitoring.py` with 7 tools following the autonomy enforcement pattern from `health.py`. All 7 tools are "auto" tier (read-only operations). Each tool:
    1. Queries `admin_agent_permissions` for autonomy level (action_name = tool function name)
    2. If blocked: return error dict
    3. If confirm: return confirmation token dict
    4. If auto (default for read operations): execute and return

    **Tools to implement:**

    1. `get_api_health_summary()` -- Returns current status of all monitored endpoints. Query latest row per endpoint from api_health_checks. Return dict with endpoint statuses, overall health, and summary string.

    2. `get_api_health_history(endpoint: str, period: str = "24h")` -- Returns trend data for a specific endpoint. period can be "24h", "7d", or "30d". Query api_health_checks for the endpoint within the time window. Return list of {checked_at, status, response_time_ms} dicts.

    3. `get_active_incidents()` -- Returns all unresolved incidents. Query api_incidents where resolved_at IS NULL (use `.is_("resolved_at", "null")`). Return list of incident dicts.

    4. `get_incident_detail(incident_id: str)` -- Returns full details for a specific incident by ID. Query api_incidents by id. Return the incident dict or error if not found.

    5. `run_diagnostic(endpoint: str)` -- Performs a fresh health check on a single endpoint. Import `_check_one` from health_monitor, create httpx client, run check, return result. Does NOT persist to DB (diagnostic only).

    6. `check_error_logs(endpoint: str = None, limit: int = 20)` -- Returns recent health check failures. Query api_health_checks where status != "healthy", optionally filtered by endpoint, ordered by checked_at DESC, limited. Return list of failure dicts.

    7. `check_rate_limits()` -- Returns a summary of current API usage patterns. Query api_health_checks for the last hour, compute average response times per endpoint, count failures. Return a summary dict with per-endpoint stats.

    All tools must have proper docstrings (interrogate 80%+). Use `from __future__ import annotations`. Use `get_service_client()` and `execute_async()` for all Supabase queries.
