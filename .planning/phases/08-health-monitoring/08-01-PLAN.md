    1. `HEALTH_ENDPOINTS` dict mapping 5 endpoint names to paths:
       - `"live": "/health/live"`, `"connections": "/health/connections"`, `"cache": "/health/cache"`, `"embeddings": "/health/embeddings"`, `"video": "/health/video"`

    2. `_check_one(client, name, path)` async function:
       - Uses `time.monotonic()` for elapsed time
       - 10-second timeout via `httpx.Timeout(10.0, connect=5.0)`
       - Returns dict: `{endpoint, status, status_code, response_time_ms, error_message}`
       - status = "healthy" if resp.status_code == 200, else "unhealthy"
       - Catches all exceptions, returns status="unhealthy" with error_message=str(exc)[:500]

    3. `_get_rolling_stats(supabase_client, endpoint)` async function:
       - Query last 10 health checks for this endpoint from api_health_checks, ordered by checked_at DESC, limit 10
       - Compute and return a dict: `{"avg_response_time_ms": float, "error_count": int, "total_count": int}`
       - If fewer than 3 records exist, return None (not enough data for anomaly detection)

    4. `_detect_anomaly(check_result, rolling_stats)` function (sync, pure logic):
       - Takes the current check result dict and rolling stats dict
       - Returns the incident_type string or None:
         - If `status_code` is None or not 200: return `"down"`
         - If `response_time_ms > 2 * rolling_stats["avg_response_time_ms"]`: return `"latency_spike"`
         - If `rolling_stats["error_count"] / rolling_stats["total_count"] > 0.05`: return `"error_spike"`
         - Otherwise: return None
       - Also updates the check_result["status"] to "degraded" for latency_spike (endpoint is up but slow)

    5. `_update_incidents(supabase_client, endpoint, incident_type, checked_at)` async function:
       - Query open incident: `.table("api_incidents").select("id, incident_type").eq("endpoint", endpoint).is_("resolved_at", "null").limit(1)`
       - IMPORTANT: Use `.is_("resolved_at", "null")` NOT `.eq("resolved_at", None)` -- PostgREST requires IS NULL syntax
       - If `incident_type is not None` AND no open incident: insert new incident with the detected `incident_type`, `started_at=checked_at`, `details={"detection_method": "anomaly_detection"}`
       - If `incident_type is None` (healthy) AND open incident exists: update incident with `resolved_at=checked_at`
       - If `incident_type is not None` AND open incident exists with different type: resolve old, create new (type escalation/change)
       - Wrap all in execute_async

    6. `_prune_old_records(supabase_client)` async function:
       - Delete records from api_health_checks where `checked_at < now() - interval '30 days'`
       - For each endpoint in HEALTH_ENDPOINTS: if count > 1000, delete oldest records beyond 1000
       - Use raw RPC or a two-step query: count per endpoint, then delete where id NOT IN (select id from api_health_checks where endpoint=X order by checked_at desc limit 1000)
       - Wrap in try/except -- prune failures should log a warning but NOT fail the health check loop

    7. `run_health_checks()` async function:
       - Reads `BACKEND_API_URL` from env (default `http://localhost:8000`)
       - Creates `httpx.AsyncClient(base_url=base_url)` context manager
       - Fires all 5 checks via `asyncio.gather(*tasks, return_exceptions=True)`
       - Filters out Exception results
       - Writes all results to api_health_checks via execute_async: `get_service_client().table("api_health_checks").insert(rows)`
       - For each result: fetch rolling stats via `_get_rolling_stats()`, detect anomaly via `_detect_anomaly()`, then call `_update_incidents()` with the detected incident_type (or None if healthy)
       - Calls `_prune_old_records()` at the end of the loop (auto-prune per CONTEXT.md)
       - Calls `log_admin_action(admin_user_id=None, action="scheduled_health_check", target_type="system", target_id=None, details={"checked": len(rows)}, source="monitoring_loop")`
       - Returns the list of result dicts
