       ```python
       # Pre-warm skill embedding index at startup (non-blocking background task)
       if not BYPASS_IMPORT:
           try:
               from app.skills.skill_embeddings import build_index as _build_skill_index
               import asyncio as _asyncio_embed

               _embed_task = _asyncio_embed.create_task(
                   _build_skill_index(),
                   name="skill-embedding-warmup",
               )
               # Fire-and-forget: don't await — let it run in background
               # The task will log its own result
               _embed_task.add_done_callback(
                   lambda t: logger.warning("Skill embedding warmup failed: %s", t.exception())
                   if t.exception() else None
               )
               logger.info("Skill embedding warmup task started")
           except Exception as e:
               logger.warning("Skill embedding warmup setup failed (non-fatal): %s", e)
       ```

    2. This uses `create_task` (fire-and-forget) so it does not delay server startup. The embedding warmup runs concurrently with request handling. This matches the project pattern used for StitchMCPService startup.
