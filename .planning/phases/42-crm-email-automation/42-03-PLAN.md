**File 1: app/agents/tools/hubspot_tools.py** -- New file following shopify_tools.py pattern exactly.

Create 5 agent tools as raw async functions (NOT FunctionTool wrapped -- sanitize_tools handles that):

1. `async def search_hubspot_contacts(query: str) -> dict[str, Any]`:
   """Search HubSpot contacts by name, email, or company."""
   - Get user_id from _get_user_id()
   - Call HubSpotService().search_contacts(user_id, query)
   - Return {"contacts": [...], "count": N} or {"error": "..."}

2. `async def get_hubspot_deal_context(contact_name_or_id: str) -> dict[str, Any]`:
   """Get HubSpot deal pipeline context for a contact. Use this BEFORE answering any sales query about a specific contact or company to provide CRM-aware responses."""
   - Call HubSpotService().get_deal_context(user_id, contact_name_or_id)
   - Return {"contact": ..., "deals": [...], "recent_activity": [...]} or {"error": "..."}
   - This is the CRM-05 tool -- makes agent responses CRM-aware

3. `async def create_hubspot_contact(email: str, name: str, company: str | None = None, phone: str | None = None) -> dict[str, Any]`:
   """Create a new contact in HubSpot CRM. Also syncs to Pikar contacts table."""
   - Call HubSpotService() to create in HubSpot + sync to Pikar
   - Return {"contact": {...}, "hubspot_id": "..."} or {"error": "..."}

4. `async def update_hubspot_deal(deal_id: str, stage: str | None = None, amount: float | None = None, close_date: str | None = None) -> dict[str, Any]`:
   """Update a HubSpot deal's stage, amount, or close date."""
   - Build properties dict from non-None params
   - Call HubSpotService().push_deal_to_hubspot(user_id, deal_id, properties)
   - Return updated deal or {"error": "..."}

5. `async def list_hubspot_deals(pipeline: str | None = None, stage: str | None = None) -> dict[str, Any]`:
   """List HubSpot deals with optional pipeline and stage filters."""
   - Query hubspot_deals table via AdminService with filters
   - Return {"deals": [...], "count": N, "total_value": sum_of_amounts}

Export as: `HUBSPOT_TOOLS = [search_hubspot_contacts, get_hubspot_deal_context, create_hubspot_contact, update_hubspot_deal, list_hubspot_deals]`

Use the _get_user_id() helper pattern from shopify_tools.py. Lazy import HubSpotService inside each function. Each function returns a structured dict. Wrap all operations in try/except returning {"error": str(exc)}.

**File 1: app/agents/tools/email_sequence_tools.py** -- New file following same tool pattern.

Create 6 agent tools:

1. `async def create_email_sequence(name: str, steps: str) -> dict[str, Any]`:
   """Create a multi-step email sequence. Steps should be a JSON string array of objects with: subject_template, body_template, delay_hours (default 24), delay_type ('after_previous' or 'at_time')."""
   - Parse steps from JSON string (agent passes structured data as string)
   - Call EmailSequenceService().create_sequence(user_id, name, parsed_steps)
   - Return created sequence or {"error": "..."}

2. `async def enroll_contacts_in_sequence(sequence_id: str, contact_ids: str, timezone: str = "UTC") -> dict[str, Any]`:
   """Enroll contacts into an email sequence. contact_ids is a comma-separated string of contact UUIDs."""
   - Parse contact_ids from comma-separated string
   - Call EmailSequenceService().enroll_contacts(user_id, sequence_id, parsed_ids, timezone)
   - Return enrollment result

3. `async def get_sequence_performance(sequence_id: str) -> dict[str, Any]`:
   """Get email sequence performance metrics: open rate, click rate, bounce rate, completion rate."""
   - Call EmailSequenceService().get_sequence_performance(user_id, sequence_id)
   - Return performance dict

4. `async def generate_sequence_content(campaign_context: str, contact_segment: str, num_steps: int = 3) -> dict[str, Any]`:
   """Generate AI-powered email sequence content based on campaign context and target audience. Returns a sequence with subject lines and body templates using Jinja2 variables like {{first_name}}, {{company}}."""
   - This tool uses the agent's own LLM capability -- it constructs a prompt describing the campaign context and contact segment, then generates email templates
   - Build structured prompt: "Create a {num_steps}-step email sequence for: Campaign: {campaign_context}, Target audience: {contact_segment}. Each step should have a subject line and HTML body template using variables: {{first_name}}, {{company}}, {{deal_name}}. Step 1 should be introductory, middle steps should provide value, final step should have a CTA."
   - Return {"steps": [{"subject_template": "...", "body_template": "...", "delay_hours": 24}], "campaign_context": campaign_context}
   - NOTE: The actual AI generation happens at the agent level -- this tool returns a structured prompt/suggestion that the agent can refine. For a simpler implementation, generate default templates programmatically (welcome, value prop, CTA pattern) and let the agent customize them via its own LLM capability.

5. `async def pause_email_sequence(sequence_id: str) -> dict[str, Any]`:
   """Pause an active email sequence. All pending sends will be held."""
   - Call EmailSequenceService().update_sequence_status(user_id, sequence_id, "paused")

6. `async def resume_email_sequence(sequence_id: str) -> dict[str, Any]`:
   """Resume a paused email sequence. Pending sends will resume from where they stopped."""
   - Call EmailSequenceService().update_sequence_status(user_id, sequence_id, "active")

Export as: `EMAIL_SEQUENCE_TOOLS = [create_email_sequence, enroll_contacts_in_sequence, get_sequence_performance, generate_sequence_content, pause_email_sequence, resume_email_sequence]`
