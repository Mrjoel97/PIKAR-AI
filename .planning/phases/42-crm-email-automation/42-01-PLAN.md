2. **Create hubspot_deals table:**
   - Columns: id UUID PK, user_id UUID FK auth.users, hubspot_deal_id TEXT NOT NULL, deal_name TEXT NOT NULL, pipeline TEXT, stage TEXT, amount NUMERIC(14,2), close_date DATE, associated_contacts UUID[] DEFAULT '{}', properties JSONB DEFAULT '{}', created_at TIMESTAMPTZ, updated_at TIMESTAMPTZ
   - UNIQUE constraint on (user_id, hubspot_deal_id)
   - RLS: enable, policies for SELECT/INSERT/UPDATE/DELETE on authenticated where user_id = auth.uid()
   - updated_at trigger (same pattern as contacts table)
   - Indexes: user_id, (user_id, pipeline, stage)

3. **Create email_sequences table:**
   - Columns: id UUID PK, user_id UUID FK auth.users, name TEXT NOT NULL, status TEXT NOT NULL DEFAULT 'draft' CHECK (status IN ('draft', 'active', 'paused', 'completed')), campaign_id UUID, created_at TIMESTAMPTZ, updated_at TIMESTAMPTZ
   - RLS policies for authenticated users
   - updated_at trigger

4. **Create email_sequence_steps table:**
   - Columns: id UUID PK, sequence_id UUID FK email_sequences ON DELETE CASCADE, step_number INT NOT NULL, subject_template TEXT NOT NULL, body_template TEXT NOT NULL, delay_hours INT NOT NULL DEFAULT 24, delay_type TEXT NOT NULL DEFAULT 'after_previous' CHECK (delay_type IN ('after_previous', 'at_time')), created_at TIMESTAMPTZ
   - UNIQUE on (sequence_id, step_number)
   - RLS via sequence ownership join

5. **Create email_sequence_enrollments table:**
   - Columns: id UUID PK, sequence_id UUID FK email_sequences ON DELETE CASCADE, contact_id UUID FK contacts ON DELETE CASCADE, current_step INT NOT NULL DEFAULT 0, status TEXT NOT NULL DEFAULT 'active' CHECK (status IN ('active', 'completed', 'bounced', 'unsubscribed', 'paused')), enrolled_at TIMESTAMPTZ DEFAULT now(), next_send_at TIMESTAMPTZ, timezone TEXT DEFAULT 'UTC'
   - UNIQUE on (sequence_id, contact_id)
   - Critical index: `CREATE INDEX idx_enrollments_next_send ON public.email_sequence_enrollments(next_send_at) WHERE status = 'active' AND next_send_at IS NOT NULL;`
   - RLS via sequence ownership join

6. **Create email_tracking_events table:**
   - Columns: id UUID PK, enrollment_id UUID FK email_sequence_enrollments ON DELETE CASCADE, step_number INT NOT NULL, event_type TEXT NOT NULL CHECK (event_type IN ('open', 'click', 'bounce', 'unsubscribe', 'delivered')), metadata JSONB DEFAULT '{}', created_at TIMESTAMPTZ DEFAULT now()
   - Index on (enrollment_id, event_type)
   - RLS via enrollment ownership chain

Use the same patterns as `20260404700000_financial_integrations.sql` for RLS policy naming, trigger functions, and index conventions. Use `gen_random_uuid()` for PKs. **File 1: app/services/hubspot_service.py** -- New file following StripeSyncService pattern.

Create `HubSpotService(BaseService)` with these methods:

1. `async def _get_client(self, user_id: str) -> HubSpot` -- Gets OAuth token via IntegrationManager.get_valid_token(user_id, "hubspot"), returns HubSpot(access_token=token). Raise ValueError if not connected.

2. `async def sync_contacts(self, user_id: str) -> dict[str, Any]` -- Full contact import from HubSpot:
   - Use `asyncio.to_thread()` to call `client.crm.contacts.basic_api.get_page(limit=100, properties=["email", "firstname", "lastname", "phone", "company", "lifecyclestage", "hs_lastmodifieddate"])`
   - Paginate through all results (follow `paging.next.after`)
   - Sleep 0.5s between pages to respect 190/10s rate limit
   - Map each HubSpot contact to Pikar contacts row using LIFECYCLE_MAP: {"subscriber": "lead", "lead": "lead", "marketingqualifiedlead": "qualified", "salesqualifiedlead": "qualified", "opportunity": "opportunity", "customer": "customer", "evangelist": "customer", "other": "lead"}
   - Upsert to contacts table using AdminService (service role) with `on_conflict="user_id,hubspot_contact_id"`
   - Store HubSpot extra properties in metadata.hubspot_properties JSONB
   - Return {"synced": count, "errors": error_count}

3. `async def sync_deals(self, user_id: str) -> dict[str, Any]` -- Full deal import:
   - Fetch deals with properties: dealname, pipeline, dealstage, amount, closedate, hs_lastmodifieddate
   - Also fetch deal-to-contact associations
   - Upsert to hubspot_deals table with `on_conflict="user_id,hubspot_deal_id"`
   - Return {"synced": count}

4. `async def push_contact_to_hubspot(self, user_id: str, contact_id: str) -> dict` -- Pikar-to-HubSpot push:
   - Read contact from Pikar DB
   - If hubspot_contact_id exists, update in HubSpot. If not, create new.
   - Set short-lived Redis flag `pikar:hubspot:skip:{hubspot_contact_id}` (TTL 30s) to prevent sync loop
   - Return HubSpot response

5. `async def push_deal_to_hubspot(self, user_id: str, deal_id: str, properties: dict) -> dict` -- Update deal in HubSpot:
   - Read deal from hubspot_deals, push properties to HubSpot via SDK
   - Set Redis skip flag
   - Return updated deal

6. `async def get_deal_context(self, user_id: str, contact_name_or_id: str) -> dict` -- For CRM-aware agent responses:
   - Search contacts by name/email, find associated deals
   - Return deal stage, amount, pipeline, recent activity summary
   - Used by the get_hubspot_deal_context agent tool

7. `async def search_contacts(self, user_id: str, query: str) -> list[dict]` -- Search HubSpot contacts by name/email/company using HubSpot search API.

8. `async def handle_contact_webhook(self, user_id: str, payload: dict) -> dict` -- Process HubSpot contact webhook:
   - Check Redis skip flag `pikar:hubspot:skip:{contact_id}` -- if set, skip (our own echo)
   - For contact.creation: create/upsert contact in Pikar
   - For contact.propertyChange: update matching contact. Compare hs_lastmodifieddate for conflict detection. If both modified since last sync, log conflict but still apply (last-write-wins per user decision).

9. `async def handle_deal_webhook(self, user_id: str, payload: dict) -> dict` -- Same pattern for deals.

Use lazy import for `from hubspot import HubSpot` inside _get_client to avoid import-time failures when SDK not installed. All HubSpot SDK calls wrapped in asyncio.to_thread(). Use AdminService for all DB writes. Add proper logging throughout.

**File 2: app/routers/webhooks.py** -- Add dedicated HubSpot webhook endpoint.

Add to the existing webhooks router:

1. `def _verify_hubspot_signature_v3(body: bytes, method: str, url: str, timestamp: str, secret: str, signature: str) -> bool`:
   - Verify timestamp within 300 seconds
   - Compute: `source = f"{method}{url}{body.decode()}{timestamp}"`
   - HMAC-SHA256 with client_secret, base64-encode
   - Compare with X-HubSpot-Signature-v3 header

2. `async def _resolve_hubspot_user(portal_id: str) -> str | None`:
   - Query integration_credentials for provider="hubspot" where account_name matches portal_id (or metadata contains portal_id)
   - Return user_id

3. `@router.post("/webhooks/hubspot") async def hubspot_webhook(request: Request)`:
   - Read raw body, extract X-HubSpot-Signature-v3 and X-HubSpot-Request-Timestamp headers
   - Verify signature using HUBSPOT_CLIENT_SECRET from settings
   - HubSpot sends batched events as a JSON array -- iterate each event
   - Resolve user_id from portalId in payload
   - Route to HubSpotService.handle_contact_webhook or handle_deal_webhook based on subscriptionType
   - Return 200 (HubSpot expects fast response; process async if needed)

Follow the exact same structure as the existing stripe_webhook and shopify_webhook handlers in the same file. Install hubspot-api-client: `uv add hubspot-api-client`. Also explicitly add jinja2: `uv add jinja2`.
