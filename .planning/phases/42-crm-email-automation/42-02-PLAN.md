Create `app/services/email_sequence_service.py` -- the complete email sequence engine.

**Class: EmailSequenceService(BaseService)**

1. **CRUD operations:**
   - `async def create_sequence(self, user_id: str, name: str, steps: list[dict]) -> dict` -- Create sequence + steps in a single transaction. Each step dict has: subject_template, body_template, delay_hours, delay_type. Returns created sequence with steps.
   - `async def get_sequence(self, user_id: str, sequence_id: str) -> dict | None` -- Fetch sequence + steps + enrollment stats (active count, completed count).
   - `async def list_sequences(self, user_id: str) -> list[dict]` -- List all sequences for user with status and enrollment counts.
   - `async def update_sequence_status(self, user_id: str, sequence_id: str, status: str) -> dict` -- Activate, pause, or complete a sequence. Validate transitions: draft->active, active->paused, paused->active, active->completed, paused->completed.

2. **Enrollment:**
   - `async def enroll_contacts(self, user_id: str, sequence_id: str, contact_ids: list[str], timezone: str = "UTC") -> dict` -- Enroll contacts in sequence. Compute next_send_at for step 0 based on delay_hours and timezone. Skip contacts already enrolled or unsubscribed. Return {"enrolled": count, "skipped": count}.
   - `async def unenroll_contact(self, user_id: str, enrollment_id: str) -> dict` -- Set enrollment status to 'paused'.

3. **Template rendering:**
   - `def _render_template(self, template: str, context: dict) -> str` -- Use Jinja2 `Environment(undefined=Undefined).from_string(template).render(context)`. Context includes: first_name, last_name, company, email, deal_name, and any custom fields from contact metadata. Use jinja2.Undefined (not StrictUndefined) so missing variables render as empty string rather than erroring.

4. **Delivery tick (module-level function):**
   - `async def run_email_delivery_tick() -> list[dict]` -- Called by WorkflowWorker every 60 seconds:
     a. Query enrollments WHERE status='active' AND next_send_at <= now() ORDER BY next_send_at LIMIT 50
     b. For each enrollment:
        - Check daily send limit via `_check_and_increment_daily_send(redis_client, user_id, max_sends)`
        - If limit hit, skip (don't advance)
        - Check bounce rate via `_check_bounce_rate(user_id)` -- if >5% over rolling 24h with >=20 sends minimum, auto-pause ALL active sequences for that user and return early
        - Load step template, render with contact data
        - Send via Resend with tracking pixel injected + links wrapped + List-Unsubscribe headers (RFC 8058)
        - Record 'delivered' tracking event
        - Advance enrollment to next step: increment current_step, compute next next_send_at. If no more steps, set status='completed'.
     c. Return list of send results

5. **Daily send limit:**
   - `async def _check_and_increment_daily_send(redis_client, user_id: str, max_sends: int) -> bool` -- Atomic Redis INCR on key `pikar:email:daily:{user_id}:{date}`. Set 90000s TTL on first use. Return True if under limit, else decrement and return False.
   - `async def _get_daily_send_limit(user_id: str) -> int` -- Compute warm-up limit: Week 1: 50, Week 2: 100, Week 3: 250, Week 4+: 500. Based on days since first email send (stored in integration_sync_state metadata for provider "email_sequences").

6. **Bounce protection:**
   - `async def _check_bounce_rate(user_id: str) -> float` -- Count bounces and total sends in last 24h from email_tracking_events. Return rate. If rate >0.05 and total >=20, also auto-pause all active sequences.
   - `async def handle_bounce_event(enrollment_id: str, step_number: int) -> None` -- Record bounce tracking event. Update enrollment status to 'bounced'.
   - `async def handle_unsubscribe(enrollment_id: str) -> None` -- Set enrollment status to 'unsubscribed'. Mark contact as unsubscribed in metadata to prevent future enrollment.

7. **Tracking helpers:**
   - `def _inject_tracking_pixel(html: str, tracking_id: str, base_url: str) -> str` -- Insert 1x1 transparent PNG img tag before closing </body>.
   - `def _wrap_links(html: str, tracking_id: str, base_url: str) -> str` -- Replace href URLs (except unsubscribe) with tracking redirect URLs.
   - `def _add_unsubscribe_footer(html: str, enrollment_id: str, base_url: str) -> str` -- Add footer with unsubscribe link.

8. **Performance stats:**
   - `async def get_sequence_performance(self, user_id: str, sequence_id: str) -> dict` -- Aggregate tracking events: open_rate, click_rate, bounce_rate, completion_rate.

Use `PIKAR_BASE_URL` from settings (or default to `https://app.pikar.ai`) for tracking URL construction. Use AdminService for all DB writes. Import Resend sending from existing EmailService. Add logging throughout.

**File 1: app/routers/email_sequences.py** -- New router for email sequence REST API + tracking.

Create FastAPI router with prefix "/email-sequences":

1. **CRUD endpoints:**
   - `POST /email-sequences/` -- Create sequence with steps. Body: {name, steps: [{subject_template, body_template, delay_hours, delay_type?}]}. Returns 201 with created sequence.
   - `GET /email-sequences/` -- List user's sequences with enrollment stats.
   - `GET /email-sequences/{sequence_id}` -- Get sequence detail with steps and stats.
   - `PATCH /email-sequences/{sequence_id}/status` -- Update status. Body: {status: "active"|"paused"|"completed"}.
   - `DELETE /email-sequences/{sequence_id}` -- Delete sequence (CASCADE deletes steps/enrollments).

2. **Enrollment endpoints:**
   - `POST /email-sequences/{sequence_id}/enroll` -- Enroll contacts. Body: {contact_ids: [], timezone?: "America/New_York"}.
   - `DELETE /email-sequences/enrollments/{enrollment_id}` -- Unenroll contact.

3. **Performance endpoint:**
   - `GET /email-sequences/{sequence_id}/performance` -- Returns open/click/bounce/completion rates.

4. **Tracking endpoints (public, no auth required):**
   - `GET /tracking/open/{tracking_id}` -- Serve 1x1 transparent PNG. Record 'open' event in email_tracking_events. tracking_id format: `{enrollment_id}_{step_number}`. Return PNG with content-type image/png, cache-control no-cache. Use a hardcoded 43-byte transparent PNG.
   - `GET /tracking/click/{tracking_id}` -- URL parameter: `url` (the original destination). Record 'click' event. Return 302 redirect to original URL. Validate URL starts with http/https to prevent open redirect.
   - `POST /unsubscribe/{enrollment_id}` -- Also GET for one-click (RFC 8058). Record 'unsubscribe' event. Set enrollment status to 'unsubscribed'. Return simple HTML page confirming unsubscribe.

Register auth dependency (get_current_user from request context) on CRUD/enrollment endpoints. Tracking endpoints are public (no auth). Add the router to fast_api_app.py.

async def run_email_sequence_tick_if_due(self):
    now = datetime.now()
    if (now - self.last_email_sequence_tick).total_seconds() < self.email_sequence_interval_seconds:
        return
    self.last_email_sequence_tick = now
    try:
        from app.services.email_sequence_service import run_email_delivery_tick
        results = await run_email_delivery_tick()

            logger.info("Processed %s email sequence sends", len(results))

        logger.error("Email sequence delivery tick failed: %s", exc, exc_info=True)
