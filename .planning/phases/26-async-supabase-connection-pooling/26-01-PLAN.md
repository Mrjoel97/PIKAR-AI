    **supabase_async.py:**
    Rewrite `execute_async` to directly `await query_builder.execute()` instead of `asyncio.to_thread(query_builder.execute)`. The supabase AsyncClient's query builders return coroutines from `.execute()`, so this is now a direct async call. Keep the same signature and timeout support via `asyncio.wait_for`. Keep the same logging.

    IMPORTANT: Some legacy callers may still pass sync query builders (from the sync Client). To handle the transition period, detect whether `query_builder.execute()` returns a coroutine. If it does, await it directly. If it doesn't (sync), fall back to `asyncio.to_thread()`. Use `inspect.isawaitable()` on the result:
    ```python
    result = query_builder.execute()
    if inspect.isawaitable(result):
        return await result  # native async
    return result  # already resolved (sync)
    ```
    This ensures zero breakage during the migration period.

    **supabase_resilience.py:**
    Replace `threading.Lock` with `asyncio.Lock` for the `_state_lock` instance variable. The class-level singleton `_lock` should use `asyncio.Lock()` as well. Since `asyncio.Lock` requires `async with`, update `should_allow_request`, `record_success`, `record_failure`, `reset`, and `get_status` to be async methods. Update `with_supabase_resilience` decorator's wrapper to use `await` for these methods.

    IMPORTANT: The singleton `__new__` pattern with asyncio.Lock is problematic (`__new__` is sync). Instead, keep `__new__` using `threading.Lock` for the one-time singleton creation (this is fine — it only runs once at import time). Only convert `_state_lock` to `asyncio.Lock` since that's the one hit on every request.
