<objective>
Equalize agent access across all personas so that every persona (solopreneur, startup, SME, enterprise) can route to every specialized agent. The preferred_agents field in PersonaPolicy currently restricts which agents the LLM is told to prefer per persona — solopreneur sees only 4 agents, startup sees 6, etc. After this plan, all personas list all agents as preferred, and the prompt wording changes from a restrictive "Preferred agents: X, Y" to an inclusive "All agents are available — prioritize based on routing priorities."

Purpose: Users on lower-tier personas currently miss out on agents like ComplianceRiskAgent or HRRecruitmentAgent because the LLM is prompted to prefer a subset. The only differentiator between personas should be rate limits (already enforced correctly) and behavioral tuning (prompt fragments, which are correct and stay).

Output: Updated policy_registry.py, prompt_fragments.py, updated existing test, new equalization test file.
</objective>
