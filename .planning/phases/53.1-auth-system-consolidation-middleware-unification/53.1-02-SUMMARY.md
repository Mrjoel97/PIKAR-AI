# Phase 53.1 Plan 02: Proxy Consolidation & Invite Boundary Summary

Completed the frontend and boundary-cleanup slice for Phase 53.1.

## Accomplishments

- Added `app/routers/teams_public.py` with `GET /teams/invites/details`, backed by `WorkspaceService.get_invite_details()`
- Kept the invite metadata contract compatible with the existing invite page by returning:
  - `workspaceName`
  - `invitedEmail`
  - `inviterName`
  - `expiresAt`
  - `isActive`
- Rebuilt `frontend/src/proxy.ts` so the active protected-route proxy now:
  - delegates auth validation to `updateSession()` from `frontend/src/lib/supabase/proxy.ts`
  - preserves sanitized `returnUrl` values on generic redirects to `/auth/login`
  - copies refreshed cookies onto redirect responses
- Updated `/invite/[token]` to fetch public invite metadata from the backend API instead of the deleted frontend service-role route
- Removed:
  - `frontend/src/app/api/teams/invite-details/route.ts`
  - `frontend/src/components/auth/ProtectedRoute.tsx`
- Added regression tests for:
  - public invite fetch path
  - protected-route returnUrl helper
  - backend public invite metadata endpoint
