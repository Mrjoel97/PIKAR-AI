    CREATE POLICY "Users can view own decisions"
        ON decision_journal FOR SELECT
        USING (auth.uid() = user_id);

    CREATE POLICY "Service role can manage decisions"
        ON decision_journal FOR ALL
        USING (true)

  <action>
    Create `app/services/onboarding_nudge_service.py`:
    - Class `OnboardingNudgeService` with singleton `get_onboarding_nudge_service()`.
    - `async def check_nudges(self, user_id: str) -> list[dict]` that:
      1. Gets `OnboardingStatus` from `get_user_onboarding_service().get_onboarding_status(user_id)`.
      2. If `is_completed` is True, checks the `onboarding_checklist` table for incomplete items.
      3. Gets user profile `created_at` to determine if within 7-day window.
      4. If user is older than 7 days, returns empty list.
      5. Checks last interaction time from `interaction_logs` table (last row for this user_id).
      6. If last interaction was < 24 hours ago, returns empty list (user is active).
      7. Otherwise, determines what's stalled and returns nudge(s):
         - If onboarding not completed: nudge for the specific incomplete step.
         - If onboarding completed but checklist items incomplete: nudge for the next uncompleted checklist item.
      8. Each nudge is a dict: `{"nudge_type": "onboarding_step" | "checklist_item", "step_name": str, "message": str, "suggested_action": str}`.
    - Nudge messages are contextual, e.g.:
      - Step 2 stalled: "You've set up your business context -- nice! Setting your preferences takes about 30 seconds and helps me match your communication style."
      - Checklist "brain_dump" stalled: "Many solopreneurs find the Brain Dump incredibly useful for getting clarity. Want to try recording your thoughts?"
    - Private method `_generate_step_nudge(status: OnboardingStatus) -> dict | None`.
    - Private method `_generate_checklist_nudge(checklist_items: list[dict], persona: str) -> dict | None`.

    Create `app/agents/tools/onboarding_nudges.py`:
    - `async def check_onboarding_nudges() -> dict` -- ADK tool. Docstring: "Check if the current user has any onboarding nudges. Call this at the START of every conversation for users in their first 7 days. If nudges exist, weave them naturally into your response."
    - Gets user_id from `get_current_user_id()`.
    - Calls `get_onboarding_nudge_service().check_nudges(user_id)`.
    - Returns `{"has_nudges": bool, "nudges": [...], "instruction": "If nudges exist, naturally mention them in your response. Don't be pushy -- frame as helpful suggestions. Example: 'By the way, I noticed you haven't tried [X] yet -- it's a quick win that [benefit].'"}`.
    - Export `ONBOARDING_NUDGE_TOOLS = [check_onboarding_nudges]`.

    - Mock UserOnboardingService, Supabase client, execute_async.
