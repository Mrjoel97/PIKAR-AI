Create a cross-agent business synthesis tool that allows the ExecutiveAgent to answer holistic questions like "How's my business doing?" by fanning out to Financial, Sales, Marketing, and Data domain services, gathering real data, and merging it into a single coherent response.

Purpose: CROSS-01 requires users to get synthesized multi-agent responses from a single question. Currently the ExecutiveAgent can delegate to individual sub-agents but cannot gather data from multiple agents in parallel and merge results.

Output: A new agent tool `synthesize_business_health` wired into ExecutiveAgent, backed by a service that queries existing domain services.

    - Test 3: Service returns empty dict with error message when all sources fail
    - Test 4: Tool function synthesize_business_health returns a dict with "success", "health_snapshot", and "sections" keys
    - Test 5: Service respects user_id scoping for data queries
  </behavior>
  <action>
    Create `app/services/cross_agent_synthesis_service.py`:
    - Class `CrossAgentSynthesisService` with singleton `get_cross_agent_synthesis_service()`.
    - Method `async gather_business_health(user_id: str) -> dict` that uses `asyncio.gather` with `return_exceptions=True` to fan out to:
      1. Financial: Query `workflow_executions` for recent financial-related workflows + query `interaction_logs` for FinancialAnalysisAgent recent summaries (last 7 days).
      2. Sales: Query `interaction_logs` for SalesIntelligenceAgent recent summaries + query `initiatives` table for sales-tagged initiatives.
      3. Marketing: Query `interaction_logs` for MarketingAutomationAgent + query `analytics_events` for campaign-related events.
      4. Data: Query `analytics_events` for recent trends/anomalies.
    - Each sub-query is a private method (`_gather_financial`, `_gather_sales`, etc.) that returns a dict with `status`, `highlights` (list of strings), and `metrics` (dict).
    - If a sub-query raises, catch it and return `{"status": "unavailable", "error": str(e)}` for that section.
    - Final return shape: `{"financial": {...}, "sales": {...}, "marketing": {...}, "data": {...}, "gathered_at": ISO timestamp}`.
    - Use `get_service_client()` for Supabase access, `execute_async()` for queries.

    Create `app/agents/tools/cross_agent_synthesis.py`:
    - `async def synthesize_business_health(time_range_days: int = 7) -> dict` -- ADK tool.
    - Docstring explains: "Get a holistic view of business health across all domains. Call this when the user asks broad questions like 'How is my business doing?', 'Give me an overview', 'What's the state of things?'"
    - Gets user_id from `get_current_user_id()`.
    - Calls `get_cross_agent_synthesis_service().gather_business_health(user_id)`.
    - Returns `{"success": True, "health_snapshot": result, "sections": list(result.keys()), "instruction": "Synthesize these findings into a coherent, conversational business health summary. Lead with the most important insight. Use plain English, not raw data dumps."}`.
    - Export `CROSS_AGENT_SYNTHESIS_TOOLS = [synthesize_business_health]`.

    Create `tests/unit/test_cross_agent_synthesis.py`:
    - Mock Supabase client and `execute_async`.
    - Test all 5 behaviors listed above.
    - Follow existing pattern: class-based tests with fixtures, `@pytest.mark.asyncio`.
