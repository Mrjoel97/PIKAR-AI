    1. `async generate_pdf(template_name, data, user_id, session_id=None, title=None) -> dict` — Main PDF generation method:
       - Retrieve brand profile via get_brand_profile(user_id=user_id). If None, use default branding (Pikar blue #4F46E5, no logo).

       - If data contains chart_data entries, call render_chart() to produce base64 PNG strings and inject into context
       - Render template with Jinja2
       - Generate PDF via asyncio.to_thread(HTML(string=rendered_html).write_pdf) — CRITICAL: must use thread pool since weasyprint is CPU-intensive sync code
       - Enforce 50-page max (weasyprint returns page count metadata; alternatively check byte size heuristic)
       - Upload to Supabase Storage at generated-documents/{user_id}/{doc_id}.pdf
       - Track in media_assets table with session_id for conversation linking
       - Return widget dict matching the document widget pattern

       - Retrieve brand profile for branding
       - Create Presentation() with 16:9 aspect ratio (Inches(13.333) x Inches(7.5))
       - For each slide in slides_data: add blank slide, add title textbox with brand primary color, add content bullets, add chart image if chart_image_bytes provided
       - Save to BytesIO, upload to generated-documents/{user_id}/{doc_id}.pptx
       - Track in media_assets, return widget dict with fileType="pptx"

    3. `render_chart(chart_data) -> bytes` — Static chart rendering:
       - Use matplotlib to render chart based on chart_data dict: {type: "bar"|"line"|"pie", labels: [...], values: [...], title: "..."}
       - Render to PNG bytes via BytesIO (fig.savefig(buf, format='png', dpi=150, bbox_inches='tight'))
       - Close figure to prevent memory leak (plt.close(fig))
       - Return PNG bytes
