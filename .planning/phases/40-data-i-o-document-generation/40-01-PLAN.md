    **Router (app/routers/data_io.py):**
    Create APIRouter with prefix="/data-io", all endpoints require get_current_user_id dependency.

    Endpoints:
    1. `GET /data-io/tables` — Returns list of importable/exportable table names with friendly labels and column schemas. Used by frontend to populate table selector.

    2. `POST /data-io/upload` — Accepts UploadFile (max 50MB check), parses CSV via DataImportService.parse_csv(), returns: column_headers (list[str]), row_count (int), preview (first 10 rows as list[dict]), suggested_mappings (from AI or saved mappings). If saved mappings exist for this user+table, return those as suggested_mappings. Otherwise call AI suggest_mappings.

    3. `POST /data-io/validate` — Accepts body: {csv_data_key (from upload step, stored in temp/Redis), column_mapping (dict), target_table (str), on_duplicate ("skip"|"update")}. Runs DataImportService.validate(). Returns: {valid: bool, errors: list[{row, column, value, reason}], valid_count: int, error_count: int}.

    4. `POST /data-io/commit` — Accepts same body as validate. Runs DataImportService.commit(). For >1000 rows, streams SSE progress via StreamingResponse. For smaller imports, returns JSON: {imported_count, skipped_count, errors}. Saves successful column mapping to csv_column_mappings table.

    5. `GET /data-io/export/{table_name}` — Optional query params for filters. Calls DataExportService.export_table(). Returns JSON: {url (signed URL), filename, size_bytes}.

    **Note on CSV data persistence between upload/validate/commit:** Store uploaded CSV bytes temporarily in Redis with a unique key (expires 30 min). The upload endpoint returns this key. Validate and commit endpoints receive this key to retrieve the CSV without re-upload.

    **FastAPI wiring (app/fast_api_app.py):**
    Add import: `from app.routers.data_io import router as data_io_router`
    Add: `app.include_router(data_io_router, tags=["Data I/O"])`
    Place near existing router includes.
