**A. Create `app/routers/teams.py`:**

A FastAPI router with prefix `/teams`, tags `["Teams"]`, and router-level dependency `Depends(require_feature("teams"))` to ensure only startup+ tiers can access team endpoints.

Define Pydantic request/response models at the top:

```python
class WorkspaceResponse(BaseModel):
    id: str
    name: str
    slug: str | None
    owner_id: str
    role: str  # current user's role in this workspace
    member_count: int

class MemberResponse(BaseModel):
    id: str
    user_id: str
    email: str
    display_name: str | None
    role: str
    joined_at: str

class CreateInviteRequest(BaseModel):
    role: str = Field(default="viewer", pattern="^(editor|viewer)$")
    expires_hours: int = Field(default=168, ge=1, le=720)  # max 30 days

class InviteResponse(BaseModel):
    id: str
    token: str
    role: str
    expires_at: str
    share_url: str  # full URL for sharing

class AcceptInviteRequest(BaseModel):
    token: str

class UpdateRoleRequest(BaseModel):
    role: str = Field(pattern="^(admin|editor|viewer)$")
```

**Endpoints:**

1. `GET /teams/workspace` — Get current user's workspace (calls get_or_create_workspace). Returns WorkspaceResponse. No role gate (any authenticated user on startup+ tier).

2. `GET /teams/members` — List workspace members. Requires any workspace member. Calls get_workspace_members. Returns list[MemberResponse].

3. `POST /teams/invites` — Create invite link. Body: CreateInviteRequest. Requires admin role via `Depends(require_role("admin"))`. Calls create_invite_link. Returns InviteResponse with share_url constructed as `{request.base_url}dashboard/team/join?token={token}`.

4. `POST /teams/invites/accept` — Accept an invite. Body: AcceptInviteRequest. No role gate (user may not be a member yet). Calls accept_invite. Returns membership record. Catch ValueError and return 400.

5. `PATCH /teams/members/{member_user_id}/role` — Update member role. Body: UpdateRoleRequest. Requires admin via `Depends(require_role("admin"))`. Calls update_member_role. Returns updated MemberResponse.

6. `DELETE /teams/members/{member_user_id}` — Remove member. Requires admin via `Depends(require_role("admin"))`. Calls remove_member. Returns `{"success": true}`.

All endpoints use `user_id: str = Depends(get_current_user_id)` for authentication. Add rate limiting with `@limiter.limit(get_user_persona_limit)`. Follow the docstring convention of existing routers (Google-style with short description).

Error handling: Wrap service calls in try/except, return appropriate HTTP status codes (400 for validation errors, 403 for permission errors, 404 for not found, 500 for unexpected errors).

**A. Create `frontend/src/contexts/WorkspaceContext.tsx`:**

A React context following the exact same pattern as SubscriptionContext.tsx — `'use client'` directive, createContext, Provider component, and hook.

**Types:**

```typescript
export type WorkspaceRole = 'admin' | 'editor' | 'viewer' | null;

export interface WorkspaceMember {
  id: string;
  user_id: string;
  email: string;
  display_name: string | null;
  role: string;
  joined_at: string;
}

export interface WorkspaceState {
  /** Whether the initial load has completed. */
  ready: boolean;
  /** Current workspace ID, null if no workspace (solo user or free tier). */
  workspaceId: string | null;
  /** Workspace display name. */
  workspaceName: string | null;
  /** Current user's role in the workspace. */
  role: WorkspaceRole;
  /** Number of members in the workspace. */
  memberCount: number;
  /** Whether current user is the workspace owner. */
  isOwner: boolean;

  /** Permission check helpers */
  canEdit: boolean;      // admin or editor
  canManageTeam: boolean; // admin only
  canView: boolean;      // any role (admin, editor, viewer)
  isTeamWorkspace: boolean; // workspace has >1 member

  /** Refresh workspace state from API. */
  refresh: () => Promise<void>;
}
```

**Provider logic:**

1. On mount, check if user is authenticated (use `createClient()` from `@/lib/supabase/client` to get user)
2. If authenticated, fetch `GET /api/teams/workspace` via Next.js API route proxy (or directly to backend — follow the pattern used by other contexts in this codebase). Use the backend URL pattern: fetch from the FastAPI backend at the configured API base URL with the auth token in Authorization header.
3. Actually — looking at the codebase pattern, other contexts query Supabase directly. For WorkspaceContext, since the workspace data lives in Supabase tables, query Supabase directly:
   - Query `workspace_members` joined with `workspaces` where `user_id = currentUser.id`
   - If no row exists, the user has no workspace yet (workspace is created lazily when they visit the team page)
   - Extract workspace_id, role, workspace name, owner_id
   - Count members via a separate count query on workspace_members

4. Derive permission booleans:
   - `canEdit = role === 'admin' || role === 'editor'`
   - `canManageTeam = role === 'admin'`
   - `canView = role !== null`
   - `isOwner = workspace.owner_id === currentUser.id`
   - `isTeamWorkspace = memberCount > 1`

5. Set `ready: true` after initial fetch completes (even if no workspace found — that's a valid state)

6. Provide a `refresh()` function that re-fetches workspace state

**Important:** If the Supabase query fails (e.g., table doesn't exist yet during migration rollout, or free-tier user), gracefully degrade — set role=null, workspaceId=null, all permissions to true (solo users should not be blocked). The permission system only restricts when a user IS a member of a workspace with a non-admin role.
