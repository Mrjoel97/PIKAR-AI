    **Create app/agents/admin/tools/users_intelligence.py:**

    Import `check_autonomy` from `app.agents.admin.tools._autonomy`. Import `get_service_client` and `execute_async` from the established service modules. Import `asyncio`, `logging`, `datetime`.

    **Tool 1: `get_at_risk_users(threshold_days_inactive: int = 7) -> dict`**
    Autonomy tier: auto (read-only analytics query). Action name for check_autonomy: `"get_at_risk_users"`.

    Logic:
    1. Query `sessions` table for all distinct user_ids with updated_at in the last 28 days.
    2. For each user, count session_events in the last 14 days vs prior 14 days. Flag users where current < 50% of prior (declining activity).
    3. For each flagged user, fetch last_sign_in_at via `asyncio.to_thread(client.auth.admin.get_user_by_id, uid)`. Filter to users where last_sign_in_at is older than `threshold_days_inactive` days ago.
    4. Attempt Stripe billing status: try to import and call `IntegrationProxyService().call("stripe", "list_customers", {"email": user_email}, session_id="admin")`. If it fails (integration not configured, any exception), set `billing_status: "unknown (Stripe not configured)"`. If it succeeds, extract subscription status.
    5. Return `{"at_risk_users": [...], "criteria": {"threshold_days_inactive": N, "activity_decline_threshold": "50%", "analysis_window_days": 14}}`.

    Each user in the list: `{"user_id": ..., "email": ..., "last_sign_in_at": ..., "activity_decline_pct": N, "billing_status": ..., "risk_factors": [...]}`.

    Use Python-side aggregation (same pattern as analytics_aggregator.py — acceptable at admin query scale, per decision from Phase 10).

    **Tool 2: `get_user_support_context(user_id: str) -> dict`**
    Autonomy tier: auto (read-only). Action name: `"get_user_support_context"`.

    Logic:
    1. Fetch user's last 10 session_events (ordered by created_at DESC) to build usage_summary: message count last 7 days, last activity timestamp.
    2. Query session_events for `event_type='tool_error'` or similar error indicators in the last 48 hours. Build error_patterns: list of `{"agent": ..., "error_type": ..., "count": N, "last_occurred": ...}`.
    3. Fetch user profile from user_executive_agents (persona, onboarding_completed).
    4. Build suggested_steps list based on patterns:
       - High error rate on specific agent: "Check agent config for {agent_name}, consider clearing session state"
       - Zero activity + active subscription: "Check if onboarding completed, suggest guided walkthrough"
       - Declining usage + no recent errors: "Check if user is aware of relevant features, consider outreach"
    5. Return `{"user_id": ..., "usage_summary": {...}, "error_patterns": [...], "suggested_steps": [...], "allow_listed_actions": list(IMPERSONATION_ALLOWED_PATHS)}`.

    Import IMPERSONATION_ALLOWED_PATHS from impersonation_service to include in response so the AdminAgent knows what actions are available during impersonation.
