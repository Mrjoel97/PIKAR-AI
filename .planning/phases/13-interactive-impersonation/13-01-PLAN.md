    **Add two new endpoints to app/routers/admin/users.py:**

    1. `POST /admin/impersonate/{user_id}/start` — requires `require_admin` dependency. After admin is verified, perform an inline super-admin check: read `SUPER_ADMIN_EMAILS` env var (comma-separated, case-insensitive). If admin email is in the list, proceed. Else, query `user_roles` table for `role='super_admin'` for the admin's user_id. If neither matches, raise HTTPException 403 with detail "Super admin access required for interactive impersonation." On success, call `create_impersonation_session(admin_user_id, target_user_id)` and return `{"session_id": row["id"], "expires_at": row["expires_at"], "mode": "interactive"}`. Log via `log_admin_action` with `source="impersonation"` and `impersonation_session_id=session_id`. Rate limit: 10/minute.

    2. `DELETE /admin/impersonate/sessions/{session_id}` — requires `require_admin`. Call `validate_impersonation_session(session_id)` first — if None, return 404. Call `deactivate_impersonation_session(session_id)`. Return `{"success": True}`. Log deactivation via `log_admin_action` with `source="impersonation"` and `impersonation_session_id=session_id`. Rate limit: 30/minute.

    **Super-admin gate pattern (Pitfall 4 from RESEARCH.md):** Scoped to the activation endpoint only, NOT a global middleware. Use `SUPER_ADMIN_EMAILS` env var as the fast path (like ADMIN_EMAILS in require_admin). Fallback: check user_roles for role='super_admin'. This keeps it minimal for Phase 13; Phase 15 builds the full role management.
