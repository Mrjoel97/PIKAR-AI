    **Tool 1: recommend_autonomy_tier (SKIL-12, auto tier)**
    - Parameters: action_name (str), action_description (str)
    - Logic: Analyze keywords in description for risk classification:
      - READ patterns ("get", "list", "search", "view", "query") -> "auto"
      - WRITE patterns ("update", "edit", "change", "toggle") -> "confirm"
      - FINANCIAL patterns ("refund", "charge", "payment", "billing") -> "confirm"
      - DESTRUCTIVE patterns ("delete", "remove", "drop", "purge", "reset") -> "blocked" (or "confirm" if targeted)
      - EXTERNAL patterns ("external", "api", "proxy", "integration") -> "confirm"
    - Return: {"recommended_tier": str, "reasoning": str, "risk_factors": list[str], "action_name": action_name}

    **Tool 2: generate_compliance_report (SKIL-13, auto tier)**
    - Parameters: start_date (str, ISO), end_date (str, ISO), include_details (bool, default False)
    - Query admin_audit_log WHERE created_at BETWEEN start_date AND end_date, ORDER BY created_at DESC, LIMIT 1000
    - Aggregate: total actions, group by source (Counter), group by action (Counter), group by admin_user_id (Counter)
    - Build narrative: "Between {start} and {end}, {N} admin actions were recorded. {X}% were AI-initiated, {Y}% manual. Top actions: ..."
    - Return: {"period": {start, end}, "total_actions": int, "by_source": dict, "by_action": dict, "by_admin": dict, "narrative": str, "key_actions": list (top 10 most frequent)}

    **Tool 3: suggest_role_permissions (SKIL-14, auto tier)**
    - Parameters: role_description (str), base_role (str, optional - defaults to "junior_admin")
    - Define SECTION_LIST = ['users', 'monitoring', 'analytics', 'approvals', 'config', 'knowledge', 'billing', 'integrations', 'settings', 'audit_log']
    - Keyword matching on description:
      - "read" / "view" / "monitor" / "analyst" -> read-only on all sections
      - "operations" / "manage" / "lead" -> read+write on relevant sections
      - "full" / "everything" / "all access" -> read+write+manage on most sections
      - Specific section keywords ("billing", "users", "config", etc.) -> write access on matched sections only
    - Return: {"suggested_permissions": list[{role, section, allowed_actions}], "base_role": str, "reasoning": str}

    **Tool 4: generate_daily_digest (SKIL-15, auto tier)**
    - Parameters: none (or optional admin_user_id for personalization)
    - Aggregates from 4 sources:
      1. Pending approvals: query approval_requests WHERE status='PENDING', count + list top 5
      2. At-risk users: query analytics_summary_tables for users with declining 7-day activity (reuse pattern from users_intelligence.py), count
      3. Anomalous metrics: query analytics_summary_tables for latest day, check if any value >2 stddev from 30-day mean (reuse detect_analytics_anomalies pattern)
      4. Upcoming expirations: query subscriptions WHERE current_period_end < now() + interval '7 days' AND is_active=true, count
    - Each section is try/except wrapped and degrades gracefully (returns count=0 + error note if table unavailable)
    - Return: {"generated_at": ISO timestamp, "pending_approvals": {count, items}, "at_risk_users": {count, summary}, "anomalies": {count, items}, "upcoming_expirations": {count, items}, "narrative": str}

    **Tool 5: classify_and_escalate (SKIL-16, confirm tier)**
    - Parameters: issue_description (str), issue_context (dict | None)
    - Severity classification based on keywords:
      - CRITICAL: "data loss", "security breach", "payment failure", "all users affected", "production down"
      - HIGH: "degraded", "multiple users", "billing issue", "auth failure", "timeout"
      - MEDIUM: "single user", "slow", "error", "failed", "missing"
      - LOW: "question", "request", "suggestion", "minor", "cosmetic"
    - If severity >= HIGH: escalated=True, log_admin_action with source="ai_agent", action="escalate_issue", details={severity, description, recommended_action}
    - Return: {"severity": str, "confidence": float, "escalated": bool, "routed_to": "super_admin" if escalated else None, "recommended_action": str, "issue_summary": str}

    **Tool 6: list_all_approvals (auto tier)**
    - Parameters: status (str, default "PENDING"), limit (int, default 20)
    - Query approval_requests with filters, return list
    - This is the agent-callable version of the REST endpoint

    **Tool 7: override_approval (confirm tier)**
    - Parameters: approval_id (str), decision (str), reason (str | None), confirmation_token (str | None)
    - Verify approval exists and is PENDING
    - Update status, log_admin_action with source="admin_override"
    - Return result dict

    **Tool 8: manage_admin_role (confirm tier)**
    - Parameters: target_user_id (str), role (str), action (str = "assign"), confirmation_token (str | None)
    - action="assign": upsert into user_roles
    - action="remove": delete from user_roles
    - log_admin_action with source="ai_agent"
    - Return result dict

       ## Autonomy Tier Recommendation (SKIL-12)

       When the admin adds a new tool or asks about appropriate autonomy settings, call

       When the admin asks for an audit summary or compliance report, call
       generate_compliance_report(start_date, end_date) to produce a narrative report.
       Present: total actions, breakdown by source (manual vs AI), top actions, and
       any notable patterns (e.g., high volume of confirm-tier overrides).

       ## Role Permission Suggestions (SKIL-14)

       When creating a new admin account, proactively call suggest_role_permissions with
       a description of the admin's responsibilities. Present the suggested section-action
       matrix and let the super admin adjust before applying.

       ## Daily Operational Digest (SKIL-15)

       At the start of each new admin chat session, call generate_daily_digest() alongside
       the existing health check greeting. Present a structured overview:
       1. Pending approvals count and top items
       2. At-risk users (declining usage or billing issues)
       3. Anomalous metrics (>2 stddev deviations)
       4. Upcoming subscription expirations (next 7 days)
       Keep the digest concise — link to relevant admin sections for details.

       ## Severity Classification and Escalation (SKIL-16)

       When the admin reports an issue or when automated monitoring detects a problem,
       call classify_and_escalate(issue_description, issue_context) to assess severity.
       For HIGH and CRITICAL severity, the tool automatically creates an escalation audit
       entry routed to super_admin. Present the severity, recommended action, and whether
       escalation was triggered. classify_and_escalate is CONFIRM tier — the admin must
       approve the escalation action.

       ## Approval Management

       list_all_approvals shows all pending approvals across users. override_approval is
       CONFIRM tier — present a confirmation card before overriding any user's approval.
       manage_admin_role is CONFIRM tier — present details before creating or removing
