  <action>
Create `frontend/src/app/invite/[token]/page.tsx` — a public page for invite acceptance.

This is similar to the existing `/dashboard/team/join/page.tsx` but uses a path parameter instead of query parameter and is at a more user-friendly URL.

Structure:
1. Extract `token` from `params.token` (Next.js dynamic route segment)
2. On mount, look up the invite to show workspace name and role:
   - Use Supabase client to query workspace_invites by token (RLS allows public SELECT on invites)
   - If invite found: show workspace name, invited role, and inviter info
   - If invite not found / expired / revoked: show error state
3. Check auth state via `supabase.auth.getUser()`:

   **If logged in:**
   - Show invite details card with "Accept Invitation" button
   - On click: POST `/teams/invites/accept` with { token } via fetchWithAuth
   - On success: call WorkspaceProvider refresh(), toast success, redirect to /dashboard
   - On error: show inline error message

   **If NOT logged in:**
   - Show invite details card with message "Sign up or log in to accept this invitation"
   - Two buttons: "Sign Up" and "Log In"
   - Both link to `/login?returnUrl=/invite/{token}` (the existing login page handles returnUrl)
   - Per CONTEXT.md: if the invite has `invited_email`, the signup form should pre-fill the email. Pass it as query param: `/login?returnUrl=/invite/{token}&email={invited_email}`
   - After auth completes and user returns to /invite/{token}, the logged-in flow above handles acceptance

UI styling:
- Centered card layout (same as existing join page): min-h-screen, flex items-center justify-center, bg-slate-50
- Card: w-full max-w-sm rounded-2xl border bg-white p-8 shadow-sm
- Indigo-themed icon and CTA button (matching existing join page)
- States: idle, loading, success (checkmark + redirect message), error (red alert)
- Success state auto-redirects to /dashboard after 2 seconds

Important: This page is NOT inside the (personas) layout group or PremiumShell since the user might not be authenticated yet. It's a standalone page.

Wrap content in Suspense boundary (required for any page using dynamic params in client components).
