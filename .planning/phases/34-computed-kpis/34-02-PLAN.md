**Create `frontend/src/hooks/useKpis.ts`:**

A custom hook that fetches KPI data from the backend:

```typescript
export interface KpiItem {
  label: string;
  value: string;
  unit: string;
}

export interface KpiData {
  persona: string;
  kpis: KpiItem[];
}

export function useKpis(): {
  kpis: KpiItem[];
  isLoading: boolean;
  error: string | null;
}
```

Implementation:
1. Uses `useState` for kpis (KpiItem[]), isLoading (boolean, default true), error (string | null).
2. Uses `useEffect` to fetch on mount. Call `fetchWithAuth('/kpis/persona')`. Parse JSON response as KpiData. Set kpis from response.kpis. Set isLoading false.
3. On fetch error: set error message, set isLoading false, set kpis to empty array. Log error to console.
4. Return { kpis, isLoading, error }.
5. Do NOT add a refetch interval — KPIs refresh on page load per requirements. The 60-second freshness requirement is satisfied because page navigation triggers re-mount.
6. Import `fetchWithAuth` from `@/services/api`.

```typescript
interface KpiBarProps {

  isLoading: boolean;
}

export function KpiBar({ kpiLabels, kpis, isLoading }: KpiBarProps)
```

Implementation:
1. Mark as `'use client'` at top.
2. Map over `kpiLabels`. For each label, find matching KpiItem in `kpis` array by label (case-insensitive match).
3. Render each KPI as a pill:
   - Same base styling as current: `"inline-flex items-center px-2.5 py-1 rounded-full bg-white/10 text-white/90 text-[11px] font-medium tracking-wide"`
   - Show: `{label}: {value}` when value is available
   - Show: `{label}: ...` when isLoading is true (pulsing dots via `animate-pulse` on the dots span)
   - Show: `{label}: —` when not loading and no matching KPI found (dash fallback)
4. The value portion should be slightly bolder: add `font-semibold` to the value span to visually distinguish it from the label.
5. Gap between label and value: use a colon and space separator.

Keep component simple. No additional state, no effects, purely presentational.
