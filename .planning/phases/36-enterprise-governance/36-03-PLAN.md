**Page structure — four sections in a grid:**

1. **Portfolio Health Score (top-left, prominent)**
   - Large circular or numeric display showing the 0-100 score
   - Below it, three sub-metrics: Initiative Completion, Risk Coverage, Resource Allocation (each as a percentage bar or small metric)
   - Use MetricCard or a custom card with the score prominently displayed
   - Color: green >= 70, yellow 40-69, red < 40

2. **Compliance Status Summary (top-right)**
   - Fetch existing compliance data: `getAudits()` from `@/services/compliance` and `getRisks()` from `@/services/compliance`
   - Show: total audits, completed audits, compliance score (use `computeComplianceScore`), open risks count
   - Use MetricCard components for each metric
   - Use StatusBadge for risk severity counts

3. **Pending Approval Chains (middle row, full width)**
   - Table/list of pending approval chains from `getApprovalChains()`
   - Each row: action_type (formatted), resource_label, status badge, step progress (e.g., "Step 1 of 3 — Reviewer"), created_at formatted
   - Empty state: "No pending approval chains"
   - Show step indicators: green checkmark for approved steps, yellow dot for current pending step, gray for future steps

4. **Audit Log (bottom, full width)**
   - Paginated table from `getAuditLog()`
   - Columns: Timestamp (formatted), Action (action_type with icon), Resource (resource_type + resource_id), Details (collapsed/expandable JSON)
   - Pagination: "Load More" button that increments offset by 50
   - Optional filter dropdown for action_type (initiative.created, workflow.executed, role.changed, approval.decided, member.joined, member.removed)
