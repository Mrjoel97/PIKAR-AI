Create `app/routers/governance.py` following the exact same pattern as `app/routers/teams.py` (copyright header, imports, router with prefix + tags + feature gate, Pydantic models, Google-style docstrings on endpoints).

**Router setup:**
```python
router = APIRouter(
    prefix="/governance",
    tags=["Governance"],
    dependencies=[Depends(require_feature("governance"))],
)
```

**Pydantic models:**
- `AuditLogEntry` — id, user_id, action_type, resource_type, resource_id, details, created_at
- `PortfolioHealthResponse` — score (int), components (dict with initiative_completion, risk_coverage, resource_allocation)
- `ApprovalChainResponse` — id, user_id, action_type, resource_id, resource_label, status, steps (list), created_at, resolved_at
- `CreateChainRequest` — action_type (str), resource_id (str | None), resource_label (str | None), steps (list[dict] | None)
- `DecideStepRequest` — decision (str, pattern "^(approved|rejected)$"), comment (str | None)

**Endpoints:**

1. `GET /governance/audit-log` — Paginated audit log
   - Query params: limit (default 50, max 200), offset (default 0), action_type (optional filter)
   - Calls `governance_service.get_audit_log(user_id, limit, offset, action_type)`
   - Returns list of AuditLogEntry
   - Rate limited with `get_user_persona_limit`

2. `GET /governance/portfolio-health` — Portfolio health score
   - Calls `governance_service.compute_portfolio_health(user_id)`
   - Returns PortfolioHealthResponse
   - Rate limited

3. `POST /governance/approval-chains` — Create a new approval chain
   - Body: CreateChainRequest
   - Calls `governance_service.create_approval_chain(...)`
   - Also calls `governance_service.log_event(...)` to record the chain creation
   - Returns ApprovalChainResponse
   - Rate limited, require_role("admin") dependency

4. `GET /governance/approval-chains` — List pending approval chains
   - Calls `governance_service.get_pending_chains(user_id)`
   - Returns list of ApprovalChainResponse
   - Rate limited

5. `GET /governance/approval-chains/{chain_id}` — Get chain status
   - Calls `governance_service.get_chain_status(chain_id)`
   - Returns ApprovalChainResponse or 404
   - Rate limited

6. `POST /governance/approval-chains/{chain_id}/steps/{step_order}/decide` — Decide on a step
   - Body: DecideStepRequest
   - Calls `governance_service.decide_step(chain_id, step_order, user_id, decision, comment)`
   - Returns updated ApprovalChainResponse
   - Rate limited

All endpoints use `user_id: str = Depends(get_current_user_id)` for auth. All use try/except with HTTPException on errors. Follow the exact error handling pattern from teams.py.

**Register in fast_api_app.py:**
Add at the end of the router imports (after teams_router):
```python
from app.routers.governance import router as governance_router
```
And add to the include_router calls:
```python
app.include_router(governance_router, tags=["Governance"])

       user_id=user_id,
       action_type="initiative.created",
       resource_type="initiative",
       resource_id=created_initiative.get("id"),
       details={"title": created_initiative.get("title"), "source": "template"},
   )
   ```

2. **Initiative deletion** (if a DELETE endpoint exists — check first; if not, skip):
