**1. `governance_audit_log` table:**
- `id` UUID PRIMARY KEY DEFAULT gen_random_uuid()
- `user_id` UUID NOT NULL (actor who performed the action)
- `action_type` TEXT NOT NULL (e.g., 'initiative.created', 'workflow.executed', 'role.changed', 'approval.decided')

- RLS: enable RLS, policy allowing users to SELECT their own rows via `auth.uid() = user_id`

**2. `approval_chains` table:**
- `id` UUID PRIMARY KEY DEFAULT gen_random_uuid()
- `user_id` UUID NOT NULL (owner / requester)
- `action_type` TEXT NOT NULL (e.g., 'bulk_workflow_execution', 'data_export', 'budget_change')
- `resource_id` TEXT (the resource the action targets)
- `resource_label` TEXT (human-readable label for display)
- `status` TEXT NOT NULL DEFAULT 'pending' CHECK (status IN ('pending', 'approved', 'rejected', 'expired'))
- `created_at` TIMESTAMPTZ NOT NULL DEFAULT now()
- `resolved_at` TIMESTAMPTZ
- `expires_at` TIMESTAMPTZ (optional deadline)
- Indexes: on `user_id`, on `status`, composite on `(user_id, status)`
- RLS: enable RLS, policy for SELECT where `auth.uid() = user_id`

**3. `approval_chain_steps` table:**

- `status` TEXT NOT NULL DEFAULT 'pending' CHECK (status IN ('pending', 'approved', 'rejected', 'skipped'))
- `decided_at` TIMESTAMPTZ
- `comment` TEXT
- UNIQUE(chain_id, step_order)
- Index on `chain_id`
- RLS: enable RLS, policy for SELECT joining to approval_chains where auth.uid() = approval_chains.user_id

**Audit logging (GOV-01):**
- `async def log_event(user_id, action_type, resource_type, resource_id=None, details=None, ip_address=None) -> None`
  - Inserts into `governance_audit_log` using service-role client
  - MUST use try/except and logger.error — audit failures must NEVER break the calling operation (same pattern as `admin_audit.py`)
  - Use `execute_async` with `op_name="governance.log_event"`

- `async def get_audit_log(user_id, limit=50, offset=0, action_type=None) -> list[dict]`
  - SELECT from `governance_audit_log` where user_id matches, ordered by created_at DESC
  - Optional `action_type` filter
  - Returns paginated list
  - Use `execute_async` with `op_name="governance.audit_log"`

**Portfolio health score (GOV-02):**
- `async def compute_portfolio_health(user_id) -> dict`
  - Computes a weighted score 0-100 from three components:
    - **Initiative completion rate (40%)**: Query `initiatives` table — count(status='completed') / count(all active) * 100. Active = status in ('in_progress', 'blocked', 'not_started', 'completed').
    - **Risk coverage (30%)**: Query `compliance_risks` table — count(rows with non-null mitigation_plan) / count(all) * 100.
    - **Resource allocation (30%)**: Query `initiatives` table — count(initiatives with a non-null owner_user_id in their checklist items) / count(all initiatives) * 100. If no data, use 0.
  - Returns: `{"score": int, "components": {"initiative_completion": float, "risk_coverage": float, "resource_allocation": float}}`
  - Each sub-query in its own try/except so a single table failure doesn't crash the whole computation (return 0 for failed components)

**Approval chain management (GOV-04):**
- `async def create_approval_chain(user_id, action_type, resource_id=None, resource_label=None, steps=None) -> dict`
  - Creates an `approval_chains` row + default 3-step chain in `approval_chain_steps` (reviewer step_order=1, approver step_order=2, executive step_order=3)
  - If `steps` list provided (list of dicts with `role_label` and optional `approver_user_id`), use those instead of defaults
  - Returns the chain dict with its steps

- `async def get_pending_chains(user_id) -> list[dict]`
  - SELECT approval_chains where user_id matches and status='pending', join with approval_chain_steps
  - Returns chains with their steps nested

- `async def decide_step(chain_id, step_order, approver_user_id, decision, comment=None) -> dict`
  - Updates the approval_chain_steps row: set status=decision ('approved'/'rejected'), decided_at=now(), approver_user_id, comment
  - If decision='rejected': update the parent chain status to 'rejected', set resolved_at
  - If decision='approved' AND this is the last step (max step_order): update parent chain status to 'approved', set resolved_at
  - If decision='approved' AND NOT last step: chain stays 'pending' (next step is now actionable)
  - Returns the updated chain with all steps
  - Also call `self.log_event(...)` to record the approval decision in the audit log

- `async def get_chain_status(chain_id) -> dict | None`
  - Returns the chain with all its steps, or None if not found
