    1. Create `app/services/monitoring_job_service.py` with class `MonitoringJobService`:

       a. `__init__(self)`: Lazy supabase client via `get_service_client()` (service role for background tasks per project pattern — AdminService pattern)

       b. `async def create_job(self, user_id: str, topic: str, monitoring_type: str = "competitor", importance: str = "normal", keyword_triggers: list[str] | None = None, pinned_urls: list[str] | None = None, excluded_urls: list[str] | None = None) -> dict`:
          - Insert into `monitoring_jobs` table via supabase
          - Return created job dict

       c. `async def list_jobs(self, user_id: str) -> list[dict]`:
          - Select from `monitoring_jobs` where `user_id` matches, ordered by `created_at desc`

       d. `async def update_job(self, job_id: str, user_id: str, **updates) -> dict`:
          - Update allowed fields: is_active, importance, keyword_triggers, pinned_urls, excluded_urls
          - Set `updated_at = now()`

       e. `async def delete_job(self, job_id: str, user_id: str) -> dict`:
          - Delete from `monitoring_jobs` where id=job_id and user_id matches

       f. `async def get_due_jobs(self, cadence: str) -> list[dict]`:
          - Importance-to-cadence mapping:
            - "daily" -> importance = "critical"
            - "weekly" -> importance in ("critical", "normal")
            - "biweekly" -> all active jobs
          - Select from `monitoring_jobs` where `is_active = true` and importance matches cadence rules

       g. `async def run_monitoring_tick(self, cadence: str = "daily") -> list[dict]`:
          - Call `get_due_jobs(cadence)`
          - For each job:
            1. Check budget via `_check_budget("research")` (lazy import from adaptive_router) — skip if exhausted
            2. Build research query from job.topic + job.pinned_urls context
            3. Call `_execute_research_job(query=job.topic, domain="research", depth="standard" if importance != "critical" else "deep", triggered_by="monitoring")`
            4. If result.success:
               - Compute SHA256 hash of synthesis findings
               - Compare with job.previous_state_hash
               - If hashes differ: run AI significance check (simple prompt: "Compare these findings and determine if there are significant changes worth alerting the user about")
               - If AI says significant OR any keyword_triggers found in synthesis text: call `dispatch_notification(user_id=job.user_id, event_type="monitoring.alert", title=f"Intelligence Alert: {job.topic}", body=2-3 sentence summary, data={"job_id": job.id, "monitoring_type": job.monitoring_type})`
               - Call `write_to_graph(synthesis, domain="research")` and `write_to_vault(synthesis, domain="research")`
               - Update job: `last_run_at = now()`, `previous_state_hash = new_hash`, `last_brief_id = vault_doc_id`
          - Return list of `{"job_id": str, "status": "success"|"skipped"|"error", "alerted": bool}`
