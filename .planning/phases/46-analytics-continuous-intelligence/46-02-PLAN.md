  <action>
    1. Extend `app/integrations/google/calendar.py` — add two methods to `GoogleCalendarService`:

       a. `def get_freebusy(self, start: datetime, end: datetime, calendar_ids: list[str] | None = None) -> dict[str, list[dict]]`:
          - Calls `self.service.freebusy().query()` with timeMin/timeMax and items
          - Default `calendar_ids = ["primary"]` — for the current user's calendar only (external attendees cannot be queried reliably per research pitfall 3)
          - Returns dict keyed by calendar_id, value is list of busy intervals `{"start": str, "end": str}`

       b. `def find_free_slots(self, start: datetime, end: datetime, duration_minutes: int = 30, calendar_ids: list[str] | None = None) -> list[dict]`:
          - Calls `get_freebusy()`, then computes available slots between busy intervals
          - Returns list of `{"start": iso_str, "end": iso_str, "duration_minutes": int}`
          - Only returns slots during business hours (9am-6pm in the user's timezone, or UTC if unknown)
          - Cap at 10 slot suggestions

    2. Add 4 new tool functions to `app/agents/tools/calendar_tool.py`:

       a. `def find_free_slots(tool_context, date: str, duration_minutes: int = 30, days_ahead: int = 3) -> dict[str, Any]`:
          - Calls `service.find_free_slots()` for the date range
          - Returns `{"status": "success", "slots": [...], "note": "These are open slots on YOUR calendar. External attendees may have other commitments."}`
          - Wraps in try/except like existing tools

       b. `async def get_meeting_context(tool_context, hours_ahead: int = 4) -> dict[str, Any]`:
          - Get upcoming events via `service.list_upcoming_events()`
          - Filter to events within `hours_ahead`
          - For each meeting, enrich with:
            - `crm_context`: Query `contacts` table for attendee emails (lazy import supabase, use `get_service_client()`) — returns name, company, deal info if available
            - `vault_context`: Search knowledge vault for attendee names/companies (lazy import `KnowledgeVault`) — returns recent snippets
            - `action_items`: Query `tasks` table for open tasks mentioning attendee name or meeting title
          - Returns `{"status": "success", "meetings": [{title, start, end, attendees, crm_context, vault_context, action_items}, ...]}`
          - Use `asyncio.to_thread()` for sync supabase calls per project pattern

       c. `def suggest_followup_meeting(tool_context, meeting_title: str, attendee_emails: list[str], days_ahead: int = 5, duration_minutes: int = 30) -> dict[str, Any]`:
          - Find free slots in the next `days_ahead` days
          - Suggest the earliest available slot (preferring mornings for follow-ups)
          - Return `{"status": "success", "suggestion": {"proposed_time": iso_str, "duration_minutes": N, "title": f"Follow-up: {meeting_title}", "attendees": attendee_emails}, "message": "I suggest scheduling at X. Shall I create this event?"}`
          - CRITICAL: Does NOT call create_event — only suggests. Per user decision, agent suggests and user confirms.

       d. `def detect_calendar_patterns(tool_context, days_back: int = 30) -> dict[str, Any]`:
          - List events from the past `days_back` days
          - Group by title similarity (lowered, stripped of dates/numbers)
          - Identify recurring patterns: same title appearing 3+ times, regular intervals
          - Return `{"status": "success", "patterns": [{"title": str, "frequency": "weekly|biweekly|monthly", "typical_day": str, "typical_time": str, "occurrences": int}]}`
          - Claude's discretion on pattern detection depth per CONTEXT.md
