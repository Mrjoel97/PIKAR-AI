1. `admin_analytics_daily`:
   - id: uuid PRIMARY KEY DEFAULT gen_random_uuid()
   - stat_date: date NOT NULL UNIQUE
   - dau: integer NOT NULL DEFAULT 0 (distinct users with session activity on that date)
   - mau: integer NOT NULL DEFAULT 0 (distinct users in trailing 30 days ending on that date)
   - messages: integer NOT NULL DEFAULT 0 (session_events count for that date)
   - workflows: integer NOT NULL DEFAULT 0 (workflow_executions count for that date)
   - created_at: timestamptz NOT NULL DEFAULT now()
   - updated_at: timestamptz NOT NULL DEFAULT now()

   Index: idx_admin_analytics_daily_stat_date ON admin_analytics_daily(stat_date DESC)

2. `admin_agent_stats_daily`:
   - id: uuid PRIMARY KEY DEFAULT gen_random_uuid()
   - stat_date: date NOT NULL
   - agent_name: text NOT NULL
   - success_count: integer NOT NULL DEFAULT 0
   - error_count: integer NOT NULL DEFAULT 0
   - timeout_count: integer NOT NULL DEFAULT 0
   - avg_duration_ms: numeric(10, 2)
   - total_calls: integer NOT NULL DEFAULT 0
   - created_at: timestamptz NOT NULL DEFAULT now()
   - UNIQUE (stat_date, agent_name)

   Index: idx_admin_agent_stats_daily_date ON admin_agent_stats_daily(stat_date DESC, agent_name)

**Implementation file (app/services/analytics_aggregator.py):**

Create `run_daily_aggregation(stat_date: str | None = None) -> dict`:
1. Default stat_date to yesterday (UTC) if not provided
2. Get service client
3. Query DAU: COUNT DISTINCT user_id from sessions WHERE DATE(updated_at) = stat_date
   - IMPORTANT: sessions.user_id is TEXT, not UUID. Use COUNT(DISTINCT user_id) directly, no JOIN to auth.users
4. Query MAU: COUNT DISTINCT user_id from sessions WHERE updated_at >= stat_date - 29 days AND updated_at < stat_date + 1 day
5. Query messages: COUNT(*) from session_events WHERE DATE(created_at) = stat_date
6. Query workflows: COUNT(*) from workflow_executions WHERE DATE(created_at) = stat_date
7. Upsert into admin_analytics_daily with on_conflict="stat_date"
8. Query agent_telemetry: GROUP BY agent_name with COUNT FILTER for each status (success/error/timeout), AVG(duration_ms), COUNT(*)
9. Upsert each agent's row into admin_agent_stats_daily with on_conflict="stat_date,agent_name"
10. Return {"date": stat_date, "rows_written": count}

Use execute_async for all queries. Use get_service_client (service-role, bypasses RLS).
Log at INFO level: "Analytics aggregation complete for {stat_date}: {rows_written} rows"

IMPORTANT: For Supabase RPC queries that involve aggregate SQL (COUNT DISTINCT, etc.), use the client.rpc() pattern if direct .select() cannot express the aggregation, OR use raw SQL via client.rpc("exec_sql", {"query": ...}) if available. Otherwise, use the simplest Supabase client query that works -- the executor should check what's available and use the appropriate method. The key constraint is: the aggregation must produce correct numbers, and all queries go through execute_async.
