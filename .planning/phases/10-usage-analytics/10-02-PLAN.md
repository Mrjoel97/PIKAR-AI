**Router (app/routers/admin/analytics.py):**

Create two endpoints following the monitoring.py pattern verbatim:

1. `GET /analytics/summary` — gated by require_admin, rate limited 120/minute
   - Query param: days (int, default 30)
   - Reads from admin_analytics_daily ORDER BY stat_date DESC LIMIT {days}
   - Reads from admin_agent_stats_daily for the same date range, computes success_rate = success_count / total_calls * 100 per agent
   - Reads feature usage from tool_telemetry (COUNT GROUP BY tool_name, last {days} days) and analytics_events (COUNT GROUP BY category, last {days} days)
   - Reads config status: count admin_agent_permissions GROUP BY autonomy_level, MAX(created_at) from admin_config_history
   - Returns JSON:
     ```json
     {
       "usage_trends": [{"stat_date": "2026-03-20", "dau": 15, "mau": 120, "messages": 340, "workflows": 12}, ...],
       "agent_effectiveness": [{"agent_name": "financial", "success_rate": 94.5, "avg_duration_ms": 1200, "total_calls": 150}, ...],
       "feature_usage": {
         "by_tool": [{"tool_name": "create_workflow", "call_count": 45}, ...],
         "by_category": [{"category": "chat", "event_count": 230}, ...]
       },
       "config_status": {
         "permission_counts": {"auto": 12, "confirm": 6, "blocked": 2},
         "last_config_change": "2026-03-20T15:30:00Z"
       },
       "days": 30,
       "data_source": "aggregated"
     }
     ```
   - If admin_analytics_daily is empty, return empty arrays with data_source: "no_data"

2. `POST /analytics/aggregate` — gated by verify_service_auth (NOT require_admin), rate limited 5/minute
   - Calls run_daily_aggregation() from app.services.analytics_aggregator
   - Returns {"status": "ok", "date": result["date"], "rows_written": result["rows_written"]}
   - This is the Cloud Scheduler entry point (same pattern as POST /monitoring/run-check)

**Registration:** Add `from app.routers.admin import analytics` and `admin_router.include_router(analytics.router)` to app/routers/admin/__init__.py with a "Phase 10: analytics endpoints" comment.

IMPORTANT: Feature usage queries read from tool_telemetry and analytics_events directly (these are not pre-aggregated -- they are moderate-size tables with reasonable query cost for a 30-day window). Config status reads admin_agent_permissions and admin_config_history directly (tiny tables, no aggregation needed).

**Tools (app/agents/admin/tools/analytics.py):**

Create 4 tools following the monitoring.py / users.py pattern exactly:
- Module-level `_check_autonomy()` copy (self-contained, not imported -- per project decision)
- Each tool: async def, docstring, autonomy gate, service client, execute_async queries, return dict

1. `get_usage_stats(days: int = 30) -> dict[str, Any]`:
   - Autonomy: auto tier
   - Query admin_analytics_daily ORDER BY stat_date DESC LIMIT {days}
   - Return {"usage_trends": [...], "summary": {"avg_dau": X, "latest_mau": Y, "total_messages": Z, "total_workflows": W}}

2. `get_agent_effectiveness(days: int = 30) -> dict[str, Any]`:
   - Autonomy: auto tier
   - Query admin_agent_stats_daily for last {days} days, aggregate per agent_name
   - Compute success_rate = sum(success_count) / sum(total_calls) * 100 per agent
   - Return {"agents": [{"agent_name": "financial", "success_rate": 94.5, "avg_duration_ms": 1200, "total_calls": 150}, ...]}

3. `get_engagement_report(days: int = 30) -> dict[str, Any]`:
   - Autonomy: auto tier
   - Query tool_telemetry: COUNT GROUP BY tool_name for last {days} days (top 20)
   - Query analytics_events: COUNT GROUP BY category for last {days} days
   - Return {"top_tools": [...], "event_categories": [...], "days": days}

4. `generate_report(days: int = 7) -> dict[str, Any]`:
   - Autonomy: auto tier
   - Calls get_usage_stats and get_agent_effectiveness internally
   - Builds a summary dict with key metrics and a human-readable "summary_text" field
   - Return {"report": {"period_days": days, "summary_text": "...", "usage": {...}, "agent_performance": {...}}}
