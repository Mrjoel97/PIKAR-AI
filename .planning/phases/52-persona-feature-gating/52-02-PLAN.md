**2. Create an upgrade-gate event system in `api.ts`:**

Add a simple event mechanism so the modal can be triggered from any fetchWithAuth call. Do NOT use a global state manager -- use a simple custom event:

```typescript
// In api.ts, after the existing fetchApiInternal function:
export interface UpgradeGateEvent {
  feature: string;
  currentTier: string;
  requiredTier: string;
}

// Custom event name for upgrade gate triggers
export const UPGRADE_GATE_EVENT = 'pikar:upgrade-gate';

function dispatchUpgradeGate(detail: UpgradeGateEvent): void {
  if (typeof window !== 'undefined') {
    window.dispatchEvent(new CustomEvent(UPGRADE_GATE_EVENT, { detail }));
  }
}
```

Then in `fetchWithAuth`, after getting the response, add a check BEFORE throwing:
```typescript
if (response.status === 403) {
  try {
    const cloned = response.clone();
    const body = await cloned.json();
    if (body?.detail?.feature) {
      dispatchUpgradeGate({
        feature: body.detail.feature,
        currentTier: body.detail.current_tier,
        requiredTier: body.detail.required_tier,
      });
    }
  } catch {
    // Not a feature gate response, let it fall through normally
  }
}
```

Important: This must NOT consume the response body -- use `response.clone()` before reading.
