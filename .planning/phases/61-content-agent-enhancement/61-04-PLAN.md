<objective>
Build a content performance feedback loop that fetches engagement data for published content, generates a summary with actionable improvement suggestions, and surfaces this to the user through the Content Agent.

Purpose: Currently content creation is fire-and-forget. After content is published and engagement data accumulates, the user has no way to see how it performed through the Content Agent or get data-driven improvement suggestions. This plan closes that feedback loop.

Output: New ContentPerformanceService, get_content_performance tool, updated agent instructions, unit tests.
</objective>

    Create `app/services/content_performance_service.py` with class `ContentPerformanceService`:

    ```python
    class ContentPerformanceService:
        """Fetches engagement data for published content and generates improvement suggestions."""

        async def get_performance_summary(
            self, user_id: str, since_days: int = 30, platform: str | None = None
        ) -> dict:
            """Full pipeline: fetch published items -> get engagement -> generate suggestions."""

        async def get_published_content(
            self, user_id: str, since_days: int = 30, platform: str | None = None
        ) -> list[dict]:
            """Fetch published calendar items from ContentCalendarService."""

        def fetch_engagement_for_item(
            self, user_id: str, item: dict
        ) -> dict:
            """Get engagement metrics for a single published item via social_analytics."""

        def generate_suggestions(
            self, items_with_metrics: list[dict]
        ) -> list[dict]:
            """Analyze metrics and generate actionable improvement suggestions."""

        def compute_aggregate_metrics(
            self, items_with_metrics: list[dict]
        ) -> dict:
            """Compute totals and averages across all published items."""
    ```

    Implementation details:

    **get_published_content:**
    - Use `ContentCalendarService().list_calendar(status="published", user_id=user_id)` to get published items
    - If `since_days` provided, compute start_date as today - since_days and pass to list_calendar
    - If `platform` provided, filter by platform
    - Return list of calendar item dicts

    **fetch_engagement_for_item:**
    - Extract platform and any resource_id from the calendar item's metadata (metadata may contain `post_id` or `resource_id` from publishing)
    - Call `get_social_analytics(user_id=user_id, platform=item["platform"], metric_type="post", resource_id=resource_id)` from `app.agents.tools.social_analytics`
    - If no resource_id available (content was published externally), return `{"metrics_available": False, "reason": "No post ID linked"}`
    - Return the engagement data dict

    **generate_suggestions:**
    - Analyze the collected metrics across all items and produce a list of suggestion dicts, each with `{"category": str, "insight": str, "action": str, "priority": "high"|"medium"|"low"}`
    - Suggestion rules (heuristic-based, no ML):
      - If avg engagement rate < 2%: `{"category": "engagement", "insight": "Overall engagement is below average", "action": "Try asking questions or using polls to boost interaction", "priority": "high"}`
      - If likes are high but shares/retweets are low: `{"category": "shareability", "insight": "People like your content but don't share it", "action": "Add stronger CTAs encouraging shares, or create more debate-worthy angles", "priority": "medium"}`
      - If engagement varies significantly by time of day: `{"category": "timing", "insight": "Your [time] posts get Nx more engagement than [other time]", "action": "Focus posting during [best_time] windows", "priority": "medium"}`
      - If video content outperforms static: `{"category": "format", "insight": "Your video content gets Nx more engagement than images", "action": "Prioritize video/Reels over static posts", "priority": "medium"}`
      - If one platform significantly outperforms others: `{"category": "platform", "insight": "[Platform] is your strongest channel", "action": "Double down on [platform] content and consider reducing effort on [weak_platform]", "priority": "low"}`
    - Return at most 5 suggestions, sorted by priority

    **compute_aggregate_metrics:**
    - Sum total likes, shares, comments, impressions across all items
    - Compute averages per post
    - Compute engagement rate = (likes + comments + shares) / impressions * 100 (if impressions available)
    - Return: `{"total_posts": N, "total_likes": N, "total_shares": N, "total_comments": N, "total_impressions": N, "avg_engagement_rate": float, "best_performing": {...top post...}, "worst_performing": {...bottom post...}}`

    **get_performance_summary:**
    - Orchestrate the above methods
    - Return: `{"success": True, "period_days": since_days, "published_count": N, "aggregate": {...}, "suggestions": [...], "items": [...top 10 with metrics...], "message": "Performance summary for your last {N} published posts..."}`
    - If no published content: `{"success": True, "published_count": 0, "message": "No published content found in the last {since_days} days. Create and publish content to start tracking performance."}`

    For tests: Mock `ContentCalendarService.list_calendar` to return sample published items. Mock `get_social_analytics` to return sample metrics. Test suggestion generation with various metric patterns. Use `pytest.mark.asyncio`.

    ```python
    async def get_content_performance(
        since_days: int = 30,
        platform: str | None = None,
    ) -> dict:
        """Get performance summary for published content with improvement suggestions.

        Fetches engagement data (likes, shares, comments, impressions) for content
        published in the specified period and generates actionable improvement suggestions.

        Args:
            since_days: Lookback period in days (default: 30).
            platform: Optional filter by platform (twitter, instagram, linkedin, etc.).

        Returns:
            Dictionary with performance summary, aggregate metrics, and suggestions.
        """
        from app.services.content_performance_service import ContentPerformanceService
        from app.services.request_context import get_current_user_id

        try:
            user_id = get_current_user_id()
            service = ContentPerformanceService()
            result = await service.get_performance_summary(
                user_id=user_id, since_days=since_days, platform=platform
            )
            return result
        except Exception as e:
            return {"success": False, "error": str(e)}
    ```

    2. **Update CONTENT_DIRECTOR_INSTRUCTION** in `app/agents/content/agent.py`. Add a new section:

    You can show the user how their published content is performing and suggest improvements.
    - Use `get_content_performance(since_days=30)` to fetch a performance summary
    - The summary includes engagement metrics (likes, shares, comments, impressions) and actionable suggestions
    - Present insights conversationally: "Your LinkedIn posts are getting 3x more engagement than Instagram. Here's what I suggest..."
    - Each suggestion has a category, insight, and specific action the user can take

    **When to surface performance data:**

    **Connecting performance to future content:**
