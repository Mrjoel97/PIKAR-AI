<!-- API endpoints this frontend calls (created in Plan 02, but API shape is defined here for parallel execution) -->

REST API (all require Bearer token in Authorization header):
```
GET    /admin/config/agents                          -> [{agent_name, version, updated_at}]
GET    /admin/config/agents/{agent_name}             -> {agent_name, current_instructions, version, updated_at}
POST   /admin/config/agents/{agent_name}/preview-diff -> {diff: string}
       Body: {proposed_instructions: string}
PUT    /admin/config/agents/{agent_name}             -> {agent_name, version, diff, status}
       Body: {new_instructions: string}
GET    /admin/config/agents/{agent_name}/history     -> [{id, config_type, config_key, previous_value, new_value, created_at}]
POST   /admin/config/agents/{agent_name}/rollback    -> {agent_name, version, status}
       Body: {history_id: string}
GET    /admin/config/flags                           -> [{flag_key, is_enabled, description, updated_at}]
PUT    /admin/config/flags/{flag_key}                -> {flag_key, is_enabled, status}
       Body: {is_enabled: boolean}
GET    /admin/config/permissions                     -> [{action_name, action_category, autonomy_level, risk_level, description}]
PUT    /admin/config/permissions/{action_name}       -> {action_name, autonomy_level, status}
       Body: {autonomy_level: "auto"|"confirm"|"blocked"}
GET    /admin/config/mcp-endpoints                   -> [{name, url, status}]
```

Create the admin config page and 6 components. Follow the established admin dark-theme patterns from integrations/page.tsx and analytics/page.tsx.

**1. DiffPanel.tsx** — Stateless component. Props: `{ diff: string }`. Renders a `<pre>` with monospace font on dark background. Split diff by newlines, color-code each line: green-400 for `+` lines, red-400 for `-` lines, blue-400 for `@@` lines, gray-300 for context. Use `text-xs font-mono bg-gray-900 rounded p-4 overflow-x-auto whitespace-pre-wrap`. Show "No changes" message when diff is empty string.

**2. AgentConfigEditor.tsx** — `'use client'` component. Props: `{ agentName: string; token: string }`. State: `instructions` (textarea value), `originalInstructions` (loaded from API), `diff` (string from preview-diff endpoint), `isSaving`, `isLoadingDiff`, `version`, `error`. On mount, fetch `GET /admin/config/agents/{agentName}` to load current instructions. Provide a `<textarea>` with dark theme (bg-gray-800, text-gray-100, font-mono, min-height 300px). When textarea content differs from originalInstructions, show a "Preview Diff" button that POSTs to `/admin/config/agents/{agentName}/preview-diff` and renders the DiffPanel below. Show a "Save Changes" button (disabled when no changes or isSaving) that PUTs to `/admin/config/agents/{agentName}`. On 422 (injection), show error with violations. On success, update version display and reset diff. Show current version number badge.

**3. VersionHistory.tsx** — `'use client'` component. Props: `{ agentName: string; token: string; onRollback: () => void }`. Fetches `GET /admin/config/agents/{agentName}/history`. Renders a list of history entries showing: created_at (formatted), config change summary (truncated previous/new value), and a "Restore" button for each entry. Restore button POSTs to `/admin/config/agents/{agentName}/rollback` with `{ history_id }`. On success, calls `onRollback()` callback so the parent can refresh the editor. Use `window.confirm("Restore this version? The current instructions will be replaced.")` before rollback.

**4. FeatureFlagRow.tsx** — `'use client'` component. Props: `{ flagKey: string; isEnabled: boolean; description: string | null; token: string; onToggle: (key: string, enabled: boolean) => void }`. Renders a row with: flag name (formatted from snake_case), description, and a toggle switch. Toggle switch is a styled checkbox (or button with green/gray states). On toggle, PUTs to `/admin/config/flags/{flagKey}` with `{ is_enabled: !isEnabled }`. Calls `onToggle` on success. Show brief "Changes take effect within 60 seconds" note in the description area.

**5. AutonomyTable.tsx** — `'use client'` component. Props: `{ token: string }`. Fetches `GET /admin/config/permissions` on mount. Renders a table with columns: Action Name, Category, Current Tier, Risk Level. The "Current Tier" column is a `<select>` dropdown with options: auto, confirm, blocked. On change, PUTs to `/admin/config/permissions/{actionName}` with `{ autonomy_level: newValue }`. Use `window.confirm("Change autonomy tier for {actionName} to {newLevel}?")` before saving. Color-code tiers: auto=emerald, confirm=amber, blocked=rose. Group rows by action_category with section headers.

**6. McpEndpoints.tsx** — `'use client'` component. Props: `{ token: string }`. Fetches `GET /admin/config/mcp-endpoints` on mount. Renders a read-only card list of MCP endpoints showing: name, URL, status badge (active=emerald, inactive=gray). Include a note: "MCP endpoint management is read-only. Contact a developer to add or remove endpoints."

**7. config/page.tsx** — `'use client'` page. Uses tab state (useState) to switch between 4 tabs: "Instructions", "Feature Flags", "Autonomy", "MCP Endpoints". Tab bar styled with dark theme (bg-gray-800, active tab = indigo-600 bottom border). Gets auth token via `supabase.auth.getSession()`.

**Instructions tab:** Shows a dropdown `<select>` of all 10 agent names (fetched from `GET /admin/config/agents`). Below the dropdown, renders `AgentConfigEditor` for the selected agent and `VersionHistory` panel (collapsible accordion). When agent changes, both components reload.

**Feature Flags tab:** Fetches `GET /admin/config/flags` and renders a `FeatureFlagRow` for each flag.

**Autonomy tab:** Renders `AutonomyTable`.

**MCP Endpoints tab:** Renders `McpEndpoints`.

All fetch calls use the established pattern: `const { data: { session } } = await supabase.auth.getSession()` then `fetch(url, { headers: { Authorization: \`Bearer ${session.access_token}\` } })`. Use `useCallback` for fetch functions. Handle loading states with gray skeleton placeholders. Handle errors with red text banners.
