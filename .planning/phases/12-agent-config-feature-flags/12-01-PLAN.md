Create `app/services/agent_config_service.py` with:

1. **generate_instruction_diff(old: str, new: str) -> str** — Use `difflib.unified_diff` from stdlib. Split on `splitlines(keepends=True)`, use fromfile="current" tofile="proposed". Return joined diff string. Return empty string when old == new.

2. **validate_instruction_content(text: str) -> list[str]** — Check against _INJECTION_PATTERNS regex list (as defined in RESEARCH.md Pattern 4) and _MAX_INSTRUCTION_LENGTH = 32000. Return list of violation description strings. Empty list = valid.

3. **get_flag(key: str, default: bool = False) -> bool** — Async function. Redis cache key = `admin:feature_flag:{key}` with 60s TTL. Read Redis via `get_cache_client()` first; on hit return `json.loads(cached)`. On miss or error: query `admin_feature_flags` table via `get_service_client()`, cache result in Redis with SETEX 60s, return value. Use try/except around all Redis calls (non-fatal, fail open to DB). If flag not in DB, return `default`.

4. **set_flag(key: str, enabled: bool, changed_by: str | None = None) -> dict** — Write to DB via upsert on `admin_feature_flags`, then SETEX Redis cache. Also write a row to `admin_config_history` with config_type='feature_flag', config_key=key. Return `{"flag_key": key, "is_enabled": enabled, "status": "updated"}`.

5. **get_agent_config(agent_name: str) -> dict | None** — Query `admin_agent_configs` by agent_name. Return dict with current_instructions, version, updated_at, or None if not found.

6. **save_agent_config(agent_name: str, new_instructions: str, changed_by: str | None = None) -> dict** — First call `validate_instruction_content(new_instructions)`. If violations, return `{"error": "Injection validation failed", "violations": violations}`. Otherwise: read current config, generate diff, update `admin_agent_configs` (increment version), insert `admin_config_history` row (config_type='agent_instruction', config_key=agent_name, previous_value=old instructions JSON, new_value=new instructions JSON). Return `{"agent_name": ..., "version": new_version, "diff": diff_str, "status": "updated"}`.

7. **get_config_history(agent_name: str | None = None, config_type: str | None = None, limit: int = 20) -> list[dict]** — Query `admin_config_history` ordered by created_at DESC. Filter by config_key=agent_name and/or config_type if provided.

8. **rollback_agent_config(history_id: str, agent_name: str, changed_by: str | None = None) -> dict** — Fetch the history row by ID. Extract `previous_value` JSON to get the old instructions text. Run `validate_instruction_content` on it (defence-in-depth per RESEARCH.md Pitfall 5). If valid, call `save_agent_config` with the restored text. Return result dict.

Create `tests/unit/admin/test_config_service.py` with tests covering all behaviors listed above. Use `unittest.mock.patch` to mock `get_service_client` and `get_cache_client`. Test both cache-hit and cache-miss paths for `get_flag`. Test injection patterns are detected. Test diff generation with matching and differing inputs. Test save_agent_config rejects injections and succeeds on clean input.

All Supabase calls that are synchronous must use `asyncio.to_thread` via the existing `execute_async` wrapper or direct `asyncio.to_thread`. Follow the analytics.py and integrations.py patterns for sync Supabase client usage from async functions.
