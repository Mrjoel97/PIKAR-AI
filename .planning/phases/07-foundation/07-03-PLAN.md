  <action>
    1. Create `app/routers/admin/chat.py`:

       **Request model:** Accept JSON body with: `session_id` (optional str — create new if missing), `message` (str, required), `confirmation_token` (optional str for confirm flow).

       **Session management:**
       - If no session_id: create new row in `admin_chat_sessions` (admin_user_id from require_admin, title from first 50 chars of message), return session_id in first SSE event
       - If session_id provided: verify it belongs to admin_user_id (query admin_chat_sessions), reject if not found/not owned
       - Save user message to `admin_chat_messages` (session_id, role="user", content=message)

       **Chat history endpoint:**
       - Add `GET /admin/chat/history/{session_id}` — returns all messages for a session ordered by created_at
       - Add `GET /admin/chat/sessions` — returns all sessions for the admin user ordered by updated_at desc

       **Confirmation flow:**
       - If confirmation_token provided: call `consume_confirmation_token(token)`
       - If consumed successfully: extract action_details, execute the confirmed action directly (call the tool function), log via `log_admin_action(source="ai_agent")`
       - If token invalid/expired: return SSE error event "Confirmation token expired or already used"

       **SSE streaming:**
       - Create ADK `Runner` with `InMemorySessionService` and `admin_agent`
       - Follow the existing pattern from `fast_api_app.py` for running the agent and yielding SSE events
       - For each agent response chunk, yield `data: {json}\n\n` format
       - After agent completes: save agent response to `admin_chat_messages` (role="agent")
       - If agent response contains `requires_confirmation`, call `store_confirmation_token` to persist the token in Redis before streaming the confirmation details to the client
       - Log chat interaction via `log_admin_action(source="ai_agent")`

       **Rate limiting:** `@limiter.limit("30/minute")` on POST /admin/chat
       **Auth:** `Depends(require_admin)` on all endpoints

       **Error handling:**
       - Wrap entire generator in try/except
       - On error, yield SSE error event and close stream
       - Log errors but do not crash the server

    2. Update `app/routers/admin/__init__.py`:
       - Import chat router: `from app.routers.admin.chat import router as chat_router`
       - Include it: `admin_router.include_router(chat_router)`

    3. Verify the endpoint works with lint check.
