<objective>
Create the database schema, admin authentication middleware, encryption service, and check-access endpoint that form the security foundation for the entire admin panel.

Purpose: Every subsequent admin feature depends on these three pillars: (1) the database tables that store admin state, (2) the require_admin dependency that gates all admin endpoints, and (3) the MultiFernet encryption that protects secrets. Without this plan, no admin endpoint can be secured and no admin data can be stored.

Output: Supabase migration with 9 admin tables, require_admin FastAPI dependency, MultiFernet encryption service, GET /admin/check-access endpoint, and test scaffolds for all of the above.
</objective>
