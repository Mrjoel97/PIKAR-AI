# Phase 27.1 Plan 02: DOMPurify Sanitizer and Community Router Validation Summary

**DOMPurify replaces incomplete hand-rolled HTML sanitizer; Pydantic Field constraints close unbounded string injection on community posts/comments**

## Performance

- **Duration:** ~10 min
- **Started:** 2026-03-27T00:16:00Z
- **Completed:** 2026-03-27T00:25:55Z
- **Tasks:** 1 (TDD: RED + GREEN)
- **Files modified:** 4

## Accomplishments

- Replaced 15-line hand-rolled DOM sanitizer (missed data: URIs, SVG XSS, CSS injection) with a 12-line DOMPurify wrapper in a shared utility at `frontend/src/lib/sanitize.ts`
- Removed misleading "V1 MVP trust the agent HTML" comment from the public page renderer
- Added Pydantic Field constraints to CreatePostRequest (title 1-200, body 1-10000, tags 0-10 items/50 chars each, category 0-50) and CreateCommentRequest (body 1-5000)
- Added field_validator for per-tag length enforcement and blank-tag rejection with automatic whitespace stripping
- 22 validation tests covering all boundary conditions (exact limits, one-over, empty, None), all passing

## Task Commits

1. **Task 1: Replace hand-rolled sanitizer with DOMPurify and add community router validation** - `403d55e` (feat)

## Files Created/Modified

- `frontend/src/lib/sanitize.ts` — Centralized DOMPurify wrapper with USE_PROFILES html, FORBID style tags/attrs
- `frontend/src/app/p/[id]/page.tsx` — Removed hand-rolled sanitizeHtml function, imported shared utility
- `app/routers/community.py` — Added Field constraints + field_validator to CreatePostRequest and CreateCommentRequest
- `tests/unit/test_router_validation.py` — 22 Pydantic validation boundary tests, all passing

## Decisions Made

- Used `USE_PROFILES: { html: true }` with `FORBID_TAGS: ['style']` and `FORBID_ATTR: ['style']` to block CSS injection vectors while allowing standard content markup
- Chose `Field(max_length=10)` on the list field for item count limit (Pydantic v2 list max_length) combined with `field_validator` for per-element length — separating count vs. element constraints is cleaner
- Tag whitespace stripping done in the validator return value rather than pre-validation to normalize stored data
