## Advisory Finding (Warning — Not a Gap)

**DOMPurify is not a direct `package.json` dependency.**

`frontend/src/lib/sanitize.ts` imports from `dompurify`, which is present in `node_modules` today as an optional dependency of `jspdf` (`package-lock.json:9783`). The module resolves correctly and DOMPurify 3.3.1 bundles its own TypeScript declarations, so the import compiles and works.

However, the dependency is implicit: if `jspdf` drops DOMPurify as an optional dep in a future release, or if `npm install --omit=optional` is run in CI, the import will fail at runtime without any compile-time warning.

**Recommendation:** Add `"dompurify": "^3.3.1"` directly to `frontend/package.json` `dependencies` to make the reliance explicit and robust against transitive dep changes. This is a cleanup item, not a blocker — goal achievement is not affected today.
