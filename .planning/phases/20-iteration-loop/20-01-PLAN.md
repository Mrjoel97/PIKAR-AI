Create the backend iteration service and router endpoints for screen editing, version history, rollback, and approval.

Purpose: Users can request natural-language changes to screens via Stitch edit_screens, view iteration history, roll back to any previous version, and explicitly approve screens before the workflow advances. This is the backend foundation that the frontend (Plan 02) will consume.

Output: `iteration_service.py` with `edit_screen_variant` async generator, 4 new router endpoints in `app_builder.py`, and comprehensive unit tests for both.

  <action>
    Write tests first (RED), then implement.

    **Test file** (`tests/unit/app_builder/test_iteration_service.py`):
    Mock `get_stitch_service`, `persist_screen_assets`, and `get_service_client` using `unittest.mock.patch`. For each test, verify the exact arguments passed to `call_tool` and `persist_screen_assets`, and the events yielded by the async generator. Use `async for` to collect events into a list.

    **Service file** (`app/services/iteration_service.py`):
    Create two exported functions:

    1. `_get_locked_design_markdown(project_id: str, user_id: str) -> str | None` — Query `design_systems` table for the project. If `locked=true`, return `raw_markdown`. If unlocked or no row, return `None`.

    2. `edit_screen_variant(project_id, screen_id, user_id, stitch_project_id, stitch_screen_id, change_description, design_system_markdown, iteration_number) -> AsyncIterator[dict[str, Any]]` — Async generator that:
       a. Builds prompt: if `design_system_markdown` is not None, prepend it as `"{markdown}\n\nEdits: {change_description}"`, else use `change_description` as-is
       b. Yields `{"step": "editing", "message": f"Applying edit: {change_description[:60]}..."}`
       c. Calls `service.call_tool("edit_screens", {"projectId": stitch_project_id, "prompt": prompt, "selectedScreenIds": [stitch_screen_id]})` — MUST be an array, even for single screen
       d. **Fallback**: If response lacks `html_url`/`htmlUrl`, call `service.call_tool("get_screen", {"projectId": stitch_project_id, "screenId": new_screen_id, "name": f"projects/{stitch_project_id}/screens/{new_screen_id}"})` to get download URLs
       e. Calls `persist_screen_assets(stitch_response=response, user_id=user_id, project_id=project_id, screen_id=screen_id, variant_index=iteration_number)`
       f. Extracts `new_stitch_screen_id` from response (`screenId` or `screen_id` key)
       g. Inserts new `screen_variants` row with `is_selected=True`, `iteration=iteration_number`, `stitch_screen_id=new_stitch_screen_id`
       h. Deselects all other variants for this screen: `update({"is_selected": False}).eq("screen_id", screen_id).neq("id", variant_id)`
       i. Yields `{"step": "edit_complete", "variant_id": variant_id, "screenshot_url": ..., "html_url": ..., "iteration": iteration_number, "screen_id": screen_id}`
       j. Yields `{"step": "ready", "screen_id": screen_id, "iteration": iteration_number}`

    Use the same import pattern as `screen_generation_service.py`: `from app.services.stitch_assets import persist_screen_assets`, `from app.services.stitch_mcp import get_stitch_service`, `from app.services.supabase import get_service_client`.

    CRITICAL anti-patterns to avoid:
    - Do NOT use `asyncio.gather` — sequential await only (Lock constraint)
    - Do NOT yield SSE events before `persist_screen_assets` returns
    - Do NOT pass bare string to `selectedScreenIds` — must be a list
    - Do NOT use `app_screens.id` as stitch_screen_id — use `screen_variants.stitch_screen_id`
