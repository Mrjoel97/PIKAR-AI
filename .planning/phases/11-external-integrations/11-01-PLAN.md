<objective>
Create the integration proxy service and CRUD/proxy API router for Sentry, PostHog, GitHub, and Stripe connections.

Purpose: This plan builds the entire backend data layer for external integrations — the proxy service that handles decrypt-cache-call-return for all providers, the CRUD endpoints for managing integration connections (with Fernet-encrypted key storage), proxy endpoints that serve as the server-side pass-through to each provider's API, and the test-connection endpoint for verifying API keys work. This is the foundation that both the AdminAgent tools (Plan 02) and the frontend page (Plan 03) consume.

Output: `app/services/integration_proxy.py`, `app/routers/admin/integrations.py`, router registration, PyGithub dependency, and unit tests covering proxy caching, session budgets, CRUD operations, and key masking.

    **Step 1: Add PyGithub dependency.**
    Run `uv add "PyGithub~=2.8.1"` to add the GitHub SDK.

    **Step 2: Create `app/services/integration_proxy.py`.**
    This is the shared proxy service for all providers. It contains:

    A) `IntegrationProxyService` class with a static `call()` method that:
       - Accepts provider, operation, api_key, config, params
       - Builds cache key: `intg_proxy:{provider}:{operation}:{hashlib.md5(json.dumps(params, sort_keys=True).encode()).hexdigest()}`
       - Checks `cache.get_generic(key)` — returns cached value if found
       - Dispatches to provider-specific fetch functions based on provider+operation
       - Stores result via `cache.set_generic(key, data, ttl=ttl_seconds)` (default TTL: 180s for Sentry/PostHog/GitHub, 300s for Stripe)
       - Returns the data dict

    B) `check_session_budget()` async function:
       - Takes session_id, provider, max_calls=20
       - Uses `cache._ensure_connection()` to get bare Redis client
       - If Redis unavailable, returns True (fail open)
       - Key: `intg_budget:{session_id}:{provider}`
       - Calls `client.incr(key)` atomically, then `client.expire(key, 300)` if count==1
       - Returns `count <= max_calls`

    C) Provider-specific fetch functions (private):
       - `_fetch_sentry_issues(api_key, config, params)` — httpx GET to `https://sentry.io/api/0/projects/{org_slug}/{project_slug}/issues/` with Bearer auth, params: query=is:unresolved, limit from params (default 25), statsPeriod=24h. Transform response to list of `{id, title, culprit, count, first_seen, last_seen, level, status, permalink}`.
       - `_fetch_sentry_issue_detail(api_key, config, params)` — httpx GET to `https://sentry.io/api/0/organizations/{org_slug}/issues/{issue_id}/`. Transform to `{id, title, culprit, count, first_seen, last_seen, level, metadata, tags}`.
       - `_fetch_posthog_events(api_key, config, params)` — httpx GET to `{base_url}/api/projects/{project_id}/events/` with Bearer auth. base_url defaults to `https://us.posthog.com` if not set. Transform to `{results: [{event, timestamp, person, properties}], count}`.
       - `_fetch_posthog_insights(api_key, config, params)` — httpx GET to `{base_url}/api/projects/{project_id}/insights/`. Transform to `{results: [{id, name, description, last_refresh}], count}`.
       - `_fetch_github_prs(api_key, config, params)` — use PyGithub wrapped in `asyncio.to_thread()`. Sync function `_get_prs_sync(token, owner, repo, state="open")` gets repo, calls `repo.get_pulls(state=state, sort="updated", direction="desc")`, returns list of `{number, title, state, url, author, created_at, updated_at, mergeable}` limited to 25.
       - `_fetch_github_pr_status(api_key, config, params)` — PyGithub via `asyncio.to_thread()`. Gets PR by number, returns `{number, title, state, mergeable, checks: [{name, status, conclusion}], review_state}`.
       - `_fetch_stripe_summary(api_key, config, params)` — Using Stripe SDK per discretion (CONTEXT.md suggested httpx, but stripe>=7.0.0 is already installed in pyproject.toml and the SDK provides typed responses, automatic pagination, and idempotency — superior to raw httpx for Stripe). Wrapped in `asyncio.to_thread()`. Sync function calls `stripe.Subscription.list(api_key=api_key, limit=100)` and `stripe.Balance.retrieve(api_key=api_key)`. Returns `{active_subscriptions, total_subscriptions, balance: {available, pending}}`.

    D) `_test_provider_connection(provider, api_key, config)` async function:
       - Sentry: GET `https://sentry.io/api/0/` with Bearer auth (returns API info)
       - PostHog: GET `{base_url}/api/projects/{project_id}/` with Bearer auth
       - GitHub: `asyncio.to_thread(Github(token).get_rate_limit)` — if succeeds, connected
       - Stripe: `asyncio.to_thread(stripe.Balance.retrieve, api_key=api_key)` — fastest read (using Stripe SDK per discretion decision above)
       - Returns `{"healthy": True}` or `{"healthy": False, "error": str(e)}`

    All httpx calls use `httpx.AsyncClient(timeout=10.0)`. All sync SDK calls use `asyncio.to_thread()`.
    Guard against NULL api_key_encrypted before decrypting: `if not row.get("api_key_encrypted"): return {"error": "API key not configured"}`.

    **Step 3: Create `app/routers/admin/integrations.py`.**
    FastAPI router following the analytics.py pattern:

    A) Pydantic models:
       - `IntegrationUpsertBody(BaseModel)`: api_key (str, optional), base_url (str | None = None), config (dict = {}). api_key is optional because admin may update config without changing the key.
       - `IntegrationResponse(BaseModel)`: provider, is_active, health_status, key_last4 (str | None), base_url, config, updated_at

    B) CRUD endpoints:
       - `GET /integrations` — list all integrations. For each row, compute key_last4: if api_key_encrypted is not None, decrypt it and take `"****..." + plaintext[-4:]`; return null if no key. Rate limit: 120/minute.
       - `PUT /integrations/{provider}` — upsert. Validate provider in ["sentry", "posthog", "github", "stripe"]. If api_key provided, encrypt via `encrypt_secret(body.api_key)`. Upsert into admin_integrations (use Supabase `.upsert()` with `on_conflict="provider"`). Set is_active=True when key is saved. Rate limit: 30/minute.
       - `DELETE /integrations/{provider}` — delete row from admin_integrations. Rate limit: 30/minute.
       - `POST /integrations/{provider}/test` — decrypt key, call `_test_provider_connection()`, update health_status in DB row. Rate limit: 10/minute.

    C) Proxy endpoints (all require_admin, rate limited 60/minute):
       - `GET /integrations/sentry/proxy/issues` — fetch integration row, decrypt key, call `IntegrationProxyService.call("sentry", "get_issues", ...)`, return result.
       - `GET /integrations/sentry/proxy/issues/{issue_id}` — same pattern with `get_issue_detail` operation.
       - `GET /integrations/posthog/proxy/events` — same pattern for PostHog events.
       - `GET /integrations/posthog/proxy/insights` — same pattern for PostHog insights.
       - `GET /integrations/github/proxy/prs` — same pattern for GitHub PRs.
       - `GET /integrations/github/proxy/prs/{pr_number}` — same pattern for PR status.
       - `GET /integrations/stripe/proxy/summary` — same pattern for Stripe summary.

    All proxy endpoints share a helper `_get_integration(provider)` that: fetches from admin_integrations, checks is_active, decrypts api_key, guards against NULL api_key_encrypted. Returns (api_key, config, base_url) or raises HTTPException(400).
