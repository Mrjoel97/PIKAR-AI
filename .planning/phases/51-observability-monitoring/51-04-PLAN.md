  <action>
Create `frontend/src/app/(admin)/observability/page.tsx` as a client component ('use client') following the billing page pattern.

**Page structure:**

1. **Top-level:** `'use client'` component with `useState` for tab, window, loading, error states. Use `useSearchParams` and `useRouter` from `next/navigation` for tab persistence in URL (`?tab=errors`).

2. **Auth pattern:** Same as billing page — `createClient()` from `@/lib/supabase/client`, fetch with `Bearer ${session.access_token}`, handle auth errors.

3. **API URL:** `process.env.NEXT_PUBLIC_API_URL || 'http://localhost:8000'`

4. **Time-range picker (shared across all tabs):**
   - Row of 4 buttons: `1h`, `24h`, `7d`, `30d`
   - Active button highlighted with `bg-indigo-600 text-white`, inactive with `bg-gray-800 text-gray-400`
   - Changing the window triggers re-fetch for the active tab

5. **Hero metrics row (always visible above tabs):**
   - Fetches from `/admin/observability/summary` on mount and on window change
   - Four compact cards in a grid (`grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-4 gap-4`):
     a. **Error Rate 24h** — large number (e.g., "2.1%"), status color: green if <2%, amber if 2-5%, red if >5%. If `threshold_breach` is not null, show a small red dot indicator.
     b. **MTD AI Spend** — dollar amount (e.g., "$12.50"), subtitle "Projected: $45.00/mo (7-day avg)"
     c. **p95 Latency** — milliseconds (e.g., "450ms"), subtitle "24h · {sample_count} requests"
     d. **System Health** — traffic light: green circle if all health endpoints ok, amber if any degraded, red if any down. Fetches from existing `/admin/monitoring/status` endpoint.

6. **Tab bar:**
   - Four tabs: `Errors`, `Performance`, `AI Cost`, `Health`
   - Tab state synced with URL `?tab=errors|performance|cost|health`
   - Default tab: `errors`
   - Use Tailwind styling: active tab has `border-b-2 border-indigo-500 text-white`, inactive has `text-gray-400 hover:text-gray-200`

7. **Errors tab:**
   - Fetches from `/admin/observability/errors?window={window}`
   - Display: error count, total count, error rate percentage
   - `recharts` AreaChart (if cost-by-day endpoint returns trend data) or simple stat display
   - Per-agent breakdown if `agent_name` filter is applied (dropdown of agent names)
   - Empty state: "No agent activity in the last {window}. Make some requests to see metrics here."

8. **Performance tab:**
   - Fetches from `/admin/observability/latency?window={window}`
   - Display: three metric cards for p50, p95, p99 (milliseconds)
   - Agent name dropdown filter (optional, defaults to "All agents")
   - `recharts` BarChart showing p50/p95/p99 side by side if per-agent data available
   - Empty state message

9. **AI Cost tab:**
   - Fetches from `/admin/observability/cost?window={window}&group_by=agent` and `&group_by=day`
   - Top section: Total cost and monthly projection from summary data
   - `recharts` PieChart for cost-by-agent breakdown
   - `recharts` LineChart for cost-by-day trend (daily spend over window)
   - Subtitle on projection: "projection based on last 7 days" (honesty per CONTEXT.md decision)
   - Empty state message

10. **Health tab:**
    - Fetches from existing `/admin/monitoring/status` endpoint (reuse, don't duplicate)
    - Display each endpoint as a card with status indicator (green/amber/red circle)
    - Show response_time_ms and latest_check_at
    - List any open_incidents
    - Empty state: "No health check data available."

**Styling:** Reuse billing dashboard patterns — `bg-gray-900` cards with `border-gray-700/50` borders, `text-gray-100` text, `p-6` padding. Use `recharts` with dark theme colors: stroke `#818cf8` (indigo-400), fill with opacity.

**Auto-refresh:** 60-second polling interval (same as billing page).

**Loading state:** Skeleton cards with `animate-pulse bg-gray-800` during fetch.

**Error state:** Red banner with error message and retry button.
