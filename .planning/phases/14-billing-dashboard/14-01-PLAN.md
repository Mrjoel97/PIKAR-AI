    c) `issue_refund(charge_id: str, amount_cents: int | None = None, reason: str = "requested_by_customer", confirmation_token: str | None = None) -> dict`: Confirm tier. Calls `_check_autonomy("issue_refund")` which returns confirmation dict on first call. After confirmation, calls `_get_integration_config("stripe")`, then `asyncio.to_thread(_create_refund_sync, api_key, charge_id, amount_cents, reason)` directly (not through proxy -- refunds are not cached). Logs to audit trail via `log_admin_action(action="issue_refund", details={charge_id, amount_cents, reason, refund_id}, source="ai_agent")`. Import `_create_refund_sync` from integration_proxy.

    d) `detect_analytics_anomalies(days: int = 30) -> dict` (SKIL-05): Auto tier. Imports `get_usage_stats` and `get_agent_effectiveness` from the analytics tools module. Calls both, extracts daily arrays. Uses `statistics.mean()` and `statistics.stdev()` (stdlib) to compute mean and stddev. Flags values where `abs(current - mean) > 2 * stdev`. Checks DAU, MAU, and per-agent success rates. Returns `{anomalies: [{metric, current, mean, stddev, deviation}], period_days}`. Requires >= 3 data points for stddev computation; returns empty anomalies otherwise.

    e) `generate_executive_summary(days: int = 30) -> dict` (SKIL-06): Auto tier. Calls `get_usage_stats(days)` and `get_billing_metrics()` internally (reuses tools, NOT calling Stripe again if billing_metrics fails -- degrades gracefully). Builds narrative text: DAU trend direction + percentage, top/bottom agent by effectiveness, MRR/ARR summary, any anomalies from detect_analytics_anomalies. Returns `{summary_text: str, recommendations: [str], period_days}`. Recommendations are actionable strings (e.g., "Investigate declining DAU -- down 15% week-over-week").

    f) `forecast_revenue(months_ahead: int = 1) -> dict` (SKIL-10): Auto tier. Queries `subscriptions` table grouped by `created_at` month to compute approximate historical MRR per month. Uses simple linear extrapolation (least-squares on monthly MRR values). If < 7 data points (subscription rows), returns `{insufficient_data: true, reason: "Need at least 7 subscriptions for trend projection"}`. Otherwise returns `{current_mrr, projected_mrr, growth_rate_pct, confidence: "low"|"medium", months_ahead}`. Confidence is "medium" with 7-30 points, "low" otherwise.

    g) `assess_refund_risk(user_id: str) -> dict` (SKIL-11): Auto tier. Queries `subscriptions` table for the user's row (stripe_customer_id, tier, created_at, is_active). Queries `admin_analytics_daily` for the user's usage (if available, degrade gracefully). Computes: tenure_months (from created_at to now), estimated_ltv (tenure * tier price approximation), usage_level ("high"/"medium"/"low" based on message count). Risk scoring: HIGH if tenure < 2 months AND usage_level = "low", LOW if tenure > 6 months AND usage_level != "low", MEDIUM otherwise. Returns `{risk_level, tenure_months, estimated_ltv, usage_level, recommendation: str}`.
