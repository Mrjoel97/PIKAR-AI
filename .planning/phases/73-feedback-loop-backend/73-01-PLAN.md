    - Test 3: log_interaction returns a string UUID (the inserted row's id) on success, None on failure
    - Test 4: update_latest_interaction fetches the most-recent row for (session_id, agent_id) and updates it with provided kwargs, returns True on success
    - Test 5: update_latest_interaction returns False when no existing row matches (session_id, agent_id)
    - Test 6: report_interaction tool calls update_latest_interaction (not log_interaction) and does not crash
  </behavior>
  <action>
    1. In `app/services/interaction_logger.py`:
       a. Add kwargs to `log_interaction` signature: `task_completed: bool | None = None`, `was_escalated: bool = False`, `had_followup: bool = False`, `user_feedback: str | None = None`.
       b. Include these in the `data` dict when not None/default: `task_completed`, `was_escalated` (only if True), `had_followup` (only if True), `user_feedback` (only if not None).
       c. Change return type from `-> None` to `-> str | None`. After the `execute_async` insert call, capture the response and return `response.data[0]["id"]` if `response.data` is non-empty, else None. Wrap in the existing try/except so failures return None.
       d. Add a new method `update_latest_interaction` that:
          - Takes `session_id: str`, `agent_id: str`, and keyword-only args for all updatable fields (`task_completed`, `was_escalated`, `had_followup`, `user_feedback`, `skill_used`, `agent_response_summary`).
          - Queries `interaction_logs` for the most-recent row matching `session_id` + `agent_id`, ordered by `created_at DESC`, limit 1.
          - If found, updates that row with the non-None provided kwargs.
          - Returns `True` if a row was updated, `False` if no matching row.
          - Fire-and-forget pattern: all exceptions caught and warned.

    2. In `app/agents/tools/self_improve.py`:
       a. Change `report_interaction` tool to call `il.update_latest_interaction(session_id=..., agent_id=..., ...)` instead of `il.log_interaction(...)`.
       b. The tool needs the session_id. Import `get_current_session_id` from `app.services.request_context` inside the tool function (lazy import). Use it to get the current session.
       c. If `update_latest_interaction` returns False (no row found), fall back to inserting via `il.log_interaction(...)` with all kwargs -- this handles the edge case where SSE logging hasn't fired yet.
       d. Remove the `task_completed=completed` kwarg from the old `log_interaction` call site since it's now handled by `update_latest_interaction`.

    3. Create `tests/unit/test_interaction_logger.py` with mocked Supabase client (patch `app.services.interaction_logger.get_service_client` and `app.services.interaction_logger.execute_async`) to verify the behavior list above.
