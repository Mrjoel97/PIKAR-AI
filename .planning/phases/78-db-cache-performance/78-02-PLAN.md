**1. `app/services/analytics_aggregator.py` -- Switch to Supabase count aggregate:**

Replace the four `.select("user_id")` queries with `.select("*", count="exact").limit(0)`. The Supabase Python client returns the count as `result.count` when `count="exact"` is passed.

Update `_extract_count` to read from `result.count`:

```python
def _extract_count(result: Any) -> int:
    """Extract count from a Supabase query result.

    Supports:
    - result.count (from count="exact" aggregate)
    - [{"count": N}] legacy shape
    - len(result.data) fallback
    """
    if hasattr(result, "count") and result.count is not None:
        return int(result.count)
    data = result.data or []
    if data and isinstance(data[0], dict) and "count" in data[0]:
        return int(data[0]["count"])
    return len(data)
```

Rewrite the module:

```python
import cachetools
import functools
import logging
import asyncio
from typing import Any

logger = logging.getLogger(__name__)

DEFAULT_TTL = 30  # seconds
MAX_CACHE_SIZE = 10_000

# Bounded TTL cache -- entries auto-expire and total size is capped
_cache: cachetools.TTLCache = cachetools.TTLCache(maxsize=MAX_CACHE_SIZE, ttl=DEFAULT_TTL)


def get_cached(key: str) -> Any | None:
    """Get a cached value if present and not expired."""
    val = _cache.get(key)
    if val is not None:
        logger.debug("[tool_cache] HIT: %s", key)
        return val
    return None


def set_cached(key: str, value: Any, ttl: int = DEFAULT_TTL) -> None:
    """Store a value in the cache.

    Note: cachetools.TTLCache uses a single TTL for the entire cache.
    If a custom ttl differs from DEFAULT_TTL, we store the value but it
    will still expire based on the cache-wide TTL. For per-key TTL,
    we recreate the cache item by setting and relying on access-time
    based eviction.
    """
    _cache[key] = value
    logger.debug("[tool_cache] SET: %s (ttl=%ds)", key, ttl)
```

IMPORTANT: `cachetools.TTLCache` uses a single global TTL set at construction. Since the current code already ignores per-key TTL (the bug), using the global TTL is actually the correct fix. The `ttl` parameter on `set_cached` is retained for API compatibility but the cache-wide TTL of 30s governs expiry. Document this clearly.

Keep `cached_tool` decorator, `invalidate`, `invalidate_prefix`, and `clear` working with the new `_cache` type. `invalidate_prefix` needs adjustment since TTLCache is a dict subclass -- iterate `.keys()` and delete.

**2. `app/services/cache.py` -- Guard get_generic and set_generic:**

Add `_ensure_connection` guard to both methods:

```python
@with_circuit_breaker
async def get_generic(self, key: str) -> CacheResult:
    """Get a value from cache by arbitrary key."""
    try:
        client = await self._ensure_connection()
        if not client:
            return CacheResult.from_error("Redis not connected")

        raw = await client.get(key)
        if raw is None:
            await client.incr("pikar:stats:misses")
            return CacheResult.miss()
        await client.incr("pikar:stats:hits")
        try:
            return CacheResult.hit(json.loads(raw))
        except (json.JSONDecodeError, TypeError):
            return CacheResult.hit(raw)
    except (RedisConnectionError, RedisTimeoutError) as exc:
        logger.warning("Redis connection error (get_generic): %s", exc)
        return CacheResult.from_error(f"Connection error: {exc}")
    except Exception as exc:
        logger.error("Cache error (get_generic): %s", exc)
        return CacheResult.from_error(str(exc))


@with_circuit_breaker
async def set_generic(self, key: str, value: Any, ttl: int = 3600) -> bool:
    """Set a value in cache with arbitrary key and TTL."""
    try:
        client = await self._ensure_connection()
        if not client:
            return False

        serialized = json.dumps(value, default=str)
        await client.set(key, serialized, ex=ttl)
        return True
    except (RedisConnectionError, RedisTimeoutError) as exc:
        logger.warning("Redis connection error (set_generic): %s", exc)
        return False
    except Exception as exc:
        logger.error("Cache error (set_generic): %s", exc)
        return False
```
