  <behavior>
    - TestTeamKpis: get_team_kpis returns aggregate dict with total_workflows, total_initiatives, total_tasks, total_approvals, active_workflows counts across all workspace members
    - TestTeamKpis: get_per_member_kpis returns list of dicts with user_id, display_name, and individual counts for each workspace member
    - TestRoleVisibility: get_team_kpis works for any workspace member (no role check — that is in the router layer)
    - TestTeamSharing: get_shared_initiatives returns initiatives created by any workspace member (using get_workspace_user_ids scoping)
    - TestTeamSharing: get_shared_workflows returns workflow executions created by any workspace member
    - TestActivityFeed: get_activity_feed queries governance_audit_log for all workspace member_ids, groups results by (resource_type, resource_id), returns list of {resource_type, resource_id, events: [...]} clusters
    - TestActivityFeed: get_activity_feed limits to 100 most recent audit rows to prevent N+1; single query, Python grouping
  </behavior>
  <action>
    1. Create `app/services/team_analytics_service.py` with class `TeamAnalyticsService`:
       - `__init__`: store `get_service_client()` as `self.client` and instantiate `WorkspaceService` as `self.ws_service`
       - `_safe_rows(query)` helper: call `execute_async(query)`, return `.data or []` (same pattern as DashboardSummaryService)

       - `async def get_team_kpis(self, workspace_id: str) -> dict`: 
         Get members via `self.ws_service.get_workspace_members(workspace_id)`, extract `member_ids`. Run 4 count queries in sequence (initiatives, workflow_executions, tasks, pending_approvals) each filtered with `.in_("user_id", member_ids)`. Return dict with `total_initiatives`, `total_workflows`, `total_tasks`, `total_approvals`, `active_workflows`, `member_count`.

       - `async def get_per_member_kpis(self, workspace_id: str) -> list[dict]`:
         Get members, then for each member query their individual initiative/workflow/task counts. Return list of `{user_id, display_name, email, initiatives, workflows, tasks, approvals}`. Batch using `.in_("user_id", [single_id])` per member — acceptable for teams <20 members.

       - `async def get_shared_initiatives(self, workspace_id: str, limit: int = 50) -> list[dict]`:
         Get member_ids, query `initiatives` table with `.in_("user_id", member_ids)` ordered by `updated_at DESC`, limit. Return rows.

       - `async def get_shared_workflows(self, workspace_id: str, limit: int = 50) -> list[dict]`:
         Get member_ids, query `workflow_executions` table with `.in_("user_id", member_ids)` ordered by `created_at DESC`, limit. Return rows.

       - `async def get_activity_feed(self, workspace_id: str, limit: int = 100) -> list[dict]`:
         Get member_ids. Single query: `governance_audit_log` with `.in_("user_id", member_ids).order("created_at", desc=True).limit(limit)`. Group in Python by `(resource_type, resource_id)` tuple key. Return list of `{resource_type, resource_id, resource_name (from first event details if present), events: [row_dicts]}` sorted by most recent event timestamp per group.

    2. Create `tests/unit/test_team_analytics.py` with test classes:
       - TestTeamKpis: mock workspace members, mock table queries returning sample rows, verify aggregate counts
       - TestRoleVisibility: verify service does NOT enforce roles (that is router-level)
       - TestTeamSharing: mock initiatives/workflow_executions queries, verify results scoped to member_ids
       - TestActivityFeed: mock audit_log rows with multiple resources, verify grouping by resource, verify limit applied, verify sort order (most recently active resource group first)
  </action>
  <verify>
    <automated>uv run pytest tests/unit/test_team_analytics.py -x</automated>
  </verify>
  <done>TeamAnalyticsService returns correct aggregate and per-member KPIs. Activity feed groups by resource correctly. All queries scoped to workspace member_ids. Tests pass.</done>
</task>
