Create `app/services/data_query_service.py` with class `DataQueryService(BaseService)`:

1. **classify_query(question: str) -> dict** — Use keyword/pattern matching (not LLM) for fast routing:
   - Revenue, income, sales, money, earnings, payments -> source="financial_records"
   - Customers, subscribers, subscriptions, churn, retention -> source="subscriptions"
   - Orders, products, inventory, Shopify -> source="shopify"
   - Events, tracking, analytics, usage -> source="analytics_events"
   - SQL, database, query my database, BigQuery, postgres -> source="external_db"
   - Return `{"source": str, "confidence": float, "parsed_intent": str}`

2. **query_internal_data(question: str, source: str, user_id: str) -> dict** — Route to appropriate data source:
   - "financial_records": Query `financial_records` table with date filtering parsed from question (use dateutil-style parsing: "last month", "this week", "Q1"). Aggregate amounts by transaction_type. Return revenue, expense totals.
   - "subscriptions": Query `subscriptions` table. Count active/churned. Return customer count, churn count.
   - "shopify": Query `shopify_orders` table with date filtering. Aggregate by totals. Return order count, total revenue.
   - "analytics_events": Query `analytics_events` table filtered by event_name/category. Return event counts, top events.
   - "external_db": Delegate to `external_db_query()` from external_db_tools.

3. **format_nl_answer(raw_data: dict, question: str) -> str** — Use Gemini Flash to generate a 2-3 sentence plain-English answer from the raw data and original question. Include the key number prominently. Format: "[Key insight]. [Supporting detail]. [One-line recommendation if applicable]."

4. **format_chart_data(raw_data: dict, source: str) -> dict** — Extract chart-friendly structure:
   - `{"chart_type": "bar"|"line"|"pie", "labels": list[str], "values": list[float], "title": str}`
   - Choose chart_type based on data shape: time series -> line, categories -> bar, proportions -> pie

Create `tests/unit/services/test_data_query_service.py` following the pattern from `test_observability_metrics_service.py`:
- Use Windows-safe sys.modules stub pattern

- Test query_internal_data with mocked DB responses

Use `BaseService` (not `AdminService`) since this service runs in user context. Import pattern: `from app.services.base_service import BaseService`.

```python
async def nl_data_query(question: str) -> dict:
    """Answer a natural language data question with plain-English response and chart data.

    Auto-routes to the correct data source (financial records, subscriptions,
    Shopify orders, analytics events, or external databases) based on the question.

    Args:
        question: Natural language question about business data.

    Returns:
        Dictionary with answer (plain-English), chart_data (for rendering),
        source (which data source was used), and raw_data (underlying numbers).
    """
```

Implementation:
- Import `DataQueryService` lazily inside function body (pattern used by all other tools in this file)
- Get user_id from `get_current_user_id()`
- Call `service.classify_query(question)` to determine source
- Call `service.query_internal_data(question, source, user_id)` to get raw data
- Call `service.format_nl_answer(raw_data, question)` to get plain-English answer
- Call `service.format_chart_data(raw_data, source)` to get chart structure
- Return `{"success": True, "answer": str, "chart_data": dict, "source": str, "raw_data": dict}`
- Wrap in try/except returning `{"success": False, "error": str}` on failure
