Create `app/services/weekly_report_service.py` with class `WeeklyReportService(BaseService)`:

1. **generate_weekly_report(user_id: str) -> dict** — Compile a 1-page weekly business report:
   - Query `financial_records` for current week and previous week (Mon-Sun), aggregate revenue, expenses, net
   - Query `analytics_events` for event counts this week vs last week
   - Query `subscriptions` for active subscriber count (if user has Stripe integration)
   - Query `shopify_orders` for order count and total (if user has Shopify integration)
   - Compute week-over-week changes as percentages
   - Flag anomalies: any metric changing >25% week-over-week gets flagged
   - Use Gemini Flash to generate a 3-sentence executive summary from the raw numbers
   - Return structured dict:
     ```python
     {
         "period": {"start": str, "end": str, "label": "Week of Apr 7"},
         "revenue_summary": {"current": float, "previous": float, "change_pct": float, "currency": str},
         "top_metrics": [{"name": str, "value": float, "change_pct": float, "trend": "up"|"down"|"stable"}],
         "anomalies": [{"metric": str, "expected": float, "actual": float, "severity": str}],
         "executive_summary": str,  # Plain-English 3-sentence summary
         "generated_at": str,
     }
     ```
   - For database queries, use `execute_async` wrapping Supabase client calls (pattern from analytics_service.py)
   - Calculate week boundaries using Monday as start of week

2. **get_data_catalog_suggestions(provider: str) -> list[dict]** — Return report suggestions based on integration type:
   - "stripe": [{"title": "Monthly Revenue Trend", "description": "Track revenue over time", "report_type": "revenue_trend"}, {"title": "Customer Churn Analysis", ...}, {"title": "Payment Success Rate", ...}]
   - "shopify": [{"title": "Sales by Product", ...}, {"title": "Order Volume Trend", ...}, {"title": "Average Order Value", ...}]
   - "google_ads"/"meta_ads": [{"title": "Ad Spend vs Revenue", ...}, {"title": "Campaign ROI", ...}]
   - "postgresql"/"bigquery": [{"title": "Custom SQL Reports", ...}, {"title": "Table Summary", ...}]
   - Default for unknown: [{"title": "Activity Summary", ...}]
   - Each suggestion includes: title, description, report_type, required_data_sources

3. **get_available_integrations(user_id: str) -> list[dict]** — Query `integration_credentials` table for user's connected integrations. Return list of `{"provider": str, "account_name": str, "connected_at": str}`.

4. **format_report_as_briefing_card(report: dict) -> dict** — Format the weekly report as a briefing-compatible card:
   ```python
   {
       "type": "weekly_report",
       "title": "Weekly Business Report",
       "summary": str,  # executive_summary field
       "generated_at": str,
       "sections": [...]  # revenue, metrics, anomalies as display sections
   }
   ```
