Create `app/services/campaign_performance_summarizer.py` with class `CampaignPerformanceSummarizer`:

1. Method `async def summarize_all_platforms(self, user_id: str, days: int = 7) -> dict`:
   - Uses AdCampaignService.list_ad_campaigns() to get all user campaigns across both platforms
   - For each campaign, calls AdSpendTrackingService.get_spend_summary() for current period (last N days) AND prior period (N days before that) to compute WoW trends
   - Builds a structured result dict with:
     - `summary_text`: Plain-English paragraph (e.g. "Your Google Ads spent $340 this week and brought 12 customers at $28.33 each -- 15% better than last week. Meta Ads spent $200 and brought 8 customers at $25 each -- new this week.")
     - `total_spend`: float
     - `total_conversions`: int
     - `overall_cpa`: float
     - `wow_spend_change_pct`: float or None
     - `wow_conversions_change_pct`: float or None
     - `per_campaign`: list of dicts with campaign-level breakdowns
     - `period`: {"start": str, "end": str, "days": int}

2. Helper method `_format_summary_text(self, platform_summaries, totals, wow)` that builds the natural-language text:
   - Uses f-strings with proper currency formatting (${amount:,.2f})
   - WoW trend phrasing: "X% better/worse than last week" or "no prior data for comparison"
   - Handles zero-conversion case: "spent $X but no conversions tracked yet"

3. Helper method `_compute_wow(self, current, prior)` that returns percentage changes.

Write tests in `tests/unit/services/test_campaign_performance_summarizer.py`:
- Mock AdCampaignService and AdSpendTrackingService
- Verify summary_text contains expected phrases for various scenarios
- Verify numeric accuracy of CPA and WoW calculations
