Update the marketing agent with a comprehensive campaign creation wizard flow. This is an instruction-driven feature -- the agent's behavior is shaped by detailed instructions rather than new code.

1. Update `_CAMPAIGN_INSTRUCTION` to add a new section "## CAMPAIGN CREATION WIZARD":

```
## CAMPAIGN CREATION WIZARD

When a user wants to create a new campaign (detects intent like "launch a campaign",
"promote my product", "run ads", "create an ad campaign"), engage the conversational wizard:

### Step 1: Understand the Goal
Ask: "What are you promoting? (a product, service, event, or content piece?)"
Wait for the user's answer. Extract the core offering.

### Step 2: Identify the Audience
Ask: "Who's your ideal customer? (e.g., age range, interests, location, or describe them)"
Use the answer to suggest audience targeting parameters.

### Step 3: Set the Budget
Ask: "What's your daily budget? (e.g., $20/day, $50/day)"
If user gives monthly, convert to daily (÷ 30).
If user is unsure, suggest: "Most campaigns start with $20-50/day. You can adjust anytime."

### Step 4: Choose the Platform
If user specifies a platform, use it. Otherwise, auto-select:
- Product/e-commerce with visual appeal → recommend Meta Ads (visual-first platform)
- Service/B2B with search intent → recommend Google Ads (search intent capture)
- Both/unsure → recommend Meta Ads for awareness + Google Ads for intent, suggest starting with one

Present recommendation: "Based on [reason], I recommend [platform]. Want to go with that, or prefer the other?"

### Step 5: Create the Campaign
Once all info gathered, summarize back to the user:
"Here's what I'll set up:
- Campaign: [name derived from product/goal]
- Platform: [Google Ads / Meta Ads]
- Daily budget: $[amount]
- Target audience: [summary]
Ready to create? (This creates it in paused mode — you'll approve activation separately.)"

On confirmation, escalate to parent MarketingAutomationAgent which delegates to AdPlatformAgent
to call create_google_ads_campaign() or create_meta_ads_campaign().
Also call generate_utm_params() and save_campaign_utm() for tracking.

### Step 6: Post-Creation
After campaign is created:
- Confirm creation with campaign details
- Remind user it's in PAUSED status and needs activation approval
- Suggest: "Want me to write ad copy for this campaign?" (delegate to AdPlatformAgent for ad copy workflow)
- Suggest: "Want to check back on performance in a few days? I can summarize results in plain English."
```

2. Update `MARKETING_AGENT_INSTRUCTION` routing table to add:
   ```
   | "Launch a campaign", "run ads", "promote my product" | **CampaignAgent** (wizard flow) |
   ```

3. Add the ad platform connection check tools to `_CAMPAIGN_TOOLS` so the CampaignAgent can verify platform connectivity before recommending:
   - Import `connect_google_ads_status, connect_meta_ads_status` from ad_platform_tools
   - Add both to `_CAMPAIGN_TOOLS` list

4. Ensure `_CAMPAIGN_TOOLS` also includes `*CAMPAIGN_PERFORMANCE_TOOLS` (from Plan 01) if not already added.
