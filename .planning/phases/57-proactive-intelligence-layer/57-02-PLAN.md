<objective>
Build business metric anomaly detection and upgrade budget pacing alerts to plain English.

Purpose: PROACT-02 requires detecting statistical anomalies in business metrics (>2 stddev from baseline) and pushing alerts. PROACT-05 requires budget pacing alerts to be in plain English, persona-aware. The existing `_check_budget_pacing()` in ad_performance_sync_service.py fires alerts but with technical language. This plan:
1. Creates an AnomalyDetectionService with rolling baseline computation and stddev-based anomaly detection
2. Creates a metric_baselines table for storing rolling statistics
3. Upgrades budget pacing to persona-aware plain-English messages
4. Adds a Cloud Scheduler endpoint for periodic anomaly checks

Output: Working anomaly detection service + plain-English budget alerts
</objective>

       - **METRIC_SOURCES**: dict mapping metric keys to their data source and query strategy:
         ```python
         METRIC_SOURCES = {
             "revenue_daily": {"table": "dashboard_summaries", "path": "metrics.revenue"},
             "customers_total": {"table": "dashboard_summaries", "path": "metrics.customers"},
         ```
