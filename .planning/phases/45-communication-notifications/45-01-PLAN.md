  <action>
Create migration `20260405970000_notification_rules.sql` with two tables:

1. `notification_rules` table:
   - id (uuid, gen_random_uuid PK)
   - user_id (uuid, NOT NULL, FK auth.users ON DELETE CASCADE)
   - provider (text, NOT NULL, CHECK IN ('slack', 'teams'))
   - event_type (text, NOT NULL) — values like 'approval.pending', 'task.created', 'task.completed', 'workflow.completed', 'workflow.failed', 'campaign.milestone'
   - channel_id (text, NOT NULL) — Slack channel ID (C0XXXXXX) or Teams webhook URL
   - channel_name (text, NOT NULL DEFAULT '')
   - enabled (boolean, NOT NULL DEFAULT true)
   - created_at (timestamptz, DEFAULT now())
   - updated_at (timestamptz, DEFAULT now())
   - UNIQUE constraint on (user_id, provider, event_type, channel_id)

2. `notification_channel_config` table:
   - id (uuid, gen_random_uuid PK)
   - user_id (uuid, NOT NULL, FK auth.users ON DELETE CASCADE)
   - provider (text, NOT NULL, CHECK IN ('slack', 'teams'))
   - daily_briefing (boolean, NOT NULL DEFAULT false)
   - briefing_channel_id (text, nullable)
   - briefing_channel_name (text, DEFAULT '')
   - briefing_time_utc (text, NOT NULL DEFAULT '08:00')
   - created_at (timestamptz, DEFAULT now())
   - updated_at (timestamptz, DEFAULT now())
   - UNIQUE constraint on (user_id, provider)

Add RLS policies: enable RLS on both tables, SELECT/INSERT/UPDATE/DELETE policies checking auth.uid() = user_id. Add indexes on (user_id) and (user_id, provider). Add moddatetime trigger on updated_at for both tables.

- Update `slack` entry: add `chat:write.public` to scopes list (after `chat:write`)

Create `app/services/slack_notification_service.py`:
- Class `SlackNotificationService` with methods:
  - `__init__(self)` — lazy: no client created at init
  - `async send_notification(self, user_id: str, channel_id: str, event_type: str, payload: dict) -> bool` — resolves bot token via `IntegrationManager().get_valid_token(user_id, "slack")`, lazy-imports `AsyncWebClient` from `slack_sdk.web.async_client`, calls `chat_postMessage` with text (fallback) + blocks (rich Block Kit)
  - `async send_approval_request(self, user_id: str, channel_id: str, description: str, approval_token: str, details: str = "") -> dict` — sends Block Kit message with Approve (style: primary) and Reject (style: danger) buttons, action_id `approval_approve`/`approval_reject`, value `APPROVED:{token}`/`REJECTED:{token}`, block_id `approval_{token[:16]}`
  - `async send_daily_briefing(self, user_id: str, channel_id: str, briefing_data: dict) -> bool` — formats briefing as Block Kit with header "Daily Briefing", sections for metrics/pending actions/upcoming tasks
  - `async list_channels(self, user_id: str) -> list[dict]` — calls `conversations_list(exclude_archived=True, types="public_channel,private_channel", limit=200)`, returns list of `{id, name, is_private}`
  - `_build_event_blocks(self, event_type: str, payload: dict) -> list[dict]` — returns Block Kit blocks based on event_type: header block with event title, section block with mrkdwn details, divider. For `approval.pending` events, delegates to `send_approval_request` instead.
  - `_build_briefing_blocks(self, briefing_data: dict) -> list[dict]` — header "Daily Briefing", section with pending_approvals count, section with upcoming_tasks, section with key_metrics, divider between each
- Use `logger = logging.getLogger(__name__)` throughout
- Catch all Slack API exceptions, log error, return False on failure
- Use `asyncio.create_task` for fire-and-forget sends where latency matters (document in docstring)

Create `app/services/teams_notification_service.py`:
- Class `TeamsNotificationService` with methods:
  - `async send_notification(self, user_id: str, webhook_url: str, event_type: str, payload: dict) -> bool` — POSTs Adaptive Card to webhook_url using httpx.AsyncClient(timeout=10.0)
  - `async send_daily_briefing(self, user_id: str, webhook_url: str, briefing_data: dict) -> bool` — Adaptive Card with briefing sections
  - `_build_adaptive_card(self, event_type: str, payload: dict) -> dict` — returns Adaptive Card body with schema 1.2, TextBlock header (size Medium, weight Bolder), TextBlock body (wrap True), optional Action.OpenUrl linking to Pikar approval page for approval events
  - `_build_briefing_card(self, briefing_data: dict) -> dict` — Adaptive Card with ColumnSet for metrics, TextBlock list for pending items
- For approval events, include `Action.OpenUrl` button "View in Pikar" linking to `{APP_URL}/approval/{token}` (NOT Action.Submit — incoming webhooks cannot handle responses)
- Handle HTTP 429 (Teams rate limit) with logged warning
- Use `get_service_client()` to look up Teams webhook URL from `integration_credentials` where provider='teams' — the webhook URL is stored as `account_name` field

For each rule, calls appropriate provider service. Uses `pikar:notif:sent:{user_id}:{event_type}:{hash(payload)}` Redis key with 60s TTL for dedup.

- Catch individual provider failures without failing entire fan-out (log and continue)
- Import lazily: `from app.services.slack_notification_service import SlackNotificationService` inside dispatch method body to avoid circular imports
