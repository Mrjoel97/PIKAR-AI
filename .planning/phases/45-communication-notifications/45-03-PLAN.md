  <action>
Add to `app/services/scheduled_endpoints.py` — new `POST /scheduled/slack-daily-briefing` endpoint:
1. Call `_verify_scheduler(x_scheduler_secret)` for auth
2. Query `notification_channel_config` WHERE daily_briefing=True using service role client
3. For each config row:
   - Build briefing_data dict by aggregating:
     - Pending approvals count: query `approval_requests` WHERE user_id AND status='PENDING'
     - Upcoming tasks: query `tasks` WHERE user_id AND status IN ('pending','in_progress') ORDER BY due_date LIMIT 5
     - Key metrics: query `dashboard_summaries` WHERE user_id ORDER BY created_at DESC LIMIT 1 (if exists, otherwise skip)
   - If `provider == "slack"`: lazy-import `SlackNotificationService`, call `send_daily_briefing(user_id, briefing_channel_id, briefing_data)`
   - If `provider == "teams"`: lazy-import `TeamsNotificationService`, call `send_daily_briefing(user_id, briefing_channel_id, briefing_data)` (briefing_channel_id is the Teams webhook URL)
4. Return `{"sent": count, "errors": error_count}` summary
5. Wrap each user send in try/except — never let one user's failure stop others

Create `app/agents/tools/communication_tools.py`:
- `async def send_notification_to_channel(user_id: str, message: str, channel: str = "", provider: str = "") -> str`:
  - If no provider specified, auto-detect connected notification providers (query integration_credentials WHERE provider IN ('slack','teams') AND user_id). If one found, use it. If both, ask user. If none, return error.
  - For Slack: if no channel specified, try to find a default from notification_channel_config. Call `SlackNotificationService().send_notification(user_id, channel, "agent.message", {"text": message})`.
  - For Teams: get webhook URL from credentials. Call `TeamsNotificationService().send_notification(user_id, webhook_url, "agent.message", {"text": message})`.
  - Return success/failure message

- `async def list_notification_rules(user_id: str) -> str`:
  - Import NotificationRuleService, call list_rules(user_id)
  - Format as readable list: "event_type -> channel_name (provider, enabled/disabled)"

- `async def configure_notification_rule(user_id: str, event_type: str, channel_name: str, provider: str = "", enabled: bool = True) -> str`:
  - Auto-detect provider if not specified (same as send_notification_to_channel)
  - For Slack: resolve channel_name to channel_id via `SlackNotificationService().list_channels(user_id)`, find matching name
  - For Teams: channel_id = webhook URL from credentials (Teams has one channel per webhook)
  - Call `NotificationRuleService().create_rule(user_id, provider, event_type, channel_id, channel_name)`
  - Return confirmation

- Export `COMMUNICATION_TOOLS = [send_notification_to_channel, list_notification_rules, configure_notification_rule]`
- Follow raw function export pattern (not FunctionTool) matching PM_TASK_TOOLS and AD_PLATFORM_TOOLS

Update `app/agents/operations/agent.py`:
- Import `COMMUNICATION_TOOLS` from `app.agents.tools.communication_tools`
- Add `*COMMUNICATION_TOOLS` to the tools list (same pattern as `*PM_TASK_TOOLS`)

  - Auto-detect the connected notification provider when the user doesn't specify one
