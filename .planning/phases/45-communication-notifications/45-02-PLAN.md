Create `app/services/notification_rule_service.py`:
- Class `NotificationRuleService`:
  - `__init__(self)` — gets supabase service client via `get_service_client()`
  - `async list_rules(self, user_id: str, provider: str | None = None) -> list[dict]` — queries notification_rules WHERE user_id, optionally filtered by provider, ordered by created_at desc
  - `async create_rule(self, user_id: str, provider: str, event_type: str, channel_id: str, channel_name: str = "") -> dict` — upserts into notification_rules using `upsert(..., on_conflict="user_id,provider,event_type,channel_id")`, returns the row
  - `async update_rule(self, user_id: str, rule_id: str, enabled: bool) -> dict` — updates enabled flag WHERE id=rule_id AND user_id=user_id
  - `async delete_rule(self, user_id: str, rule_id: str) -> bool` — deletes WHERE id=rule_id AND user_id=user_id
  - `async get_matching_rules(self, user_id: str, event_type: str) -> list[dict]` — queries notification_rules WHERE user_id AND event_type AND enabled=True
  - `async get_channel_config(self, user_id: str, provider: str) -> dict | None` — queries notification_channel_config WHERE user_id AND provider
  - `async upsert_channel_config(self, user_id: str, provider: str, daily_briefing: bool, briefing_channel_id: str | None, briefing_channel_name: str = "", briefing_time_utc: str = "08:00") -> dict` — upserts notification_channel_config on (user_id, provider)
  - `SUPPORTED_EVENTS` class constant: list of `{"type": "approval.pending", "label": "Approval Required"}, {"type": "task.created", "label": "Task Created"}, ...` covering all 7 supported event types from research
- All queries use `execute_async` with descriptive `op_name` strings
- Uses AdminService pattern (service role client) since notification rules may be accessed by background tasks

Add to `app/routers/webhooks.py` — new `POST /webhooks/slack/interact` endpoint:

1. Read raw body bytes from request
2. Verify Slack signature: lazy-import `SignatureVerifier` from `slack_sdk.signature`, instantiate with `os.environ.get("SLACK_SIGNING_SECRET", "")`. Extract `X-Slack-Request-Timestamp` and `X-Slack-Signature` headers. Call `verifier.is_valid(body.decode(), timestamp, signature)`. If invalid or signing secret empty, return 403.
3. Parse form-encoded payload: `form = await request.form()`, `payload = json.loads(form["payload"])`
4. If `payload["type"] == "block_actions"`:
   - Extract first action: `action = payload["actions"][0]`
   - Get `action_id` ("approval_approve" or "approval_reject") and `value` ("APPROVED:{token}" or "REJECTED:{token}")
   - Split value on ":" to get status and plain token
   - Hash token: `token_hash = hashlib.sha256(token.encode()).hexdigest()`
   - Update `approval_requests` table: SET status=status, resolved_at=now() WHERE token=token_hash AND status='PENDING'. Use AdminService (service role) since this runs without user JWT context.
   - If update found a row, post follow-up message via `response_url` from payload using httpx.AsyncClient — replace the original message blocks with a confirmation: "Approved by {user_name}" or "Rejected by {user_name}" (get user_name from `payload["user"]["username"]`)
   - If no row found (already resolved or expired), post "This approval has already been processed or has expired"
5. Return `JSONResponse({"ok": True})` IMMEDIATELY — process the DB update and response_url POST in `asyncio.create_task()` to stay under Slack's 3-second timeout
6. Import json at top of file (if not already imported)

Modify `app/agents/tools/approval_tool.py`:
- After the successful `approval_requests` insert, add notification dispatch:
  ```python
  import asyncio
  asyncio.create_task(_notify_approval(requester_user_id, action_type, action_description, token))
  ```

- Add helper function `async def _notify_approval(user_id, action_type, description, token)`:
  - Lazy-import `dispatch_notification` from `app.services.notification_dispatcher`
  - Call `await dispatch_notification(user_id, "approval.pending", {"action_type": action_type, "description": description, "approval_token": token})`
  - Wrap in try/except to never break approval creation if notification fails (log warning)
- The `approval_token` in the payload is the PLAIN token (not hash) — SlackNotificationService.send_approval_request uses it to build button values that Slack will send back to the interact endpoint
