<objective>
Add a centralised FastAPI ASGI middleware (`AuditLogMiddleware`) that automatically writes
one row to `governance_audit_log` for every successful (2xx) data-mutating
(POST/PUT/PATCH/DELETE) request to a user-facing API route. The middleware extracts the
actor from the request's bearer token, derives `action_type` and `resource_type` from the
HTTP method and path, and calls the existing `GovernanceService.log_event()` (which is
already write-fault-tolerant).

Purpose: AUTH-04 requires that "every data-mutating action (create, update, delete)
produces an audit log row with actor identity, action name, target resource, and timestamp."
Today only ~4 out of 30+ routers call `governance.log_event()` manually (initiatives,
teams, workflows, governance) — the remaining ~26 routers (content, campaigns, reports,
finance, sales, support, compliance, vault, briefing, pages, community, etc.) have ZERO
audit coverage. Retrofitting every endpoint individually would be 100+ touch points and
create perpetual audit gaps as new endpoints land. A centralised ASGI middleware closes the
gap in one place and keeps coverage automatic.

Output:
- `app/middleware/audit_log.py` — ASGI middleware class + a `log_mutation` helper
- `app/fast_api_app.py` — `app.add_middleware(AuditLogMiddleware, ...)` registration
- `tests/unit/app/middleware/test_audit_log_middleware.py` — pytest coverage
</objective>

<critical_decisions>
1. **ASGI middleware (BaseHTTPMiddleware) not a router decorator** — decorators would
   require touching every endpoint. ASGI middleware sees every request automatically.

2. **Inclusion list approach (NOT exclusion-only)** — start with an explicit positive list
   of router prefixes that should be audited. This is safer than trying to enumerate
   exclusions, because new routers being added later will NOT silently start producing
   audit rows until explicitly added.

       "/app_builder": "app_builder",
   }
   ```

3. **Hard exclusions** — these prefixes are NEVER audited even if matched:
   - `/health` — health checks (would spam the audit log)
   - `/a2a` — agent-to-agent SSE chat (per-message logging is too noisy; chat is already logged via session_meta)
   - `/admin` — admin actions are already audited via app/services/admin_audit.py → admin_audit_log (separate table)
   - `/webhooks` — inbound webhooks have no human actor
   - `/auth` — auth endpoints (login/logout/refresh)

4. **action_type derivation** — `f"{resource_type}.{verb}"` where verb is:
   - POST → "created"
   - PUT, PATCH → "updated"
   - DELETE → "deleted"

5. **resource_id extraction** — best-effort: parse the second segment of the path.
   - `/initiatives/abc-123` → resource_id="abc-123"
   - `/initiatives/abc-123/checklist/item-1` → resource_id="abc-123" (the parent resource)
   - `/initiatives` (POST collection create) → resource_id=None
   This is acceptable — the response body would have the new ID but middleware should not
   read response bodies (would break SSE and large downloads).

6. **Actor resolution** — use `verify_token_fast` (no DB call) on the Authorization header.
   If the header is missing or invalid, skip the audit row (anonymous request, no actor).

7. **NEVER raise** — wrap the entire log call in try/except. The middleware MUST be
   invisible from the request flow's perspective.

8. **status_code check** — only log on 2xx. For 3xx redirects, 4xx errors, 5xx server
   errors, do NOT write a row (the action did not succeed).

9. **Background task offload** — the log_event call should NOT block the response. Use
   `asyncio.create_task()` to fire-and-forget the insert AFTER the response has been
   serialised. This means the user gets their response immediately and the audit insert
   happens out-of-band.

10. **Parametrised regression test over AUDITED_ROUTES** — a typo in a single map entry
    would silently drop audit coverage for that router without any test failing. The
    middleware test suite MUST include a `@pytest.mark.parametrize` that iterates
    `AUDITED_ROUTES.items()`, spins up a minimal FastAPI test app with one POST handler
    per prefix, fires a mutation, and asserts `log_event` was called with the expected
    `resource_type`. This proves EVERY entry in the inclusion list is wired correctly.

11. **Middleware-registration assertion test** — Task 2 MUST include a test that imports
    the real `app` from `app.fast_api_app`, iterates `app.user_middleware`, finds
    `AuditLogMiddleware`, and asserts it is present AND registered AFTER
    `OnboardingGuardMiddleware` in the stack order documented in critical_decisions #1.
    This prevents a future refactor from silently dropping the middleware registration.

    # Copyright (c) 2024-2026 Pikar AI. All rights reserved.
    # Proprietary and confidential. See LICENSE file for details.

    """Centralised audit logging middleware.

    Writes one row to governance_audit_log for every successful 2xx response to a
    POST/PUT/PATCH/DELETE on a user-facing API route. Reads (GET) and failures
    (non-2xx) are NOT logged. Health, admin, a2a, webhooks, and auth routes are
    excluded.

    The middleware NEVER raises — audit failures are logged at ERROR and swallowed
    so the wrapped request flow is unaffected.
    """

    from __future__ import annotations

    import asyncio
    import logging
    from typing import Awaitable, Callable

    from fastapi import Request, Response
    from starlette.middleware.base import BaseHTTPMiddleware

    from app.app_utils.auth import verify_token_fast
    from app.services.governance_service import get_governance_service

    logger = logging.getLogger(__name__)

    # Mutating HTTP methods that should produce an audit row.
    _MUTATING_METHODS = frozenset({"POST", "PUT", "PATCH", "DELETE"})

    # Method → action verb mapping.
    _METHOD_VERBS: dict[str, str] = {
        "POST": "created",
        "PUT": "updated",
        "PATCH": "updated",
        "DELETE": "deleted",
    }

    # Hard exclusions: these prefixes are NEVER audited, even on a mutating verb.
    # /admin is excluded because admin actions are already audited to admin_audit_log
    # via app/services/admin_audit.py (Phase 7 separation).
    _EXCLUDED_PREFIXES: tuple[str, ...] = (
        "/health",
        "/a2a",
        "/admin",
        "/webhooks",
        "/auth",
        "/docs",
        "/openapi.json",
        "/redoc",
    )

    # Inclusion list: prefix → resource_type. Mutations on a path matching one of
    # these prefixes (and NOT excluded above) produce an audit row.
    AUDITED_ROUTES: dict[str, str] = {
        "/initiatives": "initiative",
        "/workflows": "workflow",
        "/content": "content",
        "/campaigns": "campaign",
        "/reports": "report",
        "/vault": "vault_item",
        "/integrations": "integration",
        "/briefing": "briefing",
        "/pages": "page",
        "/community": "community_post",
        "/finance": "finance_item",
        "/sales": "sales_item",
        "/support": "support_ticket",
        "/compliance": "compliance_item",
        "/learning": "learning_item",
        "/api_credentials": "api_credential",
        "/byok": "byok_credential",
        "/journeys": "journey",
        "/ad_approvals": "ad_approval",
        "/email_sequences": "email_sequence",
        "/files": "file",
        "/onboarding": "onboarding",
        "/teams": "workspace",
        "/governance": "governance",
        "/account": "account",
        "/configuration": "configuration",
        "/data_io": "data_io",
        "/departments": "department",
        "/kpis": "kpi",
        "/monitoring_jobs": "monitoring_job",
        "/outbound_webhooks": "outbound_webhook",
        "/self_improvement": "self_improvement",
        "/workflow_triggers": "workflow_trigger",
        "/app_builder": "app_builder",
    }


    def _resolve_resource_type(path: str) -> str | None:
        """Return the resource_type for an audited path, or None if not audited."""
        for prefix in _EXCLUDED_PREFIXES:
            if path.startswith(prefix):
                return None
        for prefix, resource_type in AUDITED_ROUTES.items():
            if path == prefix or path.startswith(f"{prefix}/"):
                return resource_type
        return None


    def _extract_resource_id(path: str, prefix: str) -> str | None:
        """Extract the first path segment after the prefix, or None for collection routes."""
        remainder = path[len(prefix):].strip("/")
        if not remainder:
            return None
        return remainder.split("/")[0] or None


    def _resolve_actor(request: Request) -> str | None:
        """Best-effort actor resolution from the Authorization header. Returns user_id or None."""
        auth = request.headers.get("authorization") or request.headers.get("Authorization")
        if not auth or not auth.lower().startswith("bearer "):
            return None
        token = auth.split(None, 1)[1].strip()
        if not token:
            return None
        try:
            claims = verify_token_fast(token)
        except Exception as exc:
            logger.debug("audit_log: token verify failed: %s", exc)
            return None
        if not claims:
            return None
        return claims.get("sub")


    async def _fire_log_event(
        user_id: str,
        action_type: str,
        resource_type: str,
        resource_id: str | None,
        method: str,
        path: str,
        status_code: int,
        ip_address: str | None,
    ) -> None:
        """Background task: call GovernanceService.log_event swallowing all errors."""
        try:
            governance = get_governance_service()
            await governance.log_event(
                user_id=user_id,
                action_type=action_type,
                resource_type=resource_type,
                resource_id=resource_id,
                details={
                    "method": method,
                    "path": path,
                    "status_code": status_code,
                },
                ip_address=ip_address,
            )
        except Exception as exc:
            # GovernanceService.log_event already swallows internally, but defence-in-depth:
            logger.error("audit_log: background log_event failed: %s", exc)


    class AuditLogMiddleware(BaseHTTPMiddleware):
        """ASGI middleware that audits successful mutating requests.

        Placed AFTER request logging and onboarding guards in the middleware stack
        so it sees the final response status code.
        """

        async def dispatch(
            self,
            request: Request,
            call_next: Callable[[Request], Awaitable[Response]],
        ) -> Response:
            response = await call_next(request)

            try:
                if request.method not in _MUTATING_METHODS:
                    return response
                if response.status_code < 200 or response.status_code >= 300:
                    return response

                path = request.url.path
                resource_type = _resolve_resource_type(path)
                if resource_type is None:
                    return response

                user_id = _resolve_actor(request)
                if not user_id:
                    return response

                # Find the matching prefix to extract resource_id
                matched_prefix = next(
                    (p for p in AUDITED_ROUTES if path == p or path.startswith(f"{p}/")),
                    None,
                )
                resource_id = (
                    _extract_resource_id(path, matched_prefix) if matched_prefix else None
                )

                verb = _METHOD_VERBS.get(request.method, "modified")
                action_type = f"{resource_type}.{verb}"

                ip_address = request.client.host if request.client else None

                # Fire-and-forget so the audit insert does not block the response.
                asyncio.create_task(
                    _fire_log_event(
                        user_id=user_id,
                        action_type=action_type,
                        resource_type=resource_type,
                        resource_id=resource_id,
                        method=request.method,
                        path=path,
                        status_code=response.status_code,
                        ip_address=ip_address,
                    )
                )
            except Exception as exc:
                # MUST NEVER raise — middleware failure cannot break the request flow.
                logger.error("audit_log: middleware dispatch error: %s", exc)

            return response
    ```
