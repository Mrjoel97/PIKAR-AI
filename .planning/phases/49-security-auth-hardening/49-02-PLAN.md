<objective>
Add a reusable React `RootErrorBoundary` class component and wire it into the root and `(personas)` layouts so that when ANY deeply-nested client component throws during render, the user sees a meaningful error UI with a recovery option instead of a blank screen.

Purpose: AUTH-02 closes the second-largest production-readiness gap. Today the app has:
- `frontend/src/app/error.tsx` — only catches errors that bubble up to a route segment boundary, not arbitrary client crashes
- `frontend/src/app/global-error.tsx` — only catches root-layout-level errors
- `frontend/src/components/ui/DashboardErrorBoundary.tsx` — exists but is NOT wired into any layout, so it currently has zero coverage on the app shell

The result: a single throwing client component (chart, widget, persona dashboard cell) crashes the entire React tree to a blank page, with no recovery option for the user. This task adds layout-level component error boundaries so a single broken widget never blanks the screen.

Output:
- `frontend/src/components/errors/RootErrorBoundary.tsx` — class component error boundary
- `frontend/src/app/layout.tsx` — wraps `<PersonaProvider>` children in the boundary
- `frontend/src/app/(personas)/layout.tsx` — wraps persona route children in the boundary
- `frontend/__tests__/RootErrorBoundary.test.tsx` — Vitest coverage
</objective>

<critical_decisions>
1. **Class component, not hook** — React still requires error boundaries to be class
   components. There is no `useErrorBoundary` hook in stable React 19 (only in third-party
   `react-error-boundary` library, which we are NOT adding as a dependency).

2. **Two boundary layers, not one** — root layout + personas layout. This means a crash in
   a persona's dashboard resets at the persona layer (preserving providers). A crash in
   the providers themselves resets at the root layer.

3. **`componentDidCatch` MUST log to console** — the verification command checks for
   `console.error` being called with the component stack. In production this is also where
   Sentry instrumentation will hook in (Phase 51 OBS-01 — leave a TODO comment marking the
   integration point).

4. **Reset key** — accept an optional `resetKeys` prop so wrapping layouts can pass the
   pathname; when the pathname changes, the boundary auto-resets. Use
   `componentDidUpdate` to compare prevProps.resetKeys with this.props.resetKeys.

5. **Do NOT delete `DashboardErrorBoundary.tsx`** — it remains for granular use inside
   widget grids. This plan ADDS the new layout-level boundary.

6. **Tailwind 4 utility classes** — use slate/indigo/red palette consistent with existing
   error.tsx (verified during planning research). Do NOT introduce shadcn/ui (project does
   not use it).
</critical_decisions>

  <action>
    Create `frontend/src/components/errors/RootErrorBoundary.tsx`:

    ```tsx
    'use client';

    // Copyright (c) 2024-2026 Pikar AI. All rights reserved.
    // Proprietary and confidential. See LICENSE file for details.

    import React, { Component, type ReactNode, type ErrorInfo } from 'react';
    import { AlertTriangle } from 'lucide-react';

    interface Props {
      children: ReactNode;
      /** Optional keys whose change auto-resets the boundary (e.g. [pathname]). */
      resetKeys?: ReadonlyArray<unknown>;
      /** Optional fallback title override. */
      fallbackTitle?: string;
    }

    interface State {
      hasError: boolean;
      error: Error | null;
    }

    /**
     * Reusable React 19 error boundary used at the layout level.
     *
     * Catches render-time errors in any descendant client component and renders a
     * recovery UI with "Try Again" (resets local state) and "Go to Dashboard"
     * (hard navigation fallback). Auto-resets when any value in `resetKeys` changes.
     *
     * TODO(phase-51 OBS-01): wire Sentry.captureException(error, { extra: errorInfo })
     * inside componentDidCatch once Sentry SDK lands.
     */
    export class RootErrorBoundary extends Component<Props, State> {
      state: State = { hasError: false, error: null };

      static getDerivedStateFromError(error: Error): State {
        return { hasError: true, error };
      }

      componentDidCatch(error: Error, errorInfo: ErrorInfo): void {
        // Log the full component stack so it can be diagnosed in browser DevTools.
        // In production this is also the Sentry hook point (Phase 51).
        console.error(
          'RootErrorBoundary caught an error:',
          error,
          errorInfo.componentStack,
        );
      }

      componentDidUpdate(prevProps: Props): void {
        if (!this.state.hasError) return;
        const prev = prevProps.resetKeys ?? [];
        const next = this.props.resetKeys ?? [];
        if (prev.length !== next.length) {
          this.handleRetry();
          return;
        }
        for (let i = 0; i < prev.length; i++) {
          if (prev[i] !== next[i]) {
            this.handleRetry();
            return;
          }
        }
      }

      handleRetry = (): void => {
        this.setState({ hasError: false, error: null });
      };

      render(): ReactNode {
        if (!this.state.hasError) {
          return this.props.children;
        }

        return (
          <div className="min-h-[60vh] flex items-center justify-center p-6">
            <div className="max-w-md w-full text-center">
              <div className="w-16 h-16 mx-auto mb-6 rounded-2xl bg-red-100 dark:bg-red-900/30 flex items-center justify-center">
                <AlertTriangle className="w-8 h-8 text-red-600 dark:text-red-400" />
              </div>
              <h1 className="text-2xl font-bold text-slate-800 dark:text-slate-100 mb-2">
                {this.props.fallbackTitle ?? 'Something went wrong'}
              </h1>
              <p className="text-slate-500 dark:text-slate-400 mb-6 text-sm">
                A part of this page failed to load. Your data is safe.
              </p>
              <div className="flex gap-3 justify-center">
                <button
                  type="button"
                  onClick={this.handleRetry}
                  className="px-5 py-2.5 bg-indigo-600 text-white text-sm font-medium rounded-xl hover:bg-indigo-700 transition-colors"
                >
                  Try again
                </button>
                <a
                  href="/dashboard"
                  className="px-5 py-2.5 bg-slate-200 dark:bg-slate-800 text-slate-700 dark:text-slate-300 text-sm font-medium rounded-xl hover:bg-slate-300 dark:hover:bg-slate-700 transition-colors"
                >
                  Go to Dashboard
                </a>
              </div>
            </div>
          </div>
        );
      }
    }

    export default RootErrorBoundary;
    ```
