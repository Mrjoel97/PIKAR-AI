    - Test: Moderate financials produce score 40-70 (yellow)
    - Test: Missing data (no records) produces a fallback score with "insufficient data" explanation
    - Test: Score factors include revenue_trend (weight 0.25), runway_months (0.25), cash_flow_ratio (0.20), collection_rate (0.15), burn_stability (0.15)
  </behavior>
  <action>
    1. Create migration `20260410200000_financial_health_score.sql`:
       - `financial_health_snapshots` table: id (UUID PK), user_id (UUID FK auth.users ON DELETE CASCADE), score (INTEGER CHECK 0-100), color (TEXT CHECK 'green'/'yellow'/'red'), explanation (TEXT), factors (JSONB), computed_at (TIMESTAMPTZ DEFAULT now())
       - RLS: user can SELECT own, service_role full access
       - Index on (user_id, computed_at DESC)
       - updated_at trigger

    2. Create `app/services/financial_health_score_service.py`:
       - Class `FinancialHealthScoreService` extending BaseService
       - Method `async compute_health_score(user_id: str) -> dict` that:
         a. Calls get_revenue_stats for current_month AND last_month to compute revenue trend
         b. Calls get_cash_position for cash flow ratio (inflows/outflows)
         c. Calls get_burn_runway_report for runway and burn stability
         d. Queries invoices table for collection_rate (paid count / total non-draft count)
         e. Computes weighted score: revenue_trend (25%) + runway_months (25%) + cash_flow_ratio (20%) + collection_rate (15%) + burn_stability (15%)
         f. Score mapping: runway >= 12mo = 100pts, 6-12mo = 70pts, 3-6mo = 40pts, <3mo = 10pts; similar brackets for other factors
         g. Color: >= 70 = "green", 40-69 = "yellow", < 40 = "red"
         h. Explanation: plain-English sentence like "Your financial health is strong. Revenue is growing and you have 8+ months of runway." or "Warning: Your burn rate exceeds revenue and runway is under 3 months."
         i. Returns: {"score": int, "color": str, "explanation": str, "factors": {"revenue_trend": float, "runway_months": float, "cash_flow_ratio": float, "collection_rate": float, "burn_stability": float}, "computed_at": str}
       - Method `async save_snapshot(user_id: str, result: dict) -> dict` to persist to financial_health_snapshots
       - Method `async get_latest_snapshot(user_id: str) -> dict | None` to retrieve most recent

    3. Write tests in `tests/unit/services/test_financial_health_score_service.py`:
       - Mock FinancialService, get_cash_position, get_burn_runway_report, and invoices query
       - Test all score brackets (green/yellow/red)
       - Test insufficient data fallback
       - Test factor weight computation

    Use `from app.services.supabase_async import execute_async` for DB queries. Follow project async patterns. Use `from app.services.request_context import get_current_user_id` where needed.
