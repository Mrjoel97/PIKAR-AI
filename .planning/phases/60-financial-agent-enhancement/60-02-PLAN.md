Build automatic expense categorization for Stripe transactions so charges and payouts are classified into business categories (marketing, SaaS tools, COGS, payroll, etc.) without manual tagging.

Purpose: FIN-02 requires users to see their expenses broken down by category automatically. The financial_records table already has a `category` column but it's never populated by the Stripe sync. This plan adds a categorization engine and wires it into the sync pipeline.

Output: ExpenseCategorizationService + migration + sync integration + tests

    2. Create `app/services/expense_categorization_service.py`:
       - Class `ExpenseCategorizationService`
       - Constant `KEYWORD_RULES: dict[str, list[str]]` mapping category names to keyword patterns:
         ```
         "marketing": ["google ads", "facebook ads", "meta ads", "tiktok ads", "linkedin ads", "mailchimp", "sendgrid", "hubspot marketing", "semrush", "ahrefs"],
         "saas_tools": ["slack", "notion", "github", "vercel", "figma", "canva", "zapier", "airtable", "jira", "confluence", "1password", "dropbox"],
         "payroll": ["gusto", "payroll", "salary", "wages", "adp", "rippling"],
         "infrastructure": ["aws", "amazon web services", "gcp", "google cloud", "heroku", "digitalocean", "cloudflare", "datadog", "sentry"],
         "professional_services": ["legal", "accounting", "consulting", "lawyer", "attorney", "cpa", "bookkeeper"],
         "office": ["office", "coworking", "wework", "rent", "utilities", "internet"],
         "travel": ["airline", "hotel", "uber", "lyft", "airbnb", "flight"],
         "taxes_fees": ["stripe fee", "tax", "irs", "state tax", "processing fee"],
         "cogs": ["manufacturing", "materials", "shipping", "fulfillment", "warehouse"],
         "transfers": []  # assigned by transaction_type, not keywords
         ```
       - Method `categorize_transaction(description: str, transaction_type: str, metadata: dict | None = None) -> str`:
         a. If transaction_type == "payout" -> return "transfers"
         b. If transaction_type == "revenue" -> return "revenue"
         c. If transaction_type == "fee" -> return "taxes_fees"
         d. Lowercase description, check against KEYWORD_RULES patterns (case-insensitive substring match)
         e. If metadata has stripe_type == "stripe_fee" -> return "taxes_fees"
         f. Default to "other"
       - Method `async categorize_batch(user_id: str, limit: int = 500) -> dict`:
         a. Query financial_records WHERE user_id = user_id AND category IS NULL AND source_type = 'stripe' LIMIT limit
         b. For each record, call categorize_transaction
         c. Batch UPDATE the category column
         d. Return {"categorized": N, "skipped": N}
       - Method `async categorize_single(record: dict) -> str`:
         a. Convenience wrapper for one record

    3. Write tests:
       - Test categorize_transaction with known keywords (each category has at least one test)
       - Test transaction_type overrides (payout, revenue, fee)
       - Test unknown description returns "other"
       - Test categorize_batch with mocked DB (mock execute_async)
