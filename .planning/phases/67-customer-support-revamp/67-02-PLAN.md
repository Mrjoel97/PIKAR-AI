    async def find_similar_resolved_tickets(self, min_count: int = 3, user_id: str | None = None) -> list[dict]:
        """Find groups of resolved tickets with similar subjects.

        Queries resolved/closed tickets, groups by normalized subject prefix (first 50 chars lowercase),
        and returns groups with >= min_count tickets. Each group is a dict with:

    **2. Add `draft_customer_response` tool** to `app/agents/customer_support/tools.py`:

    ```python
    async def draft_customer_response(
        scenario: str, context: str, customer_name: str = "Customer"
    ) -> dict:
        """Draft a professional customer-facing response for common support scenarios.

        Args:
            scenario: Type of scenario - 'refund', 'shipping_delay', 'complaint', 'follow_up', 'apology', or 'general'.
            context: Specific details about the customer's situation.
            customer_name: Customer's name for personalization.

        Returns:
            Dict with subject, body, tone, and scenario fields.
        """
    ```

    Implementation: Use template-based drafting (no LLM call needed -- the agent itself is the LLM). Build templates for each scenario:
    - `refund`: Empathetic acknowledgment, clear refund process steps, timeline expectation
    - `shipping_delay`: Apology, tracking information prompt, compensation offer template
    - `complaint`: Validation of frustration, specific action plan, follow-up commitment
    - `follow_up`: Status update structure, next steps, open invitation
    - `apology`: Sincere acknowledgment, root cause (placeholder), prevention steps
    - `general`: Professional greeting, context-aware body, clear next steps

    Each template uses `customer_name` and `context` for personalization. Returns `{"success": True, "draft": {"subject": ..., "body": ..., "tone": ..., "scenario": ...}}`.

    **3. Add `suggest_faq_from_tickets` tool** to `app/agents/customer_support/tools.py`:

    ```python
    async def suggest_faq_from_tickets(min_similar: int = 3) -> dict:
        """Analyze resolved tickets for patterns and suggest FAQ entries.

        Looks for 3+ resolved tickets with similar subjects and auto-generates
        FAQ content from the resolution patterns.

        Args:
            min_similar: Minimum number of similar resolved tickets to trigger suggestion (default 3).

        Returns:
            Dict with faq_suggestions list, each containing title, content, source_ticket_count.
        """
    ```

    Implementation: Call `SupportTicketService().find_similar_resolved_tickets(min_count=min_similar)`. For each group, generate a FAQ entry:
    - `title`: "How to resolve: {subject_pattern}" (cleaned up)
    - `content`: Synthesize from resolution texts of the grouped tickets. Join unique resolutions, format as numbered steps.
    - `source_ticket_count`: Number of tickets in the group
    - `source_ticket_ids`: List of ticket IDs

    If no groups found, return `{"success": True, "faq_suggestions": [], "message": "Not enough similar resolved tickets to suggest FAQ entries. Need at least {min_similar} similar resolved tickets."}`.
