## Cross-Phase Integration

**86/88 integration paths wired correctly**

### Integration Gaps (2)

**Gap 1: `monitoring.alert` event type not routable**
- **Path:** Phase 46 `MonitoringJobService` → Phase 45 `NotificationDispatcher`
- **Root cause:** `"monitoring.alert"` absent from `NotificationRuleService.SUPPORTED_EVENTS` (line 41)
- **Impact:** Monitoring alert notifications silently fail — `dispatch_notification` returns `{}`, no Slack/Teams delivery
- **Affects:** INTEL-04
- **Fix:** Add `{"type": "monitoring.alert", "label": "Monitoring Alert"}` to `SUPPORTED_EVENTS`

**Gap 2: `agent.message` event type not routable**
- **Path:** Phase 45 `test-notification` endpoint → Phase 45 `NotificationDispatcher`
- **Root cause:** `"agent.message"` absent from `NotificationRuleService.SUPPORTED_EVENTS` (line 41)
- **Impact:** "Send Test Notification" button always returns 502 error — users can't verify connectivity
- **Affects:** NOTIF-01, NOTIF-02, NOTIF-06
- **Fix:** Add `{"type": "agent.message", "label": "Agent Message"}` to `SUPPORTED_EVENTS`

**Both gaps share a single root cause:** 2 missing entries in one file (`app/services/notification_rule_service.py`). Estimated fix: <5 minutes.
