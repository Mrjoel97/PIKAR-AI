### Workspace receiver-side persistence (post-v10.0 hotfix)
- After v10.0 phases were marked complete, an in-flight bug surfaced: live agent output via `WORKSPACE_ITEMS_EVENT` was not being persisted to localStorage by the receiver, causing canvas contents to disappear on reload/DevTools-toggle. Fixed in commit `10a3db3e` (frontend-only) with 3 new vitest persistence tests. Riding on top of v10.0 in the production deploy. Not tracked as a Phase 88/89 gap because it surfaced during use after both phases shipped.

### deploy-fast.ps1 probe-arg quoting (production drift root-cause)
- `scripts/deploy-fast.ps1`'s `--startup-probe` arg quoting produced a corrupted probe path on Cloud Run revision `pikar-ai-00078-f2q` (path field contained the entire probe arg string with spaces instead of just `/health/live`). Production traffic was manually pinned to `pikar-ai-00077-k2p` until 2026-05-02 when the audit's deploy via `gcloud run deploy --source` + explicit traffic routing replaced it with `pikar-ai-00080-5pp`. **Should be fixed in a follow-up phase** — until then, the fast-deploy script will continue producing unbootable revisions.
