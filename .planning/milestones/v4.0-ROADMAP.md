# Milestone v4.0: Production Scale & Persona UX

**Status:** SHIPPED 2026-04-03
**Phases:** 24, 25, 26, 27, 27.1, 28, 29, 30, 31

## Overview

v4.0 hardened the backend for multi-replica Cloud Run production (multi-worker server, distributed SSE slot tracking, async Supabase connection pooling, fail-fast production guards, input sanitization and security headers) and delivered persona-differentiated frontend UX (agent equalization so all personas access all agents, persona-ordered navigation, persona-specific default widgets, shell fade-in animations, and empty states with section headers).

## Milestone Summary

**Decimal Phases:**

- Phase 27.1: Input Sanitization & Security Hardening (inserted between 27 and 28)

**Key Decisions:**

- Gunicorn preload_app=False to preserve asyncio loop isolation for ADK Runner and StitchMCP.
- SSEAcquireResult subclasses tuple for backward compatibility while adding .reason for 503/429 discrimination.
- AsyncSupabaseService uses async classmethod get_instance() because create_async_client is async.
- Production raises RuntimeError on InMemory fallback instead of silently degrading.
- stdlib html.escape() exclusively — no third-party sanitization library.
- Persona differentiates behavior (routing priorities, prompt fragments), not availability — ALL_AGENT_NAMES is the single source of truth.
- Rate limits (10/30/60/120 per minute) are the only hard persona differentiator.

**Delivered:**

- Multi-worker app server (Gunicorn + uvicorn workers)
- Distributed SSE slot tracking across Cloud Run replicas
- Native async Supabase client with connection pooling
- Fail-fast production startup guards
- Input sanitization + CSP/security headers
- Persona agent equalization (all 12 agents for all 4 personas)
- Persona-ordered sidebar navigation
- Per-persona default widgets with localStorage dismissal
- Shell header fade-in animations
- Persona empty states and section headers
