# Shared Intelligence Infrastructure — Plan 117-04: Marketing Regression Guardrails

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Capture a pre-117 baseline of every active Marketing downstream workflow (campaign-creation wizard, ad-platform activation, summarizer output, attribution view, social analytics view, A/B email path) and lock it in as a regression test suite that runs on every change to Marketing-Agent surfaces. Replay the baseline after 117-01/02/03 are in place to prove that **shape-additive** changes (new `confidence`, `band`, claim emission, cache wrapping) have not regressed any pre-existing workflow. Marketing has the highest active-workflow surface of any agent in the 114-122 rollout, so this plan is the ONE phase with a dedicated regression-prevention sub-plan.

**Architecture:** Three complementary mechanisms:

1. **Snapshot tests** — golden JSON files capturing each tool's exact response shape (keys, types, presence of confidence/band, source provenance) for canonical inputs. Snapshots are committed to `tests/snapshots/marketing/` and assert byte-identical structure (with documented additive deltas).
2. **Integration replay** — a recorded transcript of the campaign-creation wizard (6 steps), the email A/B path, and the publishing-strategy path. Each transcript is a list of `(user_message, expected_agent_response_fragment, expected_tool_calls)` triples; the test replays against the real `marketing_agent` instance.
3. **Baseline captured-pre-117 fixtures** — JSON files in `tests/fixtures/marketing_baseline/` containing pre-Plan-117 output of the four cache-wrapped + claim-emitting tools. The replay asserts that the *user-visible* fields (summary_text, recommendation_text, per-row arrays) are byte-identical to the baseline; new additive fields (confidence, band, source='graph_cache') are tolerated; removed or renamed fields fail loudly.

**Tech Stack:** `tests/snapshots/marketing/` (new dir, JSON snapshots), `tests/fixtures/marketing_baseline/` (new dir, captured pre-117 outputs), `tests/integration/test_marketing_regression_replay.py` (new), `tests/integration/test_marketing_snapshot_shape.py` (new), `pytest --snapshot-update` workflow via the `syrupy` dependency (already in `pyproject.toml` per Phase 110-04 lineage; confirm in Task 1).

**Spec reference:** `docs/superpowers/specs/2026-05-19-shared-intelligence-infra-114-122-rolling-adoption-design.md` § Phase 117 — "the only phase with a dedicated regression-prevention plan because Marketing has the most active downstream workflows."

**Out of scope:** Frontend visual regression (screenshots of `/dashboard/marketing` are an Operations-Agent concern; this plan covers backend/tool-shape only). Performance regression (Plan 117-02 already ships the perf test). E2E browser tests via Playwright (deferred — out-of-band manual smoke is acceptable for this plan because the integration replay covers the full ADK loop). Backfill of historical agent transcripts (start from now). Self-improvement engine integration tests (covered by Plan 117-01's audit; engine itself is not changed by Phase 117).

## Pre-flight context

**The capture problem.** This plan's main ambiguity is *how* to capture the baseline. There are two failure modes:

1. **Capture too late** — running the capture after 117-01/02/03 have landed means the "baseline" is already the new shape. The baseline must come from a commit *before* the first 117 plan landed.
2. **Capture from stale state** — running the capture in an environment without a real campaign / real Shopify connection produces an empty JSON that proves nothing.

**Resolution.** The capture is performed by checking out the commit immediately preceding the first 117-01 commit, running a capture script against a known fixture user (`user_id=baseline-117-fixture`) seeded with synthetic-but-realistic data, and exporting the tool responses verbatim to JSON. The capture is committed as fixture data and then the working branch returns to head. The capture script lives in `scripts/capture_marketing_baseline.py` (created during Task 2). This means **Task 2 must be executed before any of 117-01/02/03 is merged**, but the rest of this plan (Tasks 3-7) can land after 117-03 because they consume the already-captured fixtures.

**Acceptance bar:**
- Baseline fixtures captured from the pre-117 codebase and committed
- Snapshot tests cover every tool that 117-01/02/03 modified, asserting:
  - All pre-117 keys still present, same Python types
  - New additive keys `confidence`, `band` present and typed correctly
  - No keys removed
- Replay test for the campaign-creation wizard passes against the post-117 `marketing_agent`
- A/B email path test and publishing-strategy test pass
- Self-improvement engine read surface unchanged (verified via the audit from Plan 117-01)
- Lint + format clean
- Full plan-117 acceptance sign-off completed and committed in the merge commit

from __future__ import annotations

import json
from pathlib import Path

import pytest

BASELINE_DIR = Path("tests/fixtures/marketing_baseline")

# Map baseline-file -> required additive keys after Plan 117
ADDITIVE_KEYS = {
    "summarize_campaign_performance.pre_117.json": {"confidence", "band"},
    "get_cross_channel_attribution.pre_117.json": {"confidence", "band"},
    "get_social_analytics_instagram.pre_117.json": {"confidence", "band"},
    "refresh_ad_performance.pre_117.json": {"confidence", "band"},
}


def _expected_value_types(payload: dict) -> dict[str, type]:
    return {k: type(v) for k, v in payload.items()}


@pytest.mark.parametrize("baseline_file", list(ADDITIVE_KEYS.keys()))
@pytest.mark.asyncio
async def test_response_is_superset_of_pre_117_baseline(baseline_file):
    """Post-117 tool responses must preserve every pre-117 key + type."""
    from unittest.mock import AsyncMock, patch

    baseline = json.loads((BASELINE_DIR / baseline_file).read_text())
    expected_types = _expected_value_types(baseline)
    must_have_new = ADDITIVE_KEYS[baseline_file]

    # Dispatch to the matching tool and patch the underlying service to
    # return the baseline payload. The tool's confidence-attachment +
    # claim-emission wrappers must be exercised end-to-end.
    if baseline_file == "summarize_campaign_performance.pre_117.json":
        from app.agents.tools.campaign_performance_tools import (
            summarize_campaign_performance,

            "app.agents.tools.campaign_performance_tools._resolve_user_campaign_entity",
            new=AsyncMock(return_value=None),
        ), patch(
            "app.agents.tools.campaign_performance_tools.emit_campaign_lift_claim",

            actual = await summarize_campaign_performance(days=7)
    elif baseline_file == "get_cross_channel_attribution.pre_117.json":
        from app.agents.tools.attribution_tools import get_cross_channel_attribution

            "app.agents.tools.attribution_tools._get_user_id", return_value="snap-1",

        enriched = dict(baseline)

    assert 0.0 <= float(actual["confidence"]) <= 1.0
    assert actual["band"] in {"low", "medium", "high"}
