# Research-Led Content Analysis

Use this reference whenever the skill is analyzing niches, competitors, social accounts, ads, landing pages, or viral content before creating anything.

## Goal

Convert raw market observation into reusable baselines for creation and testing.

## 1. Niche Map Template

Capture:
- audience segment
- urgent problem
- desired outcome
- language they use
- channels they trust
- objections they repeat
- offers they already respond to

## 2. Competitor Account Teardown

For each relevant account, capture:
- platform and handle
- profile positioning
- follower level if visible
- posting cadence
- format mix
- content pillars
- recurring hooks
- proof style
- CTA style
- strongest visible posts
- what appears to be working
- what appears weak

## 3. Viral Content Teardown

For each strong post, ad, or asset, capture:
- source URL or reference
- platform
- creator or brand
- topic angle
- audience segment
- hook type
- format
- structure beats
- proof mechanism
- emotional trigger
- CTA
- visible performance signal
- why it likely worked
- what can be adapted for Pikar-AI

## 4. Pattern Library Schema

Use a table like:

| Pattern Name | Source | Platform | Hook Type | Structure | Proof Device | CTA Style | Why It Works | Pikar-AI Adaptation |
|--------------|--------|----------|-----------|-----------|--------------|-----------|--------------|---------------------|

## 5. Baseline-First Testing

For each A/B test, state:
- the source pattern we are starting from
- the one variable we are changing
- the success metric
- the reason this variable is worth testing

This keeps testing interpretable and reduces guesswork.
